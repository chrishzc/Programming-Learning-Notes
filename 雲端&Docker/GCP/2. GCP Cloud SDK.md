# 2. GCP Cloud SDK 安裝與常用指令

> [!ABSTRACT]
> 本章介紹 Google Cloud SDK (CLI) 的安裝與初始化步驟，並整理最常使用的 `gcloud`（管理資源、專案與虛擬機）及 `gsutil`（操作 Cloud Storage 儲存桶與檔案）命令速查表。

---

## 一、 Cloud SDK 安裝與初始化

要在本地電腦控制 GCP 服務，需要安裝 Google Cloud CLI (Cloud SDK)。

### 1. 安裝步驟
1. **下載 SDK 檔案**：前往 [Google Cloud CLI 官方網站](https://cloud.google.com/sdk/docs/install) 下載對應作業系統的安裝檔。
2. **開啟並安裝**：執行安裝程式，並勾選安裝 bundled Python 或使用系統內建 Python。
3. **初始化登入**：安裝完成後會自動或手動執行初始化命令：
   ```bash
   gcloud init
   ```
4. **驗證與設定工作區**：
   - 瀏覽器會自動彈出，要求登入您的 Google 帳戶並授權。
   - 登入後，在終端機中選擇您要操作的 **GCP 專案 (Project ID)**。
   - 選擇預設的計算區域（Zone，如 `asia-east1-a`）。

## 二、 gcloud 核心指令速查

`gcloud` 用於管理 GCP 的專案、驗證身分、Compute Engine VM 等核心資源。

### 1. 初始化與驗證 (Auth & Init)

| 指令 | 說明 |
| :--- | :--- |
| `gcloud init` | 初始化 SDK，登入並選擇工作專案與區域 |
| `gcloud auth login` | 僅執行帳戶登入授權 |
| `gcloud auth application-default login` | 設定本地開發憑證 (ADC)，供 Python/Go 等 SDK 在本機運行時代用身分 |
| `gcloud auth revoke` | 撤銷當前帳戶的登入狀態（登出） |

### 2. 設定與專案管理 (Config & Projects)

| 指令 | 說明 |
| :--- | :--- |
| `gcloud config list` | 列出當前生效的帳戶、專案及區域設定 |
| `gcloud config set project <PROJECT_ID>` | 切換當前操作的 GCP 專案 |
| `gcloud projects list` | 列出當前帳戶有權限訪問的所有 GCP 專案 |

### 3. Compute Engine 虛擬機管理 (VM)

| 指令 | 說明 |
| :--- | :--- |
| `gcloud compute instances list` | 列出專案內所有的虛擬主機 (VM) |
| `gcloud compute instances start <VM_NAME> --zone=<ZONE>` | 啟動指定的虛擬主機 |
| `gcloud compute instances stop <VM_NAME> --zone=<ZONE>` | 停止指定的虛擬主機 |
| `gcloud compute ssh <VM_NAME> --zone=<ZONE>` | 透過 SSH 安全連線至指定的虛擬主機 |

## 三、 gsutil 儲存管理指令速查

`gsutil` 用於管理 Google Cloud Storage (GCS) 的儲存桶（Bucket）與其中的檔案（Object）。

### 1. 儲存桶管理 (Bucket)

| 指令 | 說明 |
| :--- | :--- |
| `gsutil mb gs://<BUCKET_NAME>` | 建立新的儲存桶（儲存桶名稱必須全球唯一） |
| `gsutil ls` | 列出專案中所有的儲存桶 |
| `gsutil rb gs://<BUCKET_NAME>` | 刪除空白的儲存桶 |

### 2. 檔案管理 (Objects)

| 指令 | 說明 |
| :--- | :--- |
| `gsutil ls gs://<BUCKET_NAME>` | 列出儲存桶內的所有檔案與資料夾 |
| `gsutil cp <LOCAL_FILE> gs://<BUCKET_NAME>` | 上傳本地檔案到儲存桶 |
| `gsutil cp gs://<BUCKET_NAME>/<FILE> <LOCAL_PATH>` | 從儲存桶下載檔案到本地 |
| `gsutil rm gs://<BUCKET_NAME>/<FILE>` | 刪除儲存桶中的檔案 |
| `gsutil rsync -r <LOCAL_DIR> gs://<BUCKET_NAME>` | 將本地目錄與儲存桶進行雙向或單向同步 |
| `gsutil acl ch -u AllUsers:R gs://<BUCKET_NAME>/<FILE>` | 將檔案設為「公開讀取」（所有人可透過 HTTP 連結下載） |
