# Docker 使用者常用指令

> [!ABSTRACT]
> 本章介紹 Docker 的核心概念，包括映像檔（Image）與容器（Container）的定義，並彙整了常用到的 Docker 使用者基本操作指令，包含版本查詢、映像檔下載與查詢、容器運行、進入與刪除等實用指令。

---

## 一、 Docker 核心概念

Docker 主要用於建立與還原隔離的虛擬運作環境，其核心由映像檔與容器組成：

### 1. 映像檔 (Image)
- **定義**：軟體的「懶人安裝包」或是作業系統的「唯讀光碟」。它包含了程式碼、執行時期環境、系統工具與函式庫，是唯讀的靜態模板。

### 2. 容器 (Container)
- **定義**：實際運行中的隔離環境。它是映像檔的動態實例，可被啟動、停止、刪除。
- **識別方式**：當容器開啟時，會自動獲得一組隨機且唯一的 ID。在呼叫或操作容器時，只能使用該容器的 **ID** 或 **Name (名稱)**。

> [!IMPORTANT]
> **Container 的 Port (埠號) 設定規則**
> 連接並進入容器時，必須設定 Port 轉發（對應）。
> - **主機端 Port**：可以自行設定與輸入（如 `-p 8080:...`）。
> - **容器端 Port**：一般由映像檔的作者在其內部代碼/Dockerfile 中規定（如網頁服務常固定為 `80`）。

## 二、 常用 Docker 操作指令

### 1. 指令速查表

| 功能目的             | 指令範例                                                    | 詳細說明                               |
| :--------------- | :------------------------------------------------------ | :--------------------------------- |
| **查詢 Docker 版本** | `docker --version`                                      | 查詢本機當前安裝 of Docker 版本資訊。           |
| **下載映像檔**        | `docker pull <image_name>`                              | 從 Docker Hub 中下載指定名稱的映像檔到本機。       |
| **查詢本機映像檔**      | `docker images`                                         | 列出目前本機已經下載的所有映像檔。                  |
| **查詢執行中容器**      | `docker ps`                                             | 列出目前正在運作中的所有容器列表。                  |
| **啟動容器**         | `docker run -d -p <host_port>:<container_port> <image>` | 以背景模式 (`-d`) 運行映像檔並綁定 Port (`-p`)。 |
| **進入容器**         | `docker exec -it <container_id_or_name> sh`             | 連進容器內部並開啟互動式 sh 終端機進行操作。           |
| **關閉容器**         | `docker stop <container_id_or_name>`                    | 正常停止執行中的容器。                        |
| **刪除容器**         | `docker rm [-f] <container_id_or_name>`                 | 刪除已停止的容器。加入 `-f` 可以強制刪除執行中的容器。     |

### 2. 原始指令代碼塊 (方便複製)

```bash
# 查詢版本
docker --version

# 下載目標名稱的項目
docker pull <image_name>

# 查詢有多少image
docker images

# 查詢有多少Container
docker ps

# 執行image變成container (-d: 背景執行, -p: 主機port:容器port)
docker run -d -p <host_port>:<container_port> <image_id_or_name>

# 連進container (使用 sh 終端)
docker exec -it <container_id_or_name> sh

# 關閉container
docker stop <container_id_or_name>

# 刪除container (-f 可以強制刪除執行中的容器)
docker rm [-f] <container_id_or_name>
```
