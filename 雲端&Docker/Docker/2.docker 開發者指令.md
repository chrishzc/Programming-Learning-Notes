# Docker 開發者指令與映像檔發佈

> [!ABSTRACT]
> 本章為 Docker 開發者導引，介紹如何將寫好的程式撰寫為 Dockerfile 並建置（Build）為映像檔；以及將映像檔登入、標記標籤（Tag）並推送（Push）發佈到 Docker Hub 的完整操作指令流程。

---

## 一、 建立 Docker 映像檔 (Build Image)

當您的程式編寫完成且準備分發給他人時，需要先建立一個 `Dockerfile` 設定檔，再將程式與其運作環境打包成映像檔（Image）。

### 本機建置映像檔指令
```bash
docker build -t <image_name>:<tag> .
```
*(備註：指令末端的 `.` 代表在當前目錄下尋找 Dockerfile 並進行建置)*

---

## 二、 發佈映像檔至 Docker Hub (Publish Image)

如果您希望將建置好的映像檔發佈到 Docker Hub，讓其他人可以透過 `docker pull` 下載，請依序執行以下步驟：

### 1. 登入 Docker Hub
在您的終端機（Terminal）中輸入登入指令，並輸入您的 Docker Hub 帳號與密碼進行驗證：
```bash
docker login
```

### 2. 為映像檔標記對應帳號的標籤 (Tag Image)
為了將映像檔關聯至您的個人 Docker Hub 倉庫，必須使用特定格式為映像檔打上標籤：
```bash
docker tag <原本的名字> <您的DockerHub帳號>/<Image名稱>:<版本號>
```

### 3. 推送映像檔到 Docker Hub (Push Image)
將標記好標籤的映像檔推送發佈至 Docker Hub：
```bash
docker push <您的DockerHub帳號>/<Image名稱>:<版本號>
```
