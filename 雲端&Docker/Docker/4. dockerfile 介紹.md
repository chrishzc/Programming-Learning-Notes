# Dockerfile 介紹與最佳實務

> [!ABSTRACT]
> 本篇筆記介紹 Dockerfile 的定義、核心建置工作流、常用語法指令（FROM, COPY, RUN, CMD 等），並提供 Python 應用的 Dockerfile 範例與映像檔建置優化技巧。

---

## 一、 什麼是 Dockerfile？

**Dockerfile** 是一個純文字檔案，裡面包含了一系列用來建立（Build）Docker 映像檔（Image）的指令與說明。你可以將它視為自動化建立映像檔的「腳本食譜」。

### 基本建置工作流：
```
[撰寫 Dockerfile] ➔ 執行 [docker build] (生成映像檔) ➔ 執行 [docker run] (運行容器)
```

---

## 二、 Dockerfile 常用指令速查

| 指令               | 說明                                                       | 範例                                      |
| :--------------- | :------------------------------------------------------- | :-------------------------------------- |
| **`FROM`**       | **指定基礎映像檔**。必須是 Dockerfile 的第一個非註釋指令。                    | `FROM python:3.9-slim`                  |
| **`WORKDIR`**    | **設定工作目錄**。後續的 RUN, CMD, COPY 等指令都會在此目錄下執行。              | `WORKDIR /app`                          |
| **`COPY`**       | **複製檔案**。將本地主機的檔案複製到映像檔中的指定路徑。                           | `COPY . .` 或 `COPY requirements.txt ./` |
| **`ADD`**        | 類似 COPY，但功能更強（支援從 URL 下載、自動解壓縮 tar 檔）。**一般建議優先使用 COPY**。 | `ADD source.tar.gz /app/`               |
| **`RUN`**        | **在映像檔建置（Build）階段執行的指令**。通常用於安裝套件。                       | `RUN pip install -r requirements.txt`   |
| **`ENV`**        | **設定環境變數**。該變數會持續存在於映像檔中。                                | `ENV PORT=8080`                         |
| **`EXPOSE`**     | **宣告對外連接埠**。僅作為說明，並不會自動開啟連接埠映射。                          | `EXPOSE 8080`                           |
| **`CMD`**        | **設定容器啟動（Run）時預設執行的指令**。可被 `docker run` 後的指令覆蓋。          | `CMD ["python", "app.py"]`              |
| **`ENTRYPOINT`** | **設定容器啟動時的執行入口**。不易被覆蓋，常與 CMD 搭配作為參數傳遞。                  | `ENTRYPOINT ["python"]`                 |

> [!IMPORTANT]
> **`CMD` vs `RUN` 的關鍵區別**：
> - `RUN` 發生在 **建置階段 (Build)**，執行指令並將變更寫入新的 Image Layer。
> - `CMD` 發生在 **運行階段 (Run)**，是容器啟動時的「預設指令」，在建置時不會執行。

> [!TIP]
> **`COPY` vs `ADD` 的選擇建議**：
> Docker 官方建議優先使用 `COPY`，因為其行為更單純、透明且好預測。
> - `ADD` 會自動解壓縮本地的 tar 壓縮檔（如 `ADD source.tar.gz /app/`）。若只需複製而不解壓，或複製普通檔案，應一律使用 `COPY`。
> - 避免使用 `ADD` 來下載遠端 URL 檔案，因為會將下載的暫存檔留存在映像檔的 Layer 中。下載遠端檔案應優先使用 `RUN curl` 或 `RUN wget` 並在同一指令中解壓與清理。

## 三、 Python 應用 Dockerfile 實戰範例

以下是一個標準的 Python Web 應用程式 Dockerfile 設定：

```dockerfile
# 1. 使用官方輕量版 Python 映像檔作為基礎
FROM python:3.9-slim

# 2. 設定容器內的工作目錄
WORKDIR /app

# 3. 先複製依賴定義檔（利於快取優化）
COPY requirements.txt ./

# 4. 安裝依賴套件
RUN pip install --no-cache-dir -r requirements.txt

# 5. 複製其餘的專案原始碼到容器中
COPY . .

# 6. 宣告容器運行時監聽的連接埠
EXPOSE 5000

# 7. 設定環境變數
ENV FLASK_APP=app.py

# 8. 設定啟動容器的指令
CMD ["flask", "run", "--host=0.0.0.0"]
```

## 四、 Dockerfile 優化與最佳實務

### 1. 利用 Layer Caching (快取機制) 優化建置速度
Docker 是以分層（Layer）方式建置映像檔的。如果某一層的指令與內容沒有改變，Docker 就會直接使用快取。
- **作法**：將「不易變動的指令」放在上面，「常變動的指令（如複製程式碼 `COPY . .`）」放在最下面。
- **範例**：如上節範例，先 `COPY requirements.txt` 並 `RUN pip install`，最後才 `COPY . .`。這樣只要 requirements.txt 沒改，每次修改程式碼時就不用重新下載安裝 Python 套件。

### 2. 合併 `RUN` 指令以減少 Layer 數量
每一個 `RUN`、`COPY`、`ADD` 指令都會建立一個新的唯讀 Layer，增加映像檔的體積。
- **作法**：將多個相關的 Linux 指令用 `&&` 合併為單一的 `RUN` 指令，並在結尾清理快取。
- **範例**：
  ```dockerfile
  RUN apt-get update && apt-get install -y \
      curl \
      git \
   && rm -rf /var/lib/apt/lists/*
  ```

### 3. 善用 `.dockerignore` 檔案
在專案根目錄下建立 `.dockerignore` 檔案，排除不需要複製進容器的檔案（如 `.git`、`node_modules`、`.venv`、測試報告等），可以大幅縮小建置上下文（Build Context）並減小映像檔體積。
```
# .dockerignore 範例
.git
.venv
__pycache__/
*.log
```
