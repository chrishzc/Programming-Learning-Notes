# Docker Compose 容器編排指引

> [!ABSTRACT]
> 本章介紹 Docker Compose 的核心概念與 `docker-compose.yml` 配置文件結構，提供常用多容器編排指令速查，並以「Python 網頁應用 + Redis 快取資料庫」雙服務為例進行部署實戰。

---

## 一、 什麼是 Docker Compose？

當應用程式需要多個服務協同運作（例如：前端網頁、後端 API、資料庫、快取系統）時，如果手動使用 `docker run` 逐一啟動，會面臨以下問題：
- 必須手動輸入冗長且複雜的網路與資料卷設定參數。
- 難以控管容器之間的啟動先後順序。
- 專案移轉到其他主機時，設定極易遺漏。

**Docker Compose** 是 Docker 官方提供的多容器定義與執行工具。它允許開發者透過單一的 YAML 格式設定檔，將所有服務的關聯性定義清楚，並使用簡單的單一指令完成整套系統的啟動與關閉。

---

## 二、 常用 Docker Compose 指令速查

Docker Compose 的所有指令都必須在包含 `docker-compose.yml` 檔案的目錄下執行。

> [!NOTE]
> 舊版的指令為 `docker-compose <命令>`，新版 Docker 已將其整合為 **`docker compose <命令>`**（中間為空白，無連字號）。建議統一使用新版語法。

| 功能目的      | 指令範例                             | 詳細說明                                 |
| :-------- | :------------------------------- | :----------------------------------- |
| **背景啟動**  | `docker compose up -d`           | 讀取設定檔，在背景下載並啟動所有容器服務（最常用）            |
| **停止並刪除** | `docker compose down`            | 停止運行中的容器，並清除建立的網路、預設資料卷              |
| **檢視狀態**  | `docker compose ps`              | 列出目前由 Compose 管理的所有容器及其狀態與通訊埠        |
| **即時日誌**  | `docker compose logs -f`         | 追蹤並顯示所有容器的即時輸出日誌（可用 `Ctrl + C` 結束）   |
| **進入容器**  | `docker compose exec <服務名> bash` | 進入指定服務容器的終端機執行互動式指令                  |
| **重啟服務**  | `docker compose restart`         | 重新啟動所有或指定的服務容器                       |
| **重新構建**  | `docker compose build`           | 重新讀取 Dockerfile 並編譯新的映像檔             |
| **檢查設定**  | `docker compose config`          | 驗證 `docker-compose.yml` 的語法與配置內容是否正確 |
|           |                                  |                                      |

## 三、 docker-compose.yml 核心結構解析

一個標準的設定檔主要由以下四個核心要素組成：

```yaml
services:       # 1. 定義要運行的各個容器服務
  web:
    build: .
    ports:
      - "5000:5000"
  db:
    image: postgres:15

volumes:        # 2. 定義持久化儲存的共享資料卷
  db_data:

networks:       # 3. 定義容器間通訊的虛擬網路
  app_net:
```

### 1. services (服務)
定義每個容器的配置。底下的子節點（如 `web`, `db`）為服務自訂名稱。
- `image`：指定要拉取的 Docker Hub 映像檔。
- `build`：指定本機 Dockerfile 所在路徑（`.` 代表當前目錄），由 Compose 自動建置。
- `ports`：對照主機與容器的埠號（格式為 `主機Port:容器Port`）。
- `volumes`：掛載資料卷或主機實體路徑（格式為 `資料卷名:容器內路徑`）。
- `environment`：傳入容器內的環境變數。
- `depends_on`：指定啟動的先後順序（例如 Web 服務必須在 db 啟動後才啟動）。

## 四、 雙服務部署實戰 (Flask + Redis)

本範例展示如何使用 Docker Compose 快速運行一個結合了 Redis 快取的 Python 網頁應用程式。

### 1. 準備應用程式代碼與 Dockerfile
在專案目錄下建立以下三個檔案：

#### 📝 [NEW] `app.py` (Python 程式碼)
```python
import time
import redis
from flask import Flask

app = Flask(__name__)
# 使用服務名稱 'redis' 作為 Host，Docker DNS 會自動解析為 Redis 容器的 IP
cache = redis.Redis(host='redis', port=6379)

def get_hit_count():
    retries = 5
    while True:
        try:
            return cache.incr('hits')
        except redis.exceptions.ConnectionError as exc:
            if retries == 0:
                raise exc
            retries -= 1
            time.sleep(0.5)

@app.route('/')
def hello():
    count = get_hit_count()
    return f'歡迎光臨！這是第 {count} 次瀏覽。\n'

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=5000)
```

#### 📝 [NEW] `Dockerfile` (定義 Web 映像檔)
```dockerfile
FROM python:3.10-slim
WORKDIR /code
RUN pip install flask redis
COPY app.py .
CMD ["python", "app.py"]
```

### 2. 撰寫 Docker Compose 設定檔

#### 📝 [NEW] `docker-compose.yml`
```yaml
services:
  web:
    build: .                    # 使用當前目錄的 Dockerfile 建置映像檔
    ports:
      - "8080:5000"             # 將主機的 8080 埠對應到容器的 5000 埠
    depends_on:
      - redis                   # 確保 redis 服務先啟動
    networks:
      - webnet

  redis:
    image: "redis:alpine"      # 使用輕量級的 Redis 官方映像檔
    networks:
      - webnet

networks:
  webnet:                       # 宣告一個自訂的虛擬網路，使 web 與 redis 可互相通訊
```

### 3. 部署與執行步驟

> [!TIP]
> **快速啟動指令**
> 1. 開啟終端機，切換至包含上述三個檔案的目錄。
> 2. 執行指令：
>    ```bash
>    docker compose up -d
>    ```
> 3. 開啟瀏覽器並造訪 `http://localhost:8080`，即可看到累加的瀏覽次數。
> 4. 若要停止所有服務，輸入：
>    ```bash
>    docker compose down
>    ```

## 五、 網路與資料持久化重要提醒

> [!WARNING]
> **預設網路機制**
> Compose 啟動時會預設建立一個專屬網路，並把所有服務放入該網路中。容器之間可直接使用 **服務名稱**（如上述 Flask 程式碼中呼叫的 `host='redis'`）作為主機名進行通訊，無需硬編碼 IP 位置。

> [!IMPORTANT]
> **資料持久化**
> 預設情況下，一旦執行 `docker compose down`，容器內的資料將會消失。若需要保留資料庫的資料，必須在 `docker-compose.yml` 中定義具名資料卷（Named Volumes），將資料庫路徑掛載出來：
> ```yaml
> services:
>   db:
>     image: postgres:15
>     volumes:
>       - db_data:/var/lib/postgresql/data
> 
> volumes:
>   db_data: # 宣告資料卷，使資料可在容器生命週期之外持久保存
> ```
