# Google Cloud Platform (GCP) 基礎建設指引

> [!ABSTRACT]
> 本章介紹 Google Cloud Platform (GCP) 的基礎建設與初始設定，包含專案（Project）架構、建立專案、設定預算警示以防止超支、配置專案權限（IAM），以及租用與進入 Compute Engine 虛擬主機（VM）的完整步驟。

---

## 一、 GCP 專案架構

在 GCP 中，資源管理的層級與架構如下：
- **組織與專案**：
  - **有公司組織**：專案（Project）會隸屬於該組織之下的某個部門或目錄。
  - **無公司組織**：專案獨立存在，適合個人開發者或測試用途。
  - 所有的雲端服務與資源（如虛擬主機、資料庫等）都必須建立在特定的**專案 (Project)** 中。

---

## 二、 專案建立與帳務權限管理

### 1. 建立專案 (Create Project)
- **步驟**：在 GCP 主頁頂部點選專案列表，選擇「建立專案」，並設定：
  - 專案名稱
  - 專案所属的組織或位置

### 2. 建立預算與警示 (Budgets & Alerts)
為了避免不必要的扣款與帳單超支，初次使用時必須設定預算上限。
- **路徑**：`選單 ➔ Billing ➔ Budgets & alerts ➔ Create budget`
- **說明**：在此設定預算額度，並設定在達到特定比例（如 50%、90%）時發送警示通知至信箱。

> [!TIP]
> 建議設定預算警示以確保能即時收到費用通知，避免忘記關閉虛擬主機而導致高額帳單。

### 3. 設定專案權限 (IAM)
- **路徑**：`選單 ➔ IAM & Admin ➔ IAM ➔ GRANT ACCESS`
- **操作步驟**：
  1. 進入 IAM 管理頁面。
  2. 確認左上角所處的 `Project` 專案是否正確。
  3. 點選 `GRANT ACCESS` 並輸入使用者帳號，賦予相應角色權限。

> [!IMPORTANT]
> **確認專案正確性**
> 在 IAM & Admin 設定權限時，務必確認左上角顯示的 Project 確實是您要調整的專案，以免誤向其他專案授予權限。

---

## 三、 虛擬主機 (Compute Engine) 租用與連線

### 1. 租用虛擬主機 (Create VM Instance)
- **路徑**：`選單 ➔ Compute Engine ➔ Create an instance`
- **可配置項目**：
  1. **主機型號與名稱**：根據運算需求選擇 CPU 與記憶體規格（如 e2-medium）。
  2. **儲存大小與作業系統**：選擇開機磁碟大小、硬碟類型（如 SSD）及 Linux 發行版（如 Ubuntu、Debian）。
  3. **是否備份**：是否啟用快照備份。
  4. **網卡**：配置靜態 IP、子網路及防火牆防火設定。
  5. **安全**：設定 SSH 金鑰及服務帳戶權限。

### 2. 連線至虛擬主機 (SSH 連線)
- **操作**：虛擬主機建立完畢且處於執行狀態後，在控制台的該主機旁點選 **SSH** 按鈕，即可彈出瀏覽器視窗並自動登入虛擬主機的終端機（Terminal）進行命令操作。
