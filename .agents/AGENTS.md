# 🦄 專案自定義行為準則 (Project Agent Rules)

- **預設啟用 Ponytail 模式**：在此專案/資料夾的所有任務中，預設啟用 `ponytail` 技能（強度設為 `full`）。除非使用者特別指令要求使用其他方式，否則編寫程式碼時請遵循「最懶（最精簡、行數最少）且能運作的方案」：
  1. 優先使用 Python 內建標準庫（Stdlib）與原生功能，避免引入不必要的第三方套件。
  2. 遵循 YAGNI（You Aren't Gonna Need It）原則，不預留未來擴充用的複雜抽象架構（如介面、多餘的類別包裝）。
  3. 優先考慮刪除代碼與精簡邏輯，拒絕過度設計。
