# 資料庫安全與使用者權限管控 (Database Security & User Access Control)

> [!ABSTRACT]
> 介紹 MySQL 安全機制，學習如何管理使用者帳號、建立密碼，以及授權與收回資料庫操作權限 (GRANT/REVOKE)。

---

在多人使用的資料庫環境中，必須要建立嚴密的存取與使用安全機制，主要透過**認證 (Authentication)** 與**授權 (Authorization)** 兩大核心來達成。

- ==**核心機制**：MySQL 主要是利用「使用者帳號」以及「權限對照」來精細管理資料庫的安全性。==

---

## 1. 系統安全與資料安全

資料庫的安全性控制可以分為兩個層級：

### 1.1 系統安全 (System Security)
- **定義**：控制資料庫系統層次的連線許可與伺服器使用權限。
- **範疇**：管理使用者帳號/密碼、磁碟空間的使用限制，以及使用者在整個資料庫伺服器系統內可以執行的全域性操作事項。

### 1.2 資料安全 (Data Security)
- **定義**：控制使用者對特定資料庫物件的存取與使用權限。
- **範疇**：限制使用者可以操作哪些資料表、視觀表、欄位，以及可以使用哪些操作方式（如僅能查詢 `SELECT` 或能修改 `UPDATE`）。

---

## 2. 使用者帳號管理 (User Account)

使用者必須利用一個合法的使用者帳號才能連線至 MySQL 伺服器。

### 2.1 帳號的組成格式
MySQL 中的使用者帳號由兩部分精密組成：`'UserName'@'HostName'`
- **`UserName`**：使用者名稱。
- **`HostName`**：指定該使用者**被允許登入的來源機器名稱或 IP 位址**。
  - **`%`**：代表萬用字元，允許從「任何遠端主機」登入。
  - **`localhost`**：代表「本機連線」，僅允許在 MySQL 伺服器本機進行登入。

> [!NOTE]
> **常見的帳號設定範例：**
> - `root@%`：表示可以在任何一台主機遠端登入的超級管理員帳號 `root`。
> - `jih@localhost`：表示只能在 MySQL 伺服器本機登入的帳號 `jih`。

### 2.2 建立使用者帳號語法
```sql
CREATE USER 'UserName'@'HostName' [IDENTIFIED BY 'Password'];
```

![[{474A92CD-43BF-4C34-9D8A-7270BFABD544}.png]]

> [!WARNING]
> **新帳號權限注意：**
> - 使用 `CREATE USER` 剛建立好的初始帳號，**預設沒有任何操作權限**（僅有基本的登入 `USAGE` 權限），必須經過後續的 `GRANT` 授權才能進行資料存取。

### 2.3 修改使用者帳號密碼
必須具備存取 `mysql` 資料庫的權限才能進行密碼變更：

```sql
ALTER USER 'UserName'@'HostName' IDENTIFIED BY 'New_Password';
```

### 2.4 刪除使用者帳號
```sql
DROP USER 'UserName'@'HostName';
```

## 3. MySQL 權限驗證與系統字典表

### 3.1 權限的兩大類型
1. **系統權限 (System Privileges)**：用於設定系統安全，指存取和管理資料庫伺服器本身的全域權利。
2. **物件權限 (Object Privileges)**：用於設定資料安全，指維護和操作特定資料表、視圖、索引等資料庫物件的權利。

### 3.2 權限儲存的系統資料表 (System Tables)
MySQL 帳號與權限的所有設定，都儲存在系統自帶的 `mysql` 資料庫中。當使用者連線或執行 SQL 時，伺服器會依據以下系統表進行分層驗證：

| 系統資料表名稱 (Table Name) | 儲存內容與權限驗證範疇 |
| :--- | :--- |
| **`user`** | 紀錄所有使用者帳號、密碼，以及在整個 Server 中的**全域性權限 (Global Privileges)**。 |
| **`db`** | 紀錄使用者對特定**資料庫等級 (Database-level)** 的存取權限。 |
| **`tables_priv`** | 紀錄使用者對特定**資料表等級 (Table-level)** 的操作權限。 |
| **`columns_priv`** | 紀錄使用者對特定**欄位等級 (Column-level)** 的精細操作權限。 |
| **`procs_priv`** | 紀錄使用者對特定**預存程序與函數 (Stored Procedure & Function)** 的執行權限。 |

## 4. 授權指令語法 (GRANT)

我們可以使用 `GRANT` 指令，將特定的權限授予指定的使用者帳號。

### 4.1 授權基礎語法
```sql
GRANT [ALL | privs [columns]] [, ...]
ON {table | * | *.* | database.*}
TO 'UserName'@'HostName' [, ...]
[WITH GRANT OPTION];
```
### 4.2 授權參數與選項說明
- **`ALL`** (或 `ALL PRIVILEGES`)：授予所有可用的物件或系統權限。
- **`privs`**：授予特定的個別權限（如 `SELECT`、`INSERT`、`CREATE` 等）。
- **`columns`**：將權限限制在資料表的「特定欄位」上（例如僅允許更新特定欄位）。
- **`ON`**：指定授權的範圍/等級：
  - `*.*`：全域等級（對所有資料庫的所有表格皆有效）。
  - `database.*`：資料庫等級（對該資料庫內的所有表格皆有效）。
  - `database.table`：資料表等級。
- **`TO`**：指定接收權限的使用者帳號。
- **`WITH GRANT OPTION`**：授予該使用者「可以將自己擁有的權限，再度授權給其他使用者」的超級權力。

> [!NOTE]
> - 在 MySQL 的預設安全規則中，**任何註冊的使用者均可存取以 `test` 開頭的資料庫**（如預設的 `test` 資料庫，或命名為 `test_db` 的庫）。

### 4.3 授權等級與範圍 (ON 子句設定)

透過 `ON` 子句，可以將權限限縮在不同的邏輯範圍層次：

| 授權等級 (Level) | 套用範圍與說明 (Scope) | 對應系統表 (System Table) | 授權與收回語法範例 (GRANT / REVOKE Examples) |
| :--- | :--- | :--- | :--- |
| **全域授權**<br>(Global Level) | 適用於伺服器中的所有資料庫及所有物件。 | `mysql.user` | `GRANT ALL ON *.* TO 'UserName'@'HostName';`<br>`REVOKE ALL ON *.* FROM 'UserName'@'HostName';` |
| **資料庫等級**<br>(Database Level) | 適用於特定資料庫中的所有物件。<br>（`*` 代表目前的資料庫） | `mysql.db` | `GRANT ALL ON database.* TO 'UserName'@'HostName';`<br>`REVOKE ALL ON database.* FROM 'UserName'@'HostName';` |
| **資料表等級**<br>(Table Level) | 適用於特定資料庫中的特定資料表。 | `mysql.tables_priv` | `GRANT ALL ON database.table TO 'UserName'@'HostName';`<br>`REVOKE ALL ON database.table FROM 'UserName'@'HostName';` |
| **欄位等級**<br>(Column Level) | 適用於資料表中指定的特定欄位。 | `mysql.columns_priv` | `GRANT SELECT(column) ON database.table TO 'UserName'@'HostName';`<br>`REVOKE SELECT(column) ON database.table FROM 'UserName'@'HostName';` |

## 5. 權限明細對照表

### 5.1 系統管理員/系統權限 (Administrative/System Privileges)

以下是用於管理資料庫伺服器系統安全的常見全域性權限：

| 系統權限 (Privilege) | 權限功能詳細說明 |
| :--- | :--- |
| **`ALL`** | 授予所有的系統與物件操作權限。 |
| **`CREATE USER`** | 允許新增、修改、刪除與重新命名使用者帳號。 |
| **`FILE`** | 允許讀取或寫入 MySQL 伺服器實體檔案系統的權限（如 `SELECT ... INTO OUTFILE`）。 |
| **`PROCESS`** | 允許查看伺服器中正在執行中的 Session / 執行緒資訊（使用 `SHOW PROCESSLIST`）。 |
| **`RELOAD`** | 允許重新加載權限表 (`FLUSH PRIVILEGES`)、刷新日誌及清理緩衝區。 |
| **`REPLICATION CLIENT`**| 允許查詢主/從伺服器的狀態。 |
| **`REPLICATION SLAVE`** | 允許運行一個鏡像從伺服器，為主從複製架構所需。 |
| **`SHOW DATABASES`** | 允許執行 `SHOW DATABASES` 指令來查看伺服器內的所有資料庫名稱。 |
| **`SHUTDOWN`** | 允許關閉 MySQL 資料庫伺服器主程式。 |
| **`SUPER`** | 超級管理權限，允許執行 `CHANGE MASTER TO`、`KILL` 連線、`PURGE BINARY LOGS` 等管理員指令。 |
| **`USAGE`** | **無任何特權**。新帳號預設自動獲得，僅代表該帳號擁有連線登入的權利。 |

### 5.2 資料庫使用者/物件權限 (Database/Object Privileges)

以下是針對特定資料庫物件（如表、視圖、預存程序等）進行操作的常見權限：

| 授權適用對象 (Object Scope) | 可授予的物件權限 (Privileges) |
| :--- | :--- |
| **Databases** (資料庫層級) | `CREATE`, `DROP`, `EVENT`, `LOCK TABLES`, `REFERENCES`, `GRANT OPTION` |
| **Tables** (資料表層級) | `SELECT`, `UPDATE`, `INSERT`, `DELETE`, `CREATE`, `ALTER`, `DROP`, `INDEX`, `TRIGGER`, `CREATE TEMPORARY TABLES`, `REFERENCES`, `GRANT OPTION` |
| **Columns** (資料表欄位層級) | `SELECT`, `UPDATE`, `INSERT` |
| **Views** (視觀表層級) | `CREATE VIEW`, `SHOW VIEWS`, `DROP` |
| **Indexes** (索引層級) | `CREATE`, `DROP` |
| **Stored Routines** (預存程序/函數) | `CREATE ROUTINE`, `ALTER ROUTINE`, `EXECUTE`, `GRANT OPTION` |


## 6. 權限檢索與回收管理

當授權完成後，可以使用以下指令來檢查及收回權限：

### 6.1 顯示使用者權限 (SHOW GRANTS)
可用於查詢特定使用者目前所擁有的所有權限列表：

```sql
SHOW GRANTS FOR 'UserName'@'HostName';
```

### 6.2 移除權限 (REVOKE)
收回特定使用者已被授予的特權：

```sql
REVOKE [ALL] privs[columns], ... 
ON {table | * | *.* | database.*} 
FROM 'UserName'@'HostName', ...;
```

> [!WARNING]
> **移除權限與刪除帳號：**
> - `REVOKE` 指令僅能收回該帳號所擁有的資料庫特權，**無法將使用者帳號從系統中刪除**。
> - 若要徹底刪除使用者帳號，請務必使用 `DROP USER` 指令。
