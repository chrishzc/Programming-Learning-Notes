# 資料分組與視窗函數 (Grouping Data & Window Functions)

> [!ABSTRACT]
> 學習使用 GROUP BY 與 HAVING 進行資料聚合分析，並掌握視窗函數 (Window Functions) 的進階分析應用。

---

## 1. 資料分組核心規則 (GROUP BY)

在使用 `GROUP BY` 進行資料分組時，請務必嚴格遵循以下兩點，否則會產生語法錯誤或邏輯混亂：

1. **非彙總必分組**：`SELECT` 子句中除了統計彙總函數（如 `SUM()`、`COUNT()`）之外，出現的任何**一般欄位**，都**必須**列在 `GROUP BY` 子句中。
2. **混合限制**：若 `SELECT` 中出現了彙總函數，則不得單獨出現與分組無關的一般欄位。

---

## 2. GROUP BY 語法與執行順序

標準 SQL 查詢語句中，各子句的排列與實際執行順序如下：

```sql
SELECT <column>, <group_function>
FROM <table>
[WHERE <condition>]        -- 1. 分組前的初次篩選
[GROUP BY <expression>]    -- 2. 進行分組
[HAVING <group_condition>] -- 3. 分組後的再次篩選
[ORDER BY <column>];       -- 4. 最後排序
```

> [!IMPORTANT]
> **分組與篩選邏輯：**
> - **分組邏輯**：指定欄位中相同的值會被自動歸類為同一組。若分組欄位中有 `NULL` 值，所有的 `NULL` 會被視為獨立的一組。
> - **多欄位分組**：可使用逗號分隔多個欄位（如 `GROUP BY dept_id, job_id`）。
> - **WHERE vs HAVING**：`WHERE` 是在分組**前**過濾單筆資料（不可使用聚合函數）；`HAVING` 是在分組**後**對「整個群組」進行過濾（通常配合聚合函數使用）。

---

## 3. 進階群組函數 (GROUP_CONCAT)

`GROUP_CONCAT` 用於將群組內的非 NULL 值連結成一個單一字串，非常適合用於「合併顯示同群組的多筆紀錄」。

### 3.1 語法結構
```sql
GROUP_CONCAT([DISTINCT] expr [,expr ...] 
             [ORDER BY {column | expr} [ASC | DESC]] 
             [SEPARATOR string])
```
- **`DISTINCT`**：去除群組中重複的值。
- **`ORDER BY`**：在合併前先對群組內的值進行排序。
- **`SEPARATOR`**：指定合併時的分隔符號（預設為 `,`）。

---

## 4. 視窗函數 (Window Functions)

視窗函數允許在**不壓縮結果列（不改變原始行數）**的情況下，對資料分組後的每一列執行計算。這與傳統 `GROUP BY` 會將多行合併為單一行的特性截然不同。

### 4.1 組內排名類 (Ranking Functions)
用於產生排序序號。

| 函數                 | 語法範例與說明                                                                                                                                | 常見應用場景             |
| :----------------- | :------------------------------------------------------------------------------------------------------------------------------------ | :----------------- |
| **`ROW_NUMBER()`** | `ROW_NUMBER() OVER (PARTITION BY dept ORDER BY salary DESC)`<br>為每列產生唯一且遞增的序號。遇到相同值仍會給予不同序號，**絕不重複*                                    | 抓取每個部門的最新或最高薪員工。   |
| **`RANK()`**       | `RANK() OVER (PARTITION BY dept ORDER BY salary DESC)`<br>產生排名。遇到相同值會產生相同排名，且之後的排名會**跳號**（如 1, 2, 2,                                   | 計算部門內的排名，顯示同分者並跳號。 |
| **`DENSE_RANK()` `DENSE_RANK() OVER ([PARTITION BY] col ORDER BY col DESC)`<br>產生排名。遇到相同值會產生相同排名，但之後的排名**不跳號**（如 1, 2, 2, 3）。<br>PARTITION BY: 讓各組獨立排名 BY:  | 需要緊密排名的情境。         |

### 4.2 前後時序類 (Value Functions)
用於抓取相鄰行或極端行的值，常搭配時間序列使用。

| 函數                  | 語法範例與說明                                                                                                                                    | 常見應用場景            |
| :------------------ | :----------------------------------------------------------------------------------------------------------------------------------------- | :---------------- |
| **`LAG()`**         | `LAG(price, 1) OVER (PARTITION BY stock_id ORDER BY date)`<br>取**前 N 行**的值（預設為 1）。若不存在則返回 `NULL`。                                          | 與昨日的股價進行比較計算跌宕。   |
| **`LEAD()`**        | `LEAD(price, 1) OVER (PARTITION BY stock_id ORDER BY date)`<br>取**後 N 行**的值（預設為 1）。若不存在則返回 `NULL`。                                         | 預測未來的趨勢或抓取下一筆資料。  |
| **`FIRST_VALUE()`** | `FIRST_VALUE(price) OVER (PARTITION BY stock_id ORDER BY date)`<br>取分組內**第一筆**的值。                                                          | 取得某檔股票上市首日的開盤價。   |
| **`LAST_VALUE()`**  | `LAST_VALUE(price) OVER (PARTITION BY stock_id ORDER BY date ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING)`<br>取分組內**最後一筆**的值。 | 取得某檔股票分組最後一天的收盤價。 |

> [!WARNING]
> **LAST_VALUE 範圍陷阱**
> 必須手動加上明確的範圍定義 `ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING`，否則系統預設只會抓到「截止至當前行」的值。

### 4.3 累計聚合類 (Aggregate Windows)
讓普通的聚合函數（如 `SUM`、`AVG`）轉職為「滾雪球模式」或「不壓縮行數的群組統計」。

| 函數                                | 語法範例與說明                                                                                                            | 大白話與經典應用場景                 |
| :-------------------------------- | :----------------------------------------------------------------------------------------------------------------- | :------------------------- |
| **累計加總**<br>`SUM() + ORDER BY`    | `SUM(weight) OVER (ORDER BY turn)`<br>有加上 `ORDER BY`，開啟**滾雪球累計模式**。隨著順序一筆一筆往下加。                                    | 計算每日累積營業額、帳戶餘額變動、公車載重計算。   |
| **靜態群組加總**<br>`SUM() + PARTITION` | `SUM(sales) OVER (PARTITION BY dept)`<br>不寫 `ORDER BY`，不會累加。直接算出該部門的總業績，然後原封不動貼在該部門每位員工的旁邊。                        | 計算個人業績佔部門總業績的貢獻度比例。        |
| **滑動視窗/移動平均**<br>`AVG() + ROWS`   | `AVG(price) OVER (ORDER BY date ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING)`<br>站在當前位置，只計算特定範圍（如前 1 行 + 自己 + 後 1 行）。 | 消除極端值波動的「移動平均線」（如 3 日 MA）。 |
|                                   |                                                                                                                    |                            |
