# 資料處理語言 (Data Manipulation Language, DML)

> [!ABSTRACT]
> 學習如何使用 DML 進行資料維護，包含 INSERT 插入、UPDATE 更新、DELETE 刪除與 TRUNCATE 的比較。

---

DML 用於對資料庫中的資料表進行資料的**新增 (INSERT)**、**修改 (UPDATE)** 與 **刪除 (DELETE)**。這類指令屬於交易控制的範疇，在提交 (COMMIT) 前皆可以進行回復 (ROLLBACK)。

---

## 1. 新增資料 (INSERT)

### 1.1 INSERT INTO 基礎語法
- **欄位與值對應**：欄位與值的數量必須相同，位置順序必須一一對應，且資料型態要相容。
- **欄位指定方式**：可以使用「指定欄位」或「不指定欄位」兩種方式來新增資料：
  - ==**不指定欄位時**：資料表中所有的欄位均必須依序給值。==
  - **指定欄位時**：僅需對指定的欄位給值，未指定的欄位會使用預設值或 NULL。
- **確認結構**：使用 `DESC dept;` 指令可以事先查詢欄位順序與資料型態。

```sql
INSERT INTO table [(column,...)] 
VALUES(value,...);
```

### 1.2 新增空值 (NULL)
1. **隱式填入**：在 `INSERT` 時未列舉的欄位，系統會自動填入該欄位的**預設值 (DEFAULT)**。若該欄位未設定預設值，則填入**空值 (NULL)**。
2. **顯式填入**：直接使用 `NULL` 關鍵字（請注意**不要**用引號 `""` 包覆，否則會被當作字串 `"NULL"`）。

### 1.3 新增日期與時間
1. **使用函數**：可以使用 `NOW()` 或 `CURRENT_DATE()` 等函數動態填入系統時間。
2. **使用字串輸入**：以字串格式手動輸入，格式與規範可參照 [[4. 資料型態]]。

### 1.4 新增多筆數據 (Batch Insert)
```sql
INSERT INTO table_name (column_list) 
VALUES (value_list_1), 
	(value_list_2), ... 
	(value_list_n);
```

### 1.5 INSERT 敘述常發生的錯誤
> [!WARNING]
> 執行新增時，應特別注意以下限制，以防寫入失敗：
> - **Mandatory value missing for a NOT NULL column**：設定為 `NOT NULL` 的欄位沒有給值。
> - **Data type mismatch**：寫入的值與欄位設定的資料型態不相容。
> - **Value too wide to fit in column**：輸入的值長度或範圍超過該欄位設定的最大限制。
> - **Duplicate value violates uniqueness constraint**：對設定「唯一限制條件 (Unique/PK)」的欄位寫入了重複的值。
> - **Foreign key constraint violated**：新增子表格 (Child) 資料時，所填入的外來鍵值在父表格 (Parent) 中找不到對應的資料。

### 1.6 從其他資料表複製資料 (Subquery Insert)
利用子查詢 (Subquery) 的結果，將資料大量新增至指定的資料表中。
- **使用時機**：適用於資料封存或移轉。例如將已結案的訂單、已銷戶的銀行帳戶或已處理完的公文移至歷史檔儲存，以提升線上資料庫的查詢效能。

```sql
INSERT INTO table[(column,...)] 
SELECT column,... 
FROM table WHERE ...
```
- ==**規則說明**：==
  - 若目標表格的**欄位名稱、順序與數量**與來源表格完全一致，可以省略 `column_list`。
  - ==若欄位名稱或順序不盡相同，則**必須**明確寫出 `column_list` 以便正確對應。==

## 2. 修改資料 (UPDATE)

### 2.1 修改基礎語法
```sql
UPDATE table 
SET column=value [,column=value]... 
WHERE conditions;
```
- **型態相容**：所設定的新值必須與欄位的資料型態相容。
- **多欄位修改**：可使用逗號 `,` 隔開，一次修改多個欄位。
- ==**安全警示**：若遺漏了 `WHERE` 子句，將會修改資料表中**所有列 (Rows)** 的資料！==

### 2.2 安全更新模式 (SQL_SAFE_UPDATES)
> [!IMPORTANT]
> **MySQL 安全防護機制**
> - 為避免使用者不小心誤改或誤刪全表資料，MySQL 使用系統參數 `SQL_SAFE_UPDATES` 來保護資料（預設每次連線其值為 `1`，即啟用保護）。
> - 在啟用狀態下，若 `UPDATE` 或 `DELETE` 沒有加上 `WHERE` 子句且 `WHERE` 沒有使用到 `Key` 欄位，系統將會拒絕執行。
> - 若需要進行批次更改或刪除，可先在 Session 中關閉此保護：
>   ```sql
>   SET SQL_SAFE_UPDATES = 0;
>   ```

### 2.3 使用子查詢更新資料
- **限制條件**：MySQL 在使用子查詢來更新資料時，**子查詢所查詢的表格與要更新的目標表格不能是同一個**。
- **運作模式**：此時子查詢主要是扮演「取值」或「條件篩選」的角色。

---

## 3. 刪除資料 (DELETE)

### 3.1 刪除基礎語法
```sql
DELETE FROM table 
[WHERE conditions];
```
- ==**安全警示**：若遺漏了 `WHERE` 子句，將會刪除資料表中的**所有資料**！==
- **進階用法**：可使用子查詢的結果作為 `WHERE` 的刪除篩選條件。

---

## 4. DML 常見的錯誤與約束衝突

在執行資料處理語言 (DML) 時，最常見的錯誤主要圍繞在**資料完整性限制**的衝突：

1. **違反資料檢查條件 (Constraint)**：
   - **鍵值重複**：違反了主鍵 (Primary Key) 或唯一鍵 (Unique Key) 的唯一性限制。
   - **違反外來鍵限制 (Foreign Key)**：試圖插入或更新不存在的關聯鍵，或是刪除被子表參照的父表資料。
   - **非空值欄位填入空值**：對設定為 `NOT NULL` 的欄位給予了 NULL 值。
2. **資料超限或失真**：輸入的值過長，或小數點位數/大小範圍超出設定。
3. **資料型態錯誤**：例如對數值型態欄位寫入無法隱式轉換的英文字串。
