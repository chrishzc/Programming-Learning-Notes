# MySQL 基礎介紹

> [!ABSTRACT]
> 介紹 MySQL 資料庫的特點、儲存架構與基礎安裝，並掌握如何啟動與連接資料庫服務。

---

## 1. 使用工具
- **MySQL Workbench**：系統管理工具（圖形化介面 GUI）。
- **MySQL 8.0 Command Line Client**：使用者工具（命令行介面 CLI）。

---

## 2. 常用 MySQL 命令 (CLI)
在 MySQL 終端機中，常用的基礎指令如下：

```sql
-- 列出目前伺服器現存的所有資料庫
SHOW DATABASES;

-- 設定預設存取的資料庫
USE sample;

-- 建立一個新資料庫
CREATE DATABASE name;

-- 執行指定路徑的 SQL 腳本文件 (.sql)
SOURCE "path/to/script.sql";

-- 列出當前資料庫中所有資料表
SHOW TABLES;

-- 查看資料表的欄位資訊（結構）
DESCRIBE dept; -- 縮寫可用 DESC dept;

-- 查詢資料表中的所有資料
SELECT * FROM dept;
```

## 3. 常用資料型態
MySQL 支援多種資料型態，主要分為文字、數值與日期：

### 3.1 文字類型 (String Types)
- **CHAR(n)**：**固定長度**。長度不足會以空白填入，顯示時自動截斷空白。範圍 1-255。
- **VARCHAR(n)**：**可變長度**。依實際存入內容佔用空間，範圍 1-65535。
- **表示法**：標準使用單引號 `'字串'`，==MySQL 中使用雙引號 `"字串"` 亦可==。

### 3.2 數值類型 (Numeric Types)
- **DECIMAL(p, s)**：**固定精確度**的十進位數。`p` 為總位數，`s` 為小數點後的位數。
- **INT**：整數類型。
- **表示法**：直接寫入數值，如 `1000`, `10.22`。

### 3.3 日期類型 (Date and Time)
- **DATE**：日期格式 `YYYY-MM-DD`。
- **DATETIME**：日期與時間 `YYYY-MM-DD HH:MM:SS`。
- **表示法**：可接受多種分隔符號，如 `'1998-02-04'`, `'1998/02/04'`, `'19980204'`。

## 4. 資料檢查條件 (Constraints)
用來確保資料的正確性與完整性：
1. **主鍵 (Primary Key)**：一個資料表只能有一個，唯一識別每一筆記錄。
2. **唯一鍵 (Unique Key)**：值不能重複，但允許為空值 (NULL)。
3. **外來鍵 (Foreign Key)**：其值必須參照到另一個表格的主鍵。
4. **不允許空值 (NOT NULL)**：欄位內容不能為空。
5. **預設值 (Default)**：設定新增資料時，若未指定欄位值則自動填入的值。
