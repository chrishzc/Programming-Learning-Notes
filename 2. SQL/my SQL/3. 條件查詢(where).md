# WHERE 條件篩選與進階查詢

> [!ABSTRACT]
> 掌握 WHERE 子句的條件篩選技巧，包含比較運算子、邏輯運算子 (AND/OR/NOT)、模糊查詢 (LIKE) 與區間篩選。

---

## 1. 基礎語法
`WHERE` 子句位於 `FROM` 之後，用於過濾符合特定條件的資料列。
```sql
SELECT <column list> 
FROM <table> 
WHERE <conditions>;
```
- **條件組成**：包含比較運算式、邏輯運算式或 SQL 特定運算式。
- **邏輯值**：真 (TRUE)、假 (FALSE)、空值 (NULL)。

---

## 2. 運算子分類

### 2.1 比較運算子 (Comparison Operators)
用於比較兩個運算元的大小。
> [!IMPORTANT]
> ==相同資料類型才能進行比較==。字串比較通常基於內碼（預設為 Unicode）。

| 運算子 | 說明 |
| :---: | :--- |
| `=` | 等於 (Equal to) |
| `>` | 大於 (Greater than) |
| `>=` | 大於或等於 (Greater than or equal to) |
| `<` | 小於 (Less than) |
| `<=` | 小於或等於 (Less than or equal to) |
| `<>` / `!=` | 不等於 (Not equal to) |

### 2.2 邏輯運算子 (Logical Operators)
- **`AND`**：所有條件皆為真時才符合。
- **`OR`**：任一條件為真即符合。
- **`NOT`**：反向運算（非）。

### 2.3 SQL 特定運算子 (SQL Operators)
1. **`BETWEEN ... AND ...`**：查詢連續區間（包含邊界）。
   - 語法：`expr BETWEEN x1 AND x2` (x1 必須小於 x2)。
   - 適用於數值、日期與字元。
2. **`IN (...)`**：查詢列舉值。
   - 語法：`expr IN (value1, value2, ...)`。
   - 只要符合串列中任一個值即成立。
3. **`LIKE`**：模糊比對。
   - `%`：代表零或多個任意字元。
   - `_`：代表一個任意字元。
   - **自訂跳脫字元**：使用 `ESCAPE` 定義。例如 `LIKE '%&_%' ESCAPE '&'` 用來搜尋包含底線的字串。
   ![[{C892982F-34D8-45A1-8659-DF08CF686B9E}.png|463]]
   ![[{09EFD2DE-842B-4D91-81CA-01688DABA197}.png|463]]
4. **`IS NULL`**：空值專用查詢。
   - ==注意：不能使用 `= NULL`，必須使用 `IS NULL`。==
5. **`REGEXP` / `RLIKE`**：正規表達式 (Regular Expression) 比對。
   - 提供比 `LIKE` 更強大且複雜的字串模式匹配。
   - **常用符號**：
     - `^`：字串的開頭 (例如 `^A`：以 A 開頭)
     - `$`：字串的結尾 (例如 `Z$`：以 Z 結尾)
     - `.`：比對任意單一字元
     - `[...]`：比對中括號內的任意一個字元 (例如 `[abc]`：包含 a、b 或 c)
	     - [abc]：包含 a、b 或 c。
	     - [A-Za-z]：包含任意一個大小寫英文字母。
     - `|`：邏輯 OR (例如 `apple|banana`：包含 apple 或 banana)
     - `*`：前面的字元出現 0 次或多次
     - `+`：前面的字元出現 1 次或多次
- **跳脫字元 \\\
    - 點（`.`）在 Regex 中代表「任意字」。如果要在信箱或網址中比對「真正的句點」，必須在前面加上雙反斜線。
    - _範例_：`\\.com` 才是比對 `.com`；若只寫 `.com`，則 `Acom`、`#com` 都會被放行。

## 3. 資料排序 (ORDER BY)
`ORDER BY` 必須放置於 `SELECT` 敘述的最後。
```sql
ORDER BY 欄位1 [ASC|DESC], 欄位2 [ASC|DESC];
```
- **ASC**：升冪（由小到大，預設值）。
- **DESC**：降冪（由大到小）。
- **多欄位排序**：先依第一個欄位排序，若重複才參考第二個欄位。
- **排序對象**：欄位名、別名、運算式或欄位位置（從 1 開始）。

## 4. 流程控制：CASE 運算式
在 SQL 中執行條件分類動作。

### 4.1 列舉式 CASE (Simple CASE)
**特點**：僅能進行「**等於 (=)**」的判斷。適合處理固定的代碼對應（如性別、狀態碼）。
```sql
SELECT 
  CASE 欄位
    WHEN 'M' THEN '男'
    WHEN 'F' THEN '女'
    ELSE '未知'
  END AS 性別
FROM <table>;
```

### 4.2 條件式 CASE (Searched CASE)
**特點**：**最靈活**。可以進行任何布林邏輯判斷，包括範圍 (`>、<`)、模糊比對 (`LIKE`) 或空值檢查 (`IS NULL`)。
```sql
SELECT 
  CASE 
    WHEN score >= 90 THEN '優等'
    WHEN score >= 60 THEN '及格'
    WHEN score IS NULL THEN '缺考'
    ELSE '不及格'
  END AS 成績評等
FROM <table>;
```

> [!TIP]
> **兩者如何選擇？**
> - **列舉式 (Simple)**：適合「單一欄位」且僅需「等值比對」時，寫法較簡潔。
> - **條件式 (Searched)**：適合「範圍判斷」、「判斷空值」或「同時參考多個欄位」的複雜邏輯。
