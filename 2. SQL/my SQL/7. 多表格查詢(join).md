# 多表格查詢 (JOIN)

> [!ABSTRACT]
> 深入學習多資料表的關聯查詢，包含 INNER JOIN、LEFT JOIN、RIGHT JOIN、FULL JOIN 與交叉連接 (CROSS JOIN) 的原理與實作。

---

## 1. 基礎概念
### 1.1 虛擬資料表 (Virtual Table)
`FROM` 子句設定了 `SELECT` 敘述的資料來源。當我們執行 `JOIN` 運算時，會產生一個暫時性的**虛擬資料表**，後續的 `WHERE`、`GROUP BY` 等子句都是基於這個虛擬表進行操作。

### 1.2 表格別名 (Table Aliases)
為了簡化程式碼並提高查詢效率，建議為表格設定別名。
- **注意**：表格別名僅在該次 SQL 敘述中有效。
- **範例**：
  ```sql
  SELECT e.ename, e.sal, d.dname 
  FROM dept d JOIN emp e ON d.deptno = e.deptno;
  -- d 為 dept 的別名，e 為 emp 的別名
  ```

---

## 2. JOIN 的種類

### 2.1 交叉連結 (Cross Join)
又稱為 **笛卡兒乘積 (Cartesian Product)**。
- **特性**：將兩個表格的所有列進行排列組合（左表筆數 × 右表筆數）。
- **評價**：通常不具備實際查詢價值，除非有特殊統計需求。
- **語法**：`SELECT * FROM t1 CROSS JOIN t2;`

### 2.2 內部連結 (Inner Join)
最常用的連結方式，僅回傳符合連結條件的資料。

#### **相等連結 (Equal Joins)**
使用 `=` 運算子比對欄位值。
```sql
SELECT ... FROM t1 JOIN t2 ON t1.col = t2.col;
```

> [!WARNING]
> **自然連結 (Natural Join)**：MySQL 會自動尋找同名欄位進行連結。雖然方便，但因具備不可預測性且其他資料庫系統支援度不一，**建議少用**。

#### **不相等連結 (Non-Equal Joins)**
使用非 `=` 的比較運算子（如 `>=`, `<=`, `BETWEEN AND`）。
```sql
-- 範例：查詢員工薪資所屬的等級
SELECT e.ename, e.sal, s.grade 
FROM emp e JOIN salgrade s ON (e.sal BETWEEN s.losal AND s.hisal);
```

#### **多表格連結**
可以連續使用多個 `JOIN` 連結多張表。
```sql
SELECT ... 
FROM t1 
JOIN t2 ON t1.key = t2.key
JOIN t3 ON t2.key = t3.key;
```

### 2.3 外部連結 (Outer Join)
除了回傳匹配的資料外，也會顯示不匹配的資料列（不匹配的部分補為 `NULL`）。

- **LEFT JOIN**：以左表為主，保留左表所有資料。
- **RIGHT JOIN**：以右表為主，保留右表所有資料。
- **FULL JOIN**：保留兩邊所有資料（MySQL 不直接支援，通常透過 `UNION` 達成）。

```sql
SELECT e.ename, d.dname
FROM emp e LEFT JOIN dept d ON e.deptno = d.deptno;
```

### 2.4 自我連結 (Self Join)
同一張資料表自己跟自己連結。
- **關鍵**：必須使用**不同的別名**來模擬成兩張表。
- **應用**：查詢員工及其經理（經理也是員工，都在同一張表）。
```sql
SELECT e.ename AS 員工, m.ename AS 經理
FROM emp e JOIN emp m ON e.mgr_id = m.emp_no;
```

---

## 3. 集合運算：UNION
- **功能**：將兩個 `SELECT` 的結果合併成一個。
- **Excluding Join**：MySQL 雖然沒有直接的 `EXCLUDING JOIN` 語法，但可以透過 `LEFT JOIN` 配合 `WHERE col IS NULL` 或使用 `UNION` 邏輯來實現。
