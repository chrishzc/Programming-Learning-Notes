# 修改資料表結構 (ALTER TABLE)

> [!ABSTRACT]
> 學習使用 ALTER TABLE 動態修改已存在的資料表結構，包括新增、修改、刪除欄位與約束條件。

---

在建立資料表之後，若因為系統需求變更，需要調整欄位屬性、增刪欄位或修改約束限制條件，可以使用 `ALTER TABLE` 指令。這類修改屬於 DDL (Data Definition Language) 範疇，一經執行便會隱式提交，無法透過交易進行回滾 (Rollback)。

---

## 1. 欄位的管理 (ADD, MODIFY, CHANGE, DROP)

我們可以使用 `ALTER TABLE` 來進行欄位的新增、修改與刪除：

### 1.1 新增欄位 (ADD COLUMN)
可以向資料表新增一個或多個新欄位。
- **位置控制**（MySQL 專屬）：
  - **`FIRST`**：將新欄位排在資料表的最前方。
  - **`AFTER existing_column`**：將新欄位排在指定的欄位後面。
  - 若未指定，系統預設會加在資料表的**最後方**。

```sql
-- 新增單一欄位，排在最後面
ALTER TABLE table_name 
ADD COLUMN column_name data_type [constraints];

-- 新增單一欄位並指定排列位置
ALTER TABLE table_name 
ADD COLUMN phone VARCHAR(20) AFTER name;
```

### 1.2 修改欄位定義 (MODIFY COLUMN)
僅修改欄位的資料型態、長度、限制條件或排列位置，**不修改欄位名稱**。

```sql
-- 修改資料型態與限制條件
ALTER TABLE table_name 
MODIFY COLUMN email VARCHAR(100) NOT NULL;
```
> [!WARNING]
> **資料遺失風險：**
> - 將長度改小（如 `VARCHAR(50)` 改為 `VARCHAR(20)`）或改變資料型態時，若表中已有大於新限制的資料，可能會導致資料被截斷或語法報錯。

### 1.3 修改欄位名稱與屬性 (CHANGE COLUMN)
可用於**同時更改欄位名稱與屬性**（即便不更改屬性，也必須重新宣告型態）。

```sql
-- 修改欄位名稱由 old_col 改為 new_col
ALTER TABLE table_name 
CHANGE COLUMN old_column_name new_column_name new_data_type [constraints];
```

### 1.4 刪除欄位 (DROP COLUMN)
將指定的欄位從資料表中永久刪除。
- ==**警示**：刪除欄位會將該欄位的所有資料一併清除，且無法 Rollback！==

```sql
ALTER TABLE table_name 
DROP COLUMN column_name;
```

## 2. 限制條件與索引的管理

當資料表建立後，我們可以隨時為其新增或移除限制條件：

### 2.1 新增限制條件 (ADD)
- **新增主鍵 (PRIMARY KEY)**：
  ```sql
  ALTER TABLE table_name 
  ADD PRIMARY KEY (column_name);
  ```
- **新增唯一限制 (UNIQUE)**：
  ```sql
  ALTER TABLE table_name 
  ADD CONSTRAINT uq_name UNIQUE (column_name);
  ```
- **新增外來鍵 (FOREIGN KEY)**：
  ```sql
  ALTER TABLE table_name 
  ADD CONSTRAINT fk_name FOREIGN KEY (fk_column) REFERENCES parent_table (pk_column);
  ```

### 2.2 刪除限制條件 (DROP)
- **刪除主鍵**：
  ```sql
  ALTER TABLE table_name 
  DROP PRIMARY KEY;
  ```
- **刪除唯一鍵 / 索引 (UNIQUE / INDEX)**：
  - 在 MySQL 中，唯一鍵是以索引形式存在，因此必須使用 `DROP INDEX` 來移除。
  ```sql
  ALTER TABLE table_name 
  DROP INDEX uq_name;
  ```
- **刪除外來鍵 (FOREIGN KEY)**：
  - 必須指定外來鍵的 Constraints 名稱。
  ```sql
  ALTER TABLE table_name 
  DROP FOREIGN KEY fk_name;
  ```

---

## 3. 資料表名稱管理 (RENAME)

若需要變更整個資料表的名稱，可以使用以下指令：

```sql
-- 方式一：使用 ALTER TABLE
ALTER TABLE old_table_name 
RENAME TO new_table_name;

-- 方式二：使用 RENAME TABLE (可一次重新命名多張表)
RENAME TABLE old_table_name TO new_table_name;
```
