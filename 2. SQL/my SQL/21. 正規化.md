# 正規化處理

> [!ABSTRACT]
> 學習資料庫正規化理論，深入探討第一正規化 (1NF)、第二正規化 (2NF)、第三正規化 (3NF) 與 BCNF 的轉換步驟與目的。

---

透過一連串的檢視，讓每一個資料項目歸屬到適當的資料體中。
- 讓資料在新增、修改或刪除時，整個系統相關資料仍能維持正確良好的狀態。
- 具有關聯式之特性，但仍適用於其他資料模型。
- 正規化步驟: 第一正規化 、第二正規化、第三正規化
## 正規化之目的與代價
- 正規化之目的
	  -  正規化之作法一般是將表格做適當分割，因為沒有正規化會造成有資料 重覆儲存的浪費情形，也會造成「更新時的異常現象」。
- 正規化之代價 
	  - 正規化讓資料分別放在多個表格中，使用資料時==必須進行連結(Join)因而影響效能==，且正規化層級愈高對效能的負面影響也會越大。
	  - 因此有時候會適度地「反正規化」以爭取效能。
## 正規化階層
資料庫在正規化時會有一些規則，並且每條規則都 稱為「正規化形式」。如果符合第一條規則，則資 料庫就稱為「第一正規化形式(1NF) 」。如果符合 前二條規則，則資料庫就被視為屬於「第二正規化 形式(2NF)」。==大部分資料庫做到第三正規化就足夠==
1. 第一正規化的規則：去除多值屬性
2. 第二正規化的規則：去除只與部分主鍵相關欄位(部份相依)
3. 第三正規化的規則：去除鍵值外有相依關係的欄位(遞移相依)
