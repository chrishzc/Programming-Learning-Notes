# 儲存引擎與語系設定 (Storage Engine & Character Set)

> [!ABSTRACT]
> 深入分析 MySQL 的儲存引擎（特別是 InnoDB 與 MyISAM），並學習如何設定與修改資料庫語系與校對字元集。

---

儲存引擎 (Storage Engine) 是 MySQL 用來儲存、處理與保護資料的底層技術。不同的儲存引擎有不同的資料儲存結構、索引機制與功能特色。此外，字元編碼 (Character Set) 與定序規則 (Collation) 則決定了資料庫如何儲存文字與進行字串比對。

---

## 1. 核心儲存引擎對比

MySQL 提供了多種不同的儲存引擎，最核心且常用的有以下三種：

### 1.1 MyISAM
- **特性**：MySQL 早期的預設儲存引擎。結構簡單，運作效率高，但不支援事務交易 (Transaction) 與外來鍵 (Foreign Key)。
- **實體檔案結構**：每一個 MyISAM 表格在作業系統硬碟中會建立三個同名檔案（以 `employee` 資料表為例）：
  - **`employee.frm`**：儲存表格的結構定義與設定。
  - **`employee.MYD`** (MySQL Data)：儲存表格的實際記錄資料。
  - **`employee.MYI`** (MySQL Index)：儲存表格的索引資料。
- **優缺點分析**：
  - **優點**：具備極佳的**可攜性 (Portable)**，可直接複製這三個檔案到另一台伺服器使用。且因為不限制單一資料庫的表格數量，在處理大型檔案時可避免超出大小限制。
  - **缺點**：若表格數量極多，會因為產生過多實體檔案而超出作業系統的檔案控制數量限制。

### 1.2 InnoDB
- **特性**：MySQL 目前的**預設儲存引擎**。功能完善，提供了與商用大型資料庫同等級的特性，如**交易處理 (Transaction)**、**列鎖定 (Row-level Locking)** 與**外來鍵限制 (Foreign Keys)**。
- **儲存模式**：分為**獨立表空間**與**共用表空間**兩種：
  - **獨立表空間 (預設方式)**：
    - 系統會為每個表格建立一個 **`ibd`** 檔（包含資料與索引），並儲存於以資料庫命名的資料夾中。
  - **共用表空間**：
    - 所有表格的資料和索引都儲存在統一的系統檔案（如 `ibdata1`、`ib_logfile0`、`ib_logfile1`）中。
    - 雖然可以避免超出作業系統檔案數量限制，但單一表空間限制最高只能包含 200 萬個表格。
- **查詢指令**：可透過以下指令查看目前是否啟用獨立表空間：
  ```sql
  SHOW VARIABLES LIKE 'innodb_file_per_table';
  ```

### 1.3 MEMORY
- **特性**：將所有的資料都儲存在**隨機存取記憶體 (RAM)** 中，運作與讀寫速度極快。
- **限制與機制**：
  - **易失性**：一旦 MySQL 伺服器重啟或關閉，記憶體中的**資料會全部消失**，但資料表結構 (`.frm` 檔) 仍會保留在硬碟中。
  - **耗用資源**：不適合用來儲存大資料量的表格，以防耗盡伺服器實體記憶體。

## 2. 字元集 (Character Set)

字元集定義了字元的種類以及其對應的編碼規則。為了處理不同國家的語言，MySQL 支援在同一個伺服器、資料庫、資料表，甚至單一欄位上設定不同的字元集。

- **預設編碼**：MySQL 8.0 起預設字元集為 **`utf8mb4`**（相容完整 Unicode 規範，包括 Emoji 表情符號）；早期版本預設為 `utf8` (即 `utf8mb3`)。
- **字元集查詢**：
  ```sql
  -- 顯示 MySQL 支援的所有字元集
  SHOW CHARACTER SET; -- 或縮寫 SHOW CHARSET;
  ```
- **影響層面**：字元集決定了資料在硬碟中所佔用的儲存空間大小（例如 `utf8mb4` 每個字元最多佔用 4 Bytes）。

---

## 3. 字元序 / 定序規則 (Collation)

字元序 (Collation) 定義了在特定字元集下，**字元之間進行比較與排序的規則**。

- **對應關係**：字元集與字元序是一對多的關係（每個字元集至少有一種對應的字元序，且有一個預設的字元序；但不同的字元集不能共用同一個字元序）。
- **定序查詢**：
  ```sql
  -- 顯示 MySQL 支援的所有字元序
  SHOW COLLATION;
  ```
- **預設規則**：目前 MySQL 預設的字元序為 **`utf8mb4_0900_ai_ci`**。
- **後綴命名規則**：
  - **`_ci`** (Case Insensitive)：**不區分大小寫**（例如 `'A'` 與 `'a'` 視為相同）。
  - **`_cs`** (Case Sensitive)：**區分大小寫**。
  - **`_bin`** (Binary)：**二進位比較**（基於編碼內碼值進行大小排序與比對，區分大小寫）。
