# 資料完整性與檢查條件 (Constraints)

> [!ABSTRACT]
> 掌握資料庫檢查條件（主鍵、外鍵、唯一、非空與預設值），以確保資料的完整性與正確性。

---

限制條件 (Constraints) 用於確保資料表的資料正確性、有效性與完整性（Data Validation）。限制條件可以在**建立資料表時 (CREATE TABLE)** 指定，也可以在**資料表建立之後 (ALTER TABLE)** 加入。

根據定義的位置，限制條件可分為兩大階層：
- **欄位階層 (Column Level)**：直接宣告在個別欄位的定義後面。適用於單一欄位的限制條件，不可命名。
- **資料表階層 (Table Level)**：宣告在所有欄位定義完畢之後。適用於單一欄位或跨欄位的**複合限制條件**，並可使用 `CONSTRAINT` 關鍵字自訂該限制條件的名稱。

---

## 1. 欄位預設值 (DEFAULT)

在新增資料列時，若未對該欄位指定任何數值，系統會自動填入所設定的預設值。

- **預設機制**：MySQL 中所有欄位若不主動設定 `DEFAULT`，預設其值皆為空值 (`NULL`)。
- **限制**：預設值表示式 `expr` 通常需要是常值 (Literal)，但在 MySQL 8.0+ 中亦可使用部分函數（如時間型態的 `NOW()`）。

```sql
column_name data_type [NOT NULL | NULL] DEFAULT default_value
```

---

## 2. 不允許空值 (NOT NULL)

強制該欄位在新增或修改資料時，一定要輸入有效的值，不能留空。

- **特性**：僅能定義於**欄位階層 (Column Level)**。

```sql
column_name data_type NOT NULL
```

---

## 3. 主鍵限制條件 (PRIMARY KEY)

主鍵是用來唯一識別資料表中每一筆記錄的欄位。主鍵欄位的值**必須唯一**，且**絕不能為空值 (`NOT NULL`)**。一個資料表只能有一個主鍵。

### 3.1 單一主鍵 (Single Primary Key)
適用於由單一欄位構成的主鍵，可直接宣告在欄位後方。

```sql
column_name data_type PRIMARY KEY
```

### 3.2 複合主鍵 (Composite Primary Key)
適用於由多個欄位共同構成的主鍵，**必須**使用資料表階層來定義。

```sql
CREATE TABLE table_name 
(
  column1 data_type NOT NULL,
  column2 data_type NOT NULL,
  ...,
  [CONSTRAINT pk_name] PRIMARY KEY (column1, column2)
);
```
> [!NOTE]
> - `CONSTRAINT pk_name` 可用於為該主鍵命名。若省略，系統會使用預設名稱（通常為 `PRIMARY`）。

---

## 4. 唯一限制條件 (UNIQUE)

唯一鍵用來確保該欄位（或複合欄位）的所有值都是唯一的，不允許重複，但與主鍵不同的是，**唯一鍵允許輸入空值 (`NULL`)**。

### 4.1 單一唯一鍵
```sql
column_name data_type UNIQUE
```

### 4.2 複合唯一鍵
由多個欄位共同組成唯一限制：

```sql
CREATE TABLE table_name 
(
  column1 data_type,
  column2 data_type,
  ...,
  [CONSTRAINT uq_name] UNIQUE (column1, column2)
);
```

---

## 5. 自動編號選項 (AUTO_INCREMENT)

系統在新增資料列時，會自動為該欄位編製遞增的流水號。

- **基本規範**：
  - 欄位的資料型態必須為**數值型態** (如 `INT`、`BIGINT`)。
  - 該欄位**必須與主鍵 (Primary Key) 或唯一鍵 (Unique Key) 配合使用**。
  - 每個資料表**只能有一個**欄位被設定為 `AUTO_INCREMENT`。
  - 系統預設起始編號為 `1`。

### 5.1 語法與設定起始值
```sql
CREATE TABLE table_name 
(
  id INT AUTO_INCREMENT PRIMARY KEY,
  ...
) AUTO_INCREMENT = 100; -- 設定起始流水號為 100
```

### 5.2 系統運作特性與規則
- **自動給值**：新增資料 (`INSERT`) 時，對於該欄位若給予 `NULL`、`0` 或不給值，系統都會自動編號。
- **手動指定**：若給定重複的值會引發錯誤；若給予大於目前最大值的值（例如 `20`），則下一次新增時，流水號會從此值加 1（即 `21`）開始計算。
- **交易回復**：若在交易區塊中新增資料後執行 `ROLLBACK`，該次已分配出去的流水號**不會回收**，會造成流水號跳號。
- **資料刪除**：使用 `DELETE` 刪除部分列後，已被使用的流水號**不會回收再利用**。
- **LAST_INSERT_ID()**：可用於查詢該連線最近一次成功新增時所產生的自動編號值，此值為 Session 級別，不受其他連線的併發寫入干擾。

---

## 6. 外來鍵限制條件 (FOREIGN KEY)

外來鍵用來確保多張資料表之間的參照完整性（Referential Integrity），防止產生「孤兒資料」（例如：訂單關聯到一個不存在的會員帳號）。

- **基本規範**：
  - 使用外來鍵的資料表與被參照的資料表，其儲存引擎必須為 **InnoDB**。
  - 外來鍵的欄位數量、資料型態與長度，必須與被參照的主表欄位**完全一致**。
  - 外來鍵**只能參考主表的主鍵 (Primary Key) 或唯一鍵 (Unique Key)**。

### 6.1 基礎語法
```sql
CREATE TABLE child_table 
(
  ...,
  fk_column data_type,
  [CONSTRAINT fk_name] FOREIGN KEY (fk_column) REFERENCES parent_table (pk_column)
) ENGINE = InnoDB;
```

### 6.2 關聯資料變更時的處理行為 (ON DELETE / ON UPDATE)
當主表 (Parent) 被刪除或修改時，子表 (Child) 對應的關聯資料會如何連動？我們可以透過設定來規範（預設為 `RESTRICT`）：

```sql
[ON DELETE { RESTRICT | CASCADE | SET NULL }]
[ON UPDATE { RESTRICT | CASCADE | SET NULL }]
```

- **ON UPDATE (當主表更新鍵值時)**：
  - **`RESTRICT`** (或 `NO ACTION`)：若子表存在對應關聯資料，主表**不允許修改**該鍵值。
  - **`CASCADE`** (級聯)：主表更新時，子表的關聯欄位會**同步更新**。
  - **`SET NULL`**：主表更新時，子表對應資料的關聯欄位會**自動設為 NULL**。
- **ON DELETE (當主表刪除資料時)**：
  - **`RESTRICT`** (或 `NO ACTION`)：若子表存在對應關聯資料，主表**不允許刪除**該筆記錄。
  - **`CASCADE`**：主表刪除時，子表對應關聯的整列資料會**同步被刪除**。
  - **`SET NULL`**：主表刪除時，子表對應資料的關聯欄位會**自動設為 NULL**（該欄位不可設定為 `NOT NULL`）。

---

## 7. 自訂資料檢測 (CHECK)

自 **MySQL 8.0.16** 版本起，正式支援 `CHECK` 限制條件，可用於在欄位寫入前執行自訂的邏輯表示式比對。

- **用途**：可以用來限制數值範圍、比較同一個資料表內兩個欄位的大小等。

```sql
CREATE TABLE employees 
(
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(50) NOT NULL,
  age INT,
  salary DECIMAL(10,2),
  bonus DECIMAL(10,2),
  
  -- 欄位階層 CHECK：檢查年齡必須大於等於 18 歲
  CONSTRAINT chk_age_limit CHECK (age >= 18),
  
  -- 資料表階層 CHECK：檢查獎金不能超過薪水
  CONSTRAINT chk_bonus_limit CHECK (bonus <= salary)
);
```
