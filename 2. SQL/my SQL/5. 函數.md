# MySQL 函數詳解

> [!ABSTRACT]
> 彙整 MySQL 中常用的字串處理、數值計算、日期時間與控制流函數，並提供實用查詢範例。

---

SQL 函數主要分為兩大類：**單一資料列函數**與**多重資料列函數 (彙總函數)**。

---

## 1. 單一資料列函數 (Single-Row Functions)
**特點**：每一筆記錄 (Row) 執行一次，傳回一個結果。常用於資料項的運算或轉換。

### 1.1 字串函數 (String Functions)
處理字串資料，傳回字串或數值。標註 `*` 為考試重點。

| 函數                                                    | 說明                             |
| :---------------------------------------------------- | :----------------------------- |
| `LENGTH(str)`                                         | 字串長度 (Bytes)                   |
| `CHAR_LENGTH(str)`                                    | 字元個數                           |
| `LOWER(str)` / `LCASE`                                | 轉小寫                            |
| `UPPER(str)` / `UCASE`                                | 轉大寫                            |
| `*CONCAT(str1, str2, ...)`                            | **字串連結**                       |
| `GROUP_CONCAT(DISTINCT columns ORDER BY columns ASC)` | **群組字串連結**(含去重與排序)             |
| `CONCAT_WS(sep, s1, s2)`                              | 以分隔字元連結字串                      |
| `INSERT(str, pos, len, new)`                          | 在指定位置插入字串                      |
| `LEFT(str, len)`                                      | 由左至右傳回指定字元數                    |
| `RIGHT(str, len)`                                     | 由右至左傳回指定字元數                    |
| `*LPAD` / `RPAD(str, len, pad)`                       | **向左/右補齊**至指定長度                |
| `*SUBSTRING(str, pos, [len])`                         | **擷取字串** (從 pos 開始，擷取 len 個字元) |
| `*REPEAT(str, count)`                                 | **重複字串**指定次數                   |
| `REPLACE(str, from, to)`                              | 字串替換                           |
| `TRIM(str)`                                           | 去除兩端空白                         |
| `STRCMP(s1, s2)`                                      | 比較字串 (0:相等, 1:左大, -1:右大)       |

### 1.2 數值函數 (Numeric Functions)
| 函數                     | 說明                       |
| :--------------------- | :----------------------- |
| `*ROUND(X, D)`         | **四捨五入**至小數第 D 位         |
| `TRUNCATE(X, D)`       | 無條件捨去至小數第 D 位            |
| `*MOD(N, M)`           | **求餘數** (N % M)          |
| `CEIL(X)` / `FLOOR(X)` | 天花板 (不小於 X 的最小整數) / 地板函數 |
| `POWER(X, Y)`          | X 的 Y 次方                 |
| `ABS(X)`               | 絕對值                      |
| `RAND()`               | 產生 0~1 間的亂數              |

### 1.3 日期時間函數 (Date & Time Functions)
**系統時間：**
- `*CURDATE()` / `CURRENT_DATE()`：現在日期
- `*NOW()` / `CURRENT_TIMESTAMP()`：現在日期與時間

**資料擷取：**
- `*YEAR(date)` / `MONTH()` / `DAY()`：傳回年/月/日
- `HOUR()` / `MINUTE()` / `SECOND()`：擷取時/分/秒
- `DAYNAME()` / `MONTHNAME()`：日期/月份的名稱
- `EXTRACT(unit FROM date)`：擷取指定的日期單位

**日期運算：**
- `*DATEDIFF(d1, d2)`：計算兩日期相差天數
- `*ADDDATE(date, INTERVAL expr unit)`：日期加法 (如 `INTERVAL 1 DAY`)
- `SUBDATE()`：日期減法
- `TIMESTAMPDIFF(unit, d1, d2)`：計算更精確的時間差
### 1.4 常用日期動作整理

#### 🗓️ 場景一：鎖定特定年月 / 固定日期區間
專門解決「撈出某年某月」或「特定活動期間」的報表需求。

| 語法範例 (以撈取 2026 年 2 月為例)                                | 優缺點快速評估                     | 大白話與適用場景                                   |
| :----------------------------------------------------- | :-------------------------- | :----------------------------------------- |
| `WHERE date_col BETWEEN '2026-02-01' AND '2026-02-28'` | ★ 效能最高！<br>(資料庫最愛的標準寫法)     | 走索引速度最快。缺點是必須知道當月最後一天是 28、29 還是 30 號。      |
| `WHERE date_col LIKE '2026-02%'`                       | ★ 最省腦袋！<br>(LeetCode 刷題流最愛) | 把日期當文字，後面加萬用字元 `%`。不用管當月有幾天，開頭對了就全拿。       |
| `WHERE LEFT(date_col, 7) = '2026-02'`                  | ★ 優雅好讀！<br>(字串裁剪法)          | 直接剪下最左邊 7 個字元（YYYY-MM），直接比對年跟月，同樣不用管月底是幾號。 |
| `WHERE YEAR(date_col) = 2026 AND MONTH(date_col) = 2`  | ★ 邏輯最清晰！<br>(但大型資料表效能較差)    | 用內建函數強行把年、月拆出來比對。缺點是如果資料量極大，這寫法會跑比較慢。      |

#### ⏱️ 場景二：時間相減 / 計算時間差
專門解決「這個人幾歲？」、「這筆訂單隔了幾天發貨？」等問題。

| 語法範例                                               | 回傳結果範例              | 大白話與適用場景                                              |
| :------------------------------------------------- | :------------------ | :---------------------------------------------------- |
| `DATEDIFF('2026-02-15', '2026-02-10')`             | `5`<br>(回傳單位一律是：天數) | 計算「天數差」。公式：後面的日期 減 前面的日期。                             |
| `TIMESTAMPDIFF(MONTH, '2026-01-01', '2026-05-01')` | `4`<br>(回傳單位：4 個月)  | 計算「自訂單位差」。第一個參數可換：`YEAR` (年)、`MONTH` (月)、`HOUR` (小時)。 |
| `TIMESTAMPDIFF(YEAR, birth_date, CURDATE())`       | `28`<br>(回傳單位：歲數)   | 【萬年實用技】直接算出這個人現在幾歲！`CURDATE()` 是資料庫抓取今天日期的函數。         |

#### ⏳ 場景三：時間加減 / 往前推、往後算
專門解決「會員還有幾天過期？」、「找出 30 天內消費的顧客」。

| 語法範例                                                       | 計算後的日期結果                    | 大白話與適用場景                                  |
| :--------------------------------------------------------- | :-------------------------- | :---------------------------------------- |
| `DATE_ADD('2026-02-01', INTERVAL 7 DAY)`                   | `2026-02-08`<br>(往後推算 7 天)  | 「往後加天數」。<br>經典場景：計算商品保固截止日。               |
| `DATE_SUB('2026-02-01', INTERVAL 1 MONTH)`                 | `2026-01-01`<br>(往前推算 1 個月) | 「往前減月份」。<br>經典場景：計算上個月的對帳起點。              |
| `WHERE order_date >= DATE_SUB(CURDATE(), INTERVAL 30 DAY)` | (動態計算過去 30 天的資料)            | 【最常複製這行】報表系統天天在用，自動撈出「從今天往前算 30 天內」的所有訂單。 |
### 1.5 資料型態轉換 (Conversion)
- `CAST(expr AS type)`
- `*CONVERT(expr, type)`
> [!NOTE]
> 常用目標型態：`BINARY`、`CHAR`、`DATE`、`DATETIME`、`SIGNED`、`TIME`、`UNSIGNED`。

### 1.6 通用與流程控制函數
- `USER()`、`VERSION()`、`DATABASE()`：系統資訊。
- `*IFNULL(expr1, expr2)`：若 expr1 為 NULL，則傳回 expr2。
- `*IF(expr1, expr2, expr3)`：三元運算子 (若 1 為真傳回 2，否則傳回 3)。

## 2. 多重資料列函數 (Multiple-Row / Aggregate Functions)
**特點**：多筆記錄執行一次，傳回一個彙總結果。**會自動忽略 NULL 值**。

| 函數                            | 說明                              |
| :---------------------------- | :------------------------------ |
| `COUNT(*)`                    | 回傳總資料筆數 (含 NULL)                |
| `COUNT(column)`               | 回傳欄位**不為空值**的筆數                 |
| `COUNT(DISTINCT col)`         | 去除重複後的非空值筆數                     |
| `MAX(column)` / `MIN(column)` | 最大值 / 最小值                       |
| `SUM(column)`                 | 總和                              |
| `AVG(column)`                 | 平均值 (**不計入空值**)                 |
| `AVG(IFNULL(col, 0))`         | **計入空值**的平均值 (將 NULL 視為 0 納入分母) |
