# 關聯模型的五大鍵

> [!ABSTRACT]
> 深入關聯模型的基本概念，學習主鍵、外鍵、超鍵等鍵值定義，以及功能相依性 (Functional Dependency) 分析。

---

1. 超鍵(Super Key) :符合唯一性的關聯鍵。
2. 候選鍵(Candidate Key) : 符合唯一性(兩個欄位合併也算)以及最小性(欄位合併的最小值)的關聯鍵。
3. ==主鍵(Primary Key)== : 從候選鍵中，挑選出其中一個最具識別意義關聯鍵。
4. 次要鍵(Alternate Key) : 沒有被選為主鍵的其他候選鍵。
5. ==外來鍵(Foreign Key)== : 關聯中被用來參考到其他表格主鍵的關聯鍵。
## 鍵值屬性
鍵值屬性(Key Attribute): 能構成主鍵或候選鍵的所有屬性。反之，則稱為非鍵值屬性(NonKeyAttribute)。
## 關聯模型的限制
- 定義域限制: 指資料庫的表格中的每個屬性質，必須符合該屬性的定義，例如產品名稱必須是字串，薪水必須是整數數字等。
- 關聯鍵限制: 指資料庫的表格中必須有關聯鍵的定義，也就是Super key、 Candidate Key、Primary Key、Alternate Key、Foreign Key。
- 實體完整限制:如果表格中存在主鍵，則不能為空值。
- 參考完整限制:如果關聯存在外來鍵為非空值，必須有可以參考的主鍵。
- 語意完整限制:這個限制不是必須的，但是可以更完備的描述實體世界的資料。例 如交易金額高於100元才可以使用信用卡付款等。
## 功能相依
功能相依(Functional Dependency) 在資料庫設計中非常重要，因為它幫助設計師：
- 識別關聯：理解屬性之間的關聯性，有助於設計更有效的資料表。
- 正規化：透過分析功能相依，可以識別出冗餘資料，進而進行正規化 處理，以減少資料重複、提高資料完整性。 
資料庫設計中的功能相依是指在一個關聯式資料庫中，某一屬性(或屬性組 合)的值能夠唯一地決定另一個屬性(或屬性組合)的值。
若屬性 A 的值能夠唯一地確定屬性 B 的值，則我們稱「B 功能相依於 A」，通常表示為 A → B。
