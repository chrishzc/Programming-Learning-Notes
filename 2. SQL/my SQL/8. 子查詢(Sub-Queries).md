# 子查詢 (Sub-Queries)

> [!ABSTRACT]
> 掌握子查詢的編寫技巧，包括單列子查詢、多列子查詢、相關子查詢 (Correlated Subqueries) 以及 EXISTS 的應用。

---

## 1. 基礎概念
子查詢是一種**巢狀查詢**，即在一個主查詢（Outer Query）中嵌入另一個查詢（Inner Query）。
- **執行順序**：系統會先執行內部的子查詢，並將其結果傳回給外部的主查詢作為運算元使用。
- **目的**：用於將條件細分、分層過濾，或是在無法一次得知條件時進行動態查詢。

### 1.1 可出現的位置
- `SELECT` 子句
- `FROM` 子句（又稱衍生表 Derived Table）
- `WHERE` 子句（最常用）
- `HAVING` 子句

---

## 2. 子查詢的分類

### 2.1 依結果型式分類
- **純量 (Scalar)**：只回傳單一數值。
- **多值 (Multi-valued)**：回傳一整列或一組數值，需配合多筆記錄運算子。
- **多欄位 (Multiple-column)**：
  - > [!TIP]
    > **MySQL 專屬**：可以直接使用 `(col1, col2) = (SELECT ...)` 進行多欄位比較。其他資料庫系統通常需要透過字串連結 (CONCAT) 達成。
- **表格值 (Table-valued)**：回傳一個完整的虛擬資料表，常用於 `FROM` 子句。

### 2.2 依執行方式分類

#### **A. 自主子查詢 (Self-contained)**
子查詢與主查詢無關，可獨立執行。

- **單筆記錄運算子**：`=`, `>`, `>=`, `<`, `<=`, `<>`
  - 條件：子查詢必須用 `( )` 括住，且放在運算子的**右邊**。
- **多筆記錄運算子**：`IN`, `ANY`, `ALL`
  - `ANY`：符合其中一個即可（如 `> ANY` 代表大於最小的一個）。
  - `ALL`：必須符合全部（如 `> ALL` 代表大於最大的一個）。
  - > [!WARNING]
    > 當子查詢含有 `NULL` 且使用 `NOT IN` 時，主查詢將無法回傳任何資料。解決方法是在子查詢中加入 `IS NOT NULL` 過濾。

#### **B. 相關子查詢 (Correlated)**
子查詢必須依賴主查詢傳入的值才能執行。
- **適用情境**：需要「組內比較」或「逐筆比對」時。
- **範例**：查詢每個客戶最新的訂單。
```sql
SELECT custid, ordid, orderdate 
FROM ord AS o1 
WHERE orderdate = (
    SELECT MAX(orderdate) 
    FROM ord AS o2 
    WHERE o1.custid = o2.custid -- 引用外部查詢的別名
);
```

## 3. 存在性測試：EXISTS
`EXISTS` 用於測試子查詢是否能抓到任何資料（回傳真或假）。
- **特性**：它不回傳實際資料，只判斷是否存在，因此子查詢通常使用 `SELECT *`。
- **語法**：`EXISTS` 或 `NOT EXISTS`。

```sql
-- 查詢「有下過訂單」的客戶
SELECT custid, name 
FROM customer AS c 
WHERE EXISTS (
    SELECT * FROM ord AS o 
    WHERE c.custid = o.custid
);
```
