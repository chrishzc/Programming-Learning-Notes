# 資料檢索與索引 (Database Index)

> [!ABSTRACT]
> 深入學習資料庫索引的運作原理（如 B+ 樹），掌握各類索引的建立方法與效能優化策略。

---

索引 (Index) 是一種與資料表相關聯的獨立資料庫結構，主要用於加快在大型資料表中搜尋、過濾與排序資料列的速度。

---

## 1. 資料庫資料擷取方式

資料庫在檢索資料時，主要透過以下兩種模式：

### 1.1 全表掃描 (Table Scan)
- **機制**：當查詢條件的欄位沒有建立索引時，DBMS 會逐一掃描表格中的所有**資料頁 (Data Page)**，以比對並找出符合條件的資料列。
- **儲存單位**：以 MySQL 的 InnoDB 引擎為例，資料是存放在固定大小（預設為 **16 KB**）的邏輯磁碟區塊中，此區塊稱為**資料頁 (Page)**。
- **效能**：在資料量極大時，全表掃描會引發大量的磁碟 I/O，導致查詢速度極度緩慢。

### 1.2 索引掃描 (Index Scan)
- **機制**：利用事先建立好的索引結構，快速跳轉並精確定位到資料列在磁碟中的實際存放位址。
- **樹狀結構**：DBMS 內部的索引通常採用高效的 **B-Tree (平衡多元樹)** 結構。每個節點內部可以包含多個鍵值 (Keys) 和多個指向子節點的指標 (Pointers)，能以極低的搜尋複雜度完成定位。

---

## 2. 索引的核心概念

- **獨立結構**：索引是在資料表之外另外建立與儲存的結構。==刪除索引時並不會影響或破壞表格中的任何實體資料；但若刪除資料表，該表所附帶的所有索引也會被一併刪除。==
- **自動維護**：索引是由資料庫管理系統 (DBMS) 在 DML 操作時自動維護與更新，使用者無需手動干涉。
- **自動與手動建立**：
  - **自動建立**：當我們在欄位上宣告 `PRIMARY KEY`、`UNIQUE` 限制條件時，MySQL 會自動為其建立對應的主索引或唯一索引。
  - **手動建立**：可由資料庫管理員或開發者根據查詢需要，對常用欄位手動新增索引。
- **雙刃劍效能考量**：
  - **優點**：大幅加快資料查詢與過濾的效能。
  - **缺點**：==表格擁有的索引越多，不代表效能越好==。因為每一次進行 `INSERT`、`UPDATE`、`DELETE` 等 DML 操作時，DBMS 都必須同步調整並重構該表的所有 B-Tree 索引結構。過多的索引會嚴重拖慢資料的寫入與修改效能。

---

## 3. 索引的種類

在關係型資料庫中，索引主要分為以下兩大類：

### 3.1 叢集索引 (Clustered Index)
- **特性**：一個資料表**通常會有一個，且只能有一個**叢集索引。
- **儲存模式**：叢集索引的**最底層葉子節點 (Leaf Nodes) 儲存的就是實際的實體資料列**。資料表中的實體資料列，其磁碟存放順序會完全與叢集索引的鍵值順序一致。
- **建立機制**：當資料表設立 `Primary Key` 時，MySQL InnoDB 會自動將其作為叢集索引鍵。如果資料表沒有設定 PK，MySQL 會自動尋找第一個 `NOT NULL` 的 `Unique` 欄位建立；若兩者皆無，InnoDB 會在內部自動生成一個隱式的 6-Byte RowID 作為叢集索引。

### 3.2 非叢集索引 (Non-Clustered Index / 二次索引)
- **特性**：一個資料表可以擁有多個不同的非叢集索引。
- **儲存模式**：非叢集索引的葉子節點**並不存放實際的資料列**，而是存放**資料列定位符 (Row Locator / 叢集索引鍵值)**。當利用非叢集索引找到該定位符後，系統會再次回到叢集索引中檢索實體資料（此過程稱為**回表查詢**）。
- **分類**：分為唯一索引 (Unique Index) 與非唯一索引 (Non-unique Index)。

---

## 4. 索引的建立與維護特性

- ==**自動主索引**：MySQL 會在 `Primary Key` 欄位建立時自動建立主索引以提升搜尋速度。==
- **自動唯一索引**：若表格沒有設定 PK，MySQL 會自動為設定為 `UNIQUE` 限制條件的欄位建立唯一索引。
- **建立限制**：若該欄位中已經含有重複的資料，系統會拒絕為其建立唯一索引 (`UNIQUE INDEX`)。
- **限制條件與索引聯動關係**：
  - ==當主鍵 (PK) 或唯一鍵 (UK) 限制條件被刪除 (Drop) 時，其對應的自動索引會一併被自動刪除。==
  - 當外來鍵 (FK) 限制條件被刪除時，其在建立外來鍵時自動產生的索引**並不會**連同被刪除，需要手動刪除。
  - 刪除整個資料表時，所有的索引都會被自動一併刪除。

---

## 5. 索引管理語法 (CREATE, SHOW, DROP)

我們可以使用以下 SQL 指令來手動建立、查詢與刪除索引：

### 5.1 建立索引 (CREATE INDEX)
```sql
-- 建立一般索引
CREATE INDEX idx_emp_ename 
ON emp (ename);

-- 建立唯一索引 (不允許欄位值重複)
CREATE UNIQUE INDEX uq_emp_email 
ON emp (email);

-- 建立複合索引 (基於多個欄位)
CREATE INDEX idx_emp_dept_sal 
ON emp (deptno, sal);
```

### 5.2 查詢索引 (SHOW INDEX)
可以用於檢查特定資料表上已存在的索引細節與結構：

```sql
SHOW INDEX FROM table_name;
```

### 5.3 刪除索引 (DROP INDEX)
當索引不再需要，或為了加快批次資料寫入效能時，可將其移除：

```sql
-- 方式一：直接刪除
DROP INDEX index_name ON table_name;

-- 方式二：使用 ALTER TABLE
ALTER TABLE table_name 
DROP INDEX index_name;
```
