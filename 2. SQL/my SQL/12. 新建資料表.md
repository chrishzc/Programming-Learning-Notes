# 定義與新建資料表 (CREATE TABLE)

> [!ABSTRACT]
> 學習使用 DDL 新建資料表，定義資料行、資料型態，並為資料表設定主鍵與初始結構。

---

資料表 (Table) 是關係型資料庫中儲存資料的核心實體。在建立資料表時，我們必須明確定義其結構、欄位型態、長度以及各種完整性限制條件。

---

## 1. 資料庫物件簡介 (Database Objects)

資料庫物件是定義在資料庫中，用來儲存或存取資料的物件。常見的資料庫物件包括：
- **資料表 (Table)**：儲存資料的實體。
- **資料限制條件 (Constraints)**：確保資料完整性的規則。
- **視觀表 (View)**：基於 SQL 查詢的虛擬表格。
- **索引 (Index)**：加快資料檢索速度的結構。
- **使用者帳號 (User Account)**：權限控管與安全性設定。
- **內儲程式 (Stored Program)**：如預存程序 (Stored Procedure) 與觸發器 (Trigger)。

> [!NOTE]
> **資料庫物件的維護命令 (DDL)：**
> - **新建物件**：`CREATE`
> - **修改物件**：`ALTER`
> - **刪除物件**：`DROP`

---

## 2. 定義資料表屬性 (Defining Tables)

在新建資料表時，我們需要規劃兩大類型的項目：

### 2.1 必要項目 (Required Elements)
1. **資料表名稱** (Table name)
2. **欄位名稱** (Column names)
3. **欄位資料型態** (Column data types)
4. **欄位資料長度** (Column size)

### 2.2 選擇性項目 (Optional Elements)
- **欄位預設值** (Default value)
- **資料檢查限制條件** (Constraints)：
  - 不允許空值 (`NOT NULL`)
  - 主鍵 (`PRIMARY KEY`)
  - 唯一鍵 (`UNIQUE KEY`)
  - 外來鍵 (`FOREIGN KEY`)
  - 自訂資料檢測 (`CHECK`)

## 3. 新建資料表語法 (CREATE TABLE)

### 3.1 基礎建立權限與命名規則
- **權限要求**：當前連線必須擁有建立資料表的專有權限。
- **命名長度限制**：資料表與欄位名稱的最大長度限制為 **64 Bytes**。
- ==**命名規範**：名稱必須以字母開頭，可用符號包括底線 `_` 或 `$`，但**不可使用系統保留字**。==
- ==**唯一性**：同一個資料庫內，不可以有同名的資料表。==

### 3.2 基礎語法結構
```sql
CREATE TABLE table_name 
(
  column_name1 data_type,
  column_name2 data_type,
  ...
);
```

### 3.3 實務常見：自動記錄時間欄位
在實際專案中，通常會為資料表建立「建立時間」與「最後修改時間」欄位，可利用 `DEFAULT CURRENT_TIMESTAMP` 與 `ON UPDATE CURRENT_TIMESTAMP` 屬性讓資料庫自動維護，無須在應用程式中手動傳入時間：
```sql
CREATE TABLE example (
  id INT PRIMARY KEY AUTO_INCREMENT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

## 5. 新建資料表的進階選項

### 5.1 IF NOT EXISTS 選項
用來在建立表格前，先行判斷該資料表是否已存在，以避免語法報錯。
- ==若資料表已存在，系統會跳出警告但**不建立新資料表**；若不存在則建立新表。==

```sql
CREATE TABLE IF NOT EXISTS table_name 
(
  column_name data_type,
  ...
);
```

### 5.2 設定資料表儲存引擎 (ENGINE)
建立資料表時，可在結尾括號後明確指定儲存引擎（預設為 `InnoDB`）：

```sql
CREATE TABLE TableName 
( ColumnName type,...) 
ENGINE = {BDB | HEAP | ISAM | InnoDB | MERGE | MRG_MYISAM | MYISAM};
```

## 6. 使用既存資料表建立新表

### 6.1 使用子查詢建立新表 (CTAS)
- **特點**：新建資料表的欄位名稱、資料型態與內容將與子查詢回傳的欄位完全一致。
- **欄位別名**：可以在子查詢中使用欄位別名，該別名會直接成為新資料表的欄位名稱。

1. **複製結構與資料**：
   ```sql
   CREATE TABLE table_name 
   AS 
   SELECT column1, column2, ... 
   FROM source_table 
   WHERE condition;
   ```
2. **僅複製結構 (不複製資料)**：在子查詢中加入永遠不成立的篩選條件（例如 `WHERE 1 = 0`）：
   ```sql
   CREATE TABLE table_name 
   AS 
   SELECT column1, column2, ... 
   FROM source_table 
   WHERE 1 = 0;
   ```

### 6.2 使用 LIKE 子句複製結構
- **特點**：完全複製既存資料表的結構定義（包括索引、主鍵等），但**不包含資料**。
- **限制**：無法在此時變更欄位名稱或型態，新舊表的欄位完全同名同屬性。

```sql
CREATE TABLE table_name 
LIKE source_table;
```

## 7. 衍生欄位 (Generated Column)

自 MySQL 5.7.6 起，支援「衍生欄位」特性。該欄位的值並非手動新增，而是**由同資料表中的其他欄位計算/運算所得**。

### 7.1 儲存方式分類
- **`VIRTUAL` (預設值)**：
  - 衍生欄位的值**不佔用實體儲存空間**。每次執行 `SELECT` 查詢時，系統即時動態計算出該值。
- **`STORED`**：
  - 衍生欄位的值會**永久儲存於磁碟中**。每次新增 (INSERT) 或修改 (UPDATE) 時，系統會重新計算並寫入儲存。

### 7.2 衍生欄位的使用限制
- **不支援不確定性函數**：如 `CONNECTION_ID()`、`NOW()`、`RAND()` 等，因為這些函數每次執行的回傳值會隨時改變。
- **不支援預存程序與預存函數**。
- **不支援子查詢**。

### 7.3 語法結構
```sql
column_name data_type [GENERATED ALWAYS] 
AS (expression) 
[VIRTUAL | STORED] [UNIQUE [KEY]] [[NOT] NULL] [[PRIMARY] KEY];
```

## 8. 列舉型態 (ENUM)

MySQL 中的 `ENUM` 是一個字串物件，其值在資料表建立時已明確列出。

- **定義與運作機制**：
  - **資料限缩**：該欄位的值僅能等於清單中所定義的字串。
  - **高效儲存**：在內部儲存時，ENUM 值會被轉換為整數索引值（`1`, `2`, `3` 等）來儲存，比儲存字串更節省儲存空間。
  - **資料型態容納**：ENUM 欄位可以接受各種資料類型的值，包括整數、浮點數、小數和字串。
  - ==**給值方式**==：填入資料時，可以直接給予清單中的**字串值**，或者使用其在清單中的**數字索引**（**從 1 開始計算**）。
  - ==**預設值機制**==：如果此欄位設定為 `NOT NULL`，且新增資料時沒有為其指定值，則系統會自動使用**清單中的第一個選項 (即 Index 1)** 作為預設值。
  - **修改限制**：如果想要修改列舉值/成員，必須使用 `ALTER TABLE` 指令重建整個表格來完成。
  - ==**實務陷阱與警告**==：
    - **避免使用數字字串**：強烈建議**不要**使用數字字串（例如 `'1'`, `'2'`, `'3'`）作為列舉成員。因為當您在 `INSERT` 或 `WHERE` 給予數字 `1` 時，MySQL 容易混淆是指「字串值 `'1'`」還是「內部索引 Index 1（第一個成員）」，這常常會導致難以排查的邏輯 Bug。

### 8.1 語法結構
```sql
column_name ENUM('value1', 'value2', ..., 'valueN');
```
