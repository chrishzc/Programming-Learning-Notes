# 資料庫管理系統 (DBMS) 介紹

> [!ABSTRACT]
> 認識資料庫管理系統的基本定義、分類（關聯式 vs. 非關聯式）以及在資料儲存與管理上的核心優勢。

---

## 1. 什麼是 DBMS？
資料庫管理系統 (Database Management System, DBMS) 提供系統化的方式，讓使用者及其他程式來存取及管理資料，以確保資料的一致性與管理的方便性。
> [!NOTE] 
> ==MySQL 就是一種 DBMS==

---

## 2. DBMS 的核心功能
- **定義結構**：定義資料儲存的結構。
- **資料維護**：提供資料維護的機制。
- **確保安全**：確保資料的安全（防範系統當機、未經授權的存取、誤用等）。
- **併發控制**：提供多人使用下的同時存取控制機制。

---

## 3. 關聯式資料庫 (Relational Database)
1970 年時有人提出關聯式資料模型，後來發展出關聯式資料庫。主體是**用表格 (Table) 儲存資料**，利用列 (Row)、欄 (Column) 串聯信息，是目前業界主流。

### 3.1 主要組成與管理
- **物件或關聯的集合**：資料要儲存成什麼樣子。
- **關聯運算方法**：如何操作這些資料。
- **資料整合條件**：例如資料不能重複等限制。

**管理系統涵蓋：**
- 資料庫物件
- 資料運算
- 資料庫限制條件

### 3.2 重要功能
- ==**支援結構化查詢語言 (SQL)**==：專門用於查詢關聯式資料庫。
- **多人線上存取**：提供多人線上同時存取。
- **權限管理**：提供權限來做資料庫的管理。
- **備份與回復**：提供資料備份與回復的功能。

### 3.3 資料結構
因為關聯式資料庫儲存的是欄、列，因此儲存的資料結構與非關聯式不同：
- **非關聯式資料庫**：`Bit` > `Byte` > `Field` > `Record` > `File` > `Database`
- **關聯式資料庫**：`Bit` > `Byte` > `Column` > `Row` > `Table` > `Database`

> **欄位 (Column) 的特性**：每一個欄位都必須命名，而且必須設定「資料型態」和「資料長度」，用來存放欄位資料。

---

## 4. 關聯式資料庫專用語

![[{F528B765-90A6-4D04-9723-5F1858FD5D91}.png|401]]

1. **主鍵 (Primary Key, PK)**：用於識別每一橫列的欄位組合。
   - 具有唯一性 (Unique)，==不能重複==。
   - 必填，==不為空值 (Not Null)==。
2. **直欄 (Column)**：為實體 (Entity) 或表格中的屬性。
   - 具有相同的資料型態。
   - 儲存相似的內容資料。
3. **橫列 (Row)**：由 1 個以上的直欄所組成的一筆記錄 (Record)，作為 SQL 查詢所回傳的資料單位。
4. **欄位 (Field)**：直欄與橫列交會的地方，作為使用者儲存資料的地方。
5. **空值 (Null)**：欄位如果沒有儲存資料時稱空值。
6. **外來鍵 (Foreign Key, FK)**：欄位所輸入的資料必需出現在所參照的表格中主鍵或唯一鍵的鍵值。
   - 用於與其他表格的主鍵結合，表現兩張表格之間有關係。
   - ==FK 可以是空值，但如果有值的話，一定要在要結合的父表中有對應的值==。

## 5. 非關聯式資料庫 (NoSQL Database)
非關聯式資料庫 (NoSQL, Not Only SQL) 不使用傳統的表格（Row / Column）結構儲存資料，而是採用更具彈性的資料模型。

### 5.1 主要特點
- **無固定綱要 (Schema-less)**：不需要預先定義資料表結構或欄位型態，欄位可動態增減。
- **水平擴充性 (Horizontal Scalability)**：容易透過增加伺服器節點來擴展系統容量（Scale-Out）。
- **高併發讀寫**：適合處理海量資料與高讀寫吞吐量的場景。

### 5.2 常見類型與代表
1. **文件資料庫 (Document)**：以 BSON/JSON 格式儲存資料。例如：MongoDB。
2. **鍵值資料庫 (Key-Value)**：以 Key-Value 鍵值對儲存，讀寫速度極快，常用於快取。例如：Redis。
3. **資料行家族 (Column-Family)**：以資料行 (Column) 為單位儲存，適合大量寫入與分析。例如：Cassandra。
4. **圖形資料庫 (Graph)**：以節點 (Node) 與邊 (Edge) 來儲存實體間的關係。例如：Neo4j。

---

## 6. SQL vs. NoSQL 選擇與比較

| 比較項目 | 關聯式資料庫 (SQL) | 非關聯式資料庫 (NoSQL) |
| :--- | :--- | :--- |
| **資料結構** | 結構固定、嚴格定義 (Structured) | 彈性無綱要、結構多變 (Schema-less) |
| **資料關聯性** | 關聯複雜，常需要多表連接 (JOIN) | 關聯簡單，資料多為獨立或巢狀 (Nested) |
| **交易一致性** | 強調 ACID，保證資料強一致性 | 強調 BASE，追求最終一致性與高可用性 |
| **擴充方式** | 垂直擴充 (Scale-Up，提升單機規格) | 水平擴充 (Scale-Out，增加節點/伺服器) |
| **適用場景** | 金融交易、訂單系統、需要強一致性的系統 | 即時聊天、快取系統、日誌系統、海量大數據分析 |
