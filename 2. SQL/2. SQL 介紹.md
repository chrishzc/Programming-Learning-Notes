# SQL 語言基礎介紹

> [!ABSTRACT]
> 介紹結構化查詢語言 (SQL) 的基本概念、功能分類（DDL、DML、DCL）以及其在關聯式資料庫中的核心地位。

---

## 1. SQL 的核心特性
1. **高級非程序化語言**：允許用戶在高層資料結構上工作。
2. **抽象化儲存**：不需要使用者瞭解其具體的資料存放方式。
3. **通用性**：不同底層結構的資料庫之間，使用相同的 SQL 語言作為資料輸入與管理。
4. **複雜查詢能力**：SQL 語言可以寫出非常複雜的邏輯語句。

---

## 2. SQL 語言的分類
SQL 根據功能主要分為四大類：

| 分類                                       | 全稱                           | 說明                   | 常見指令                                            |
| :--------------------------------------- | :--------------------------- | :------------------- | :---------------------------------------------- |
| **==DDL==**                              | Data Definition Language     | 用於定義資料庫物件（如資料表）。     | `CREATE`, `DROP`, `ALTER`, `RENAME`, `TRUNCATE` |
| **==DML==**[[9. 資料處理語言]] | Data Manipulation Language   | 用於處理資料（新增、修改、刪除、查詢）。 | `INSERT`, `UPDATE`, `DELETE`, `SELECT`          |
| **==DCL==**                              | Data Control Language        | 用於管理權限及安全設定。         | `GRANT`, `REVOKE`                               |
| **==TCL==**                              | Transaction Control Language | 管理資料庫中的交易流程。         | `COMMIT`, `ROLLBACK`, `SAVEPOINT`               |

---

## 3. SQL 命令撰寫規則
為了提升程式碼的可讀性與正確性，請遵循以下規則：
- **大小寫**：指令不分大小寫（但建議關鍵字大寫，以利區分）。
- **多行編寫**：敘述可以編寫成多行，習慣上每個子句佔一行。
- **不可分割**：關鍵字 (Keywords) 不可以分割成多行。
- **結束符號**：以分號 (`;`) 作為敘述的結束。
- **縮排**：使用縮排提升可讀性。
- **註解方式**：
  - **單行註解**：`#` 或 `-- ` (注意 `--` 後需加一空白)。
  - **多行註解**：`/* 註解內容 */`。

---

## 4. SQL 執行順序
SQL 的執行順序與撰寫順序不同，這會影響別名的使用（特別是 [[6. 資料分組(Grouping Data)]] 時）。

### 4.1 標準 SQL 執行順序
在標準規範中，`SELECT` 是在 `WHERE` 和 `HAVING` 之後才執行的，因此欄位別名不能出現在這兩個子句中。

1. `FROM`
2. `WHERE`
3. `GROUP BY`
4. `HAVING`
5. `SELECT`
6. `ORDER BY`

### 4.2 MySQL 執行順序
> [!TIP]
> MySQL 的執行順序較為特殊，`SELECT` 會在 `GROUP BY` 之前處理，因此在 MySQL 中，**欄位別名可以出現在 `HAVING` 子句中**。但為了程式的移植性，建議仍養成遵循標準順序的習慣。

1. `FROM`
2. `WHERE`
3. **`SELECT`** (MySQL 會先解析別名)
4. `GROUP BY`
5. `HAVING`
6. `ORDER BY`
