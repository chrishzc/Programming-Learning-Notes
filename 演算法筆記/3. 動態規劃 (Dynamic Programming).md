# 3. 動態規劃 (Dynamic Programming)

> [!ABSTRACT]
> 本章介紹動態規劃（Dynamic Programming）的最佳子結構與重疊子問題兩大性質，對比備忘錄法與建表法的差別，並深入剖析七大經典實戰案例與狀態轉移方程。

---

## 一、 適用性質 (Optimal Substructure & Overlapping Subproblems)

1. **最佳子結構 (Optimal Substructure)**：問題的最佳解包含其子問題的最佳解。
2. **重疊子問題 (Overlapping Subproblems)**：在遞迴求解過程中，相同的子問題會被重複計算。
3. **自然順序（Natural Ordering）**：DP 適用於問題中的物件具備某種線性或自然的先後順序，且該順序在計算過程中不會改變。
4. **子問題數量**：適用的問題其子問題總數通常應為**多項式數量（Polynomial number）**而非指數數量，否則即使使用 DP，計算成本依然過高

---

## 二、 實作技術：Top-down vs. Bottom-up

*   **Top-down with Memoization (自上而下備忘錄)**：維持原本的遞迴架構，但建立一個查找表（如雜湊表或陣列）記錄已求解之子問題。當遞迴調用遇到已計算過的子問題時，直接回傳紀錄值，避免重複計算。
*   **Bottom-up with Tabulation (自下而上迭代建表)**：從規模最小的子問題開始迭代計算，依序將結果填入表格（二維或一維陣列）中，逐步推導出大問題的解。這種方法沒有遞迴的調用棧開銷，速度通常更快。

---

## 三、 動態規劃七大經典實戰案例

### 1. 帶權重區間調度 (Weighted Interval Scheduling)
相對於一般區間調度（目標是執行最多數量的任務），帶權重版本要求在不重疊的限制下，極大化所選任務的**總價值（Total Value）**。
*   **問題描述**：給定 $n$ 個活動，每個活動 $i$ 有開始時間 $s_i$、結束時間 $f_i$ 與其價值權重 $v_i$。目標是選出一個互不衝突（時間不重疊）的活動子集，使該子集的總價值權重最大化。

#### 1. 前置處理 (Preprocessing)
在使用 DP 公式前，必須先進行兩項準備工作：
*   **按結束時間排序**：將所有 $n$ 個活動按照結束時間 $f_i$ 由小到大排序，確保 $f_1 \le f_2 \le \dots \le f_n$。這是為了建立子問題之間的自然順序。
*   **定義 $p(j)$ 函數**：對於每個活動 $j$，定義 $p(j)$ 為在 $j$ 開始之前就已經結束的最晚活動索引（即與 $j$ 相容且編號最大的活動）。如果沒有任何活動相容，則 $p(j) = 0$。

#### 2. 定義遞迴關係式 (Recurrence Relation)
這是 DP 的核心。我們定義 $OPT(j)$ 為考慮前 $j$ 個活動時所能得到的最大價值。對於第 $j$ 個活動，我們只有兩種選擇：
1. **包含活動 $j$**：獲得價值 $v_j$，再加上與 $j$ 相容的前面活動的最佳解，即 $OPT(p(j))$。
2. **不包含活動 $j$**：直接繼承前 $j-1$ 個活動的最佳解，即 $OPT(j-1)$。

**狀態轉移方程為：**
$$OPT(j) = \max(v_j + OPT(p(j)), OPT(j-1))$$
*基底情況 (Base Case)：* $OPT(0) = 0$。

#### 3. 實作方式
單純使用遞迴會導致大量重複計算（重疊子問題），效能極差。DP 提供兩種實作技巧：
*   **由上而下（Top-down）與備忘錄法（Memoization）**：維持遞迴結構，但在每次計算前先檢查「小抄」（Lookup Table）是否已有答案。若有則直接取用，若無才計算並存入表中。
*   **由下而上（Bottom-up）與迭代法（Iteration）**：使用 `for` 迴圈從小規模子問題開始填表（從 $OPT[1]$ 到 $OPT[n]$）。因為計算 $OPT[j]$ 時所需的 $OPT[p(j)]$ 和 $OPT[j-1]$ 都已事先算好，故能穩定求得解答。

#### 💻 Python 實作
```python
def weighted_interval_scheduling(intervals):
    # intervals 格式: [(start, finish, weight), ...]
    # 1. 依結束時間排序
    intervals = sorted(intervals, key=lambda x: x[1])
    n = len(intervals)
    
    # 為了讓索引從 1 開始，在前面補一個哨兵節點
    intervals = [(0, 0, 0)] + intervals
    
    # 2. 計算 p(j)：尋找與活動 j 相容且結束時間最晚的活動索引
    p = [0] * (n + 1)
    for j in range(1, n + 1):
        s_j = intervals[j][0]
        # 從 j-1 開始往前線性搜尋，找到第一個結束時間 <= s_j 的活動
        for i in range(j - 1, -1, -1):
            if intervals[i][1] <= s_j:
                p[j] = i
                break

    # 3. 迭代填表
    opt = [0] * (n + 1)
    for j in range(1, n + 1):
        opt[j] = max(intervals[j][2] + opt[p[j]], opt[j-1])
        
    return opt[n]
```

#### 4. 效能分析與優點
*   **時間複雜度**：
    *   **基礎版（線性搜尋 $p(j)$）**：排序需要 $O(n \log n)$。對於每個活動 $j$，最壞情況下需要往前掃描所有活動來尋找 $p(j)$，因此計算所有 $p(j)$ 需要 $O(n^2)$。迭代填表需要 $O(n)$。**總時間複雜度最壞情況下為 $O(n^2)$**。
    *   **優化版（二分搜尋 $p(j)$）**：由於活動已按結束時間排序，計算 $p(j)$ 時可改用二分搜尋（如 Python 的 `bisect` 模組）在 $O(\log n)$ 時間內找到相容位置。此時計算所有 $p(j)$ 的總時間降至 $O(n \log n)$，**總時間複雜度優化至 $O(n \log n)$**。
*   **最佳子結構（Optimal Substructure）**：此問題展現了全域最佳解可以由子問題的最佳解組合而成，這是 DP 適用的關鍵特性。
*   **解決貪婪法的不足**：傳統貪婪法（如選最早結束者）在有權重的狀況下無法保證最優，而 DP 透過系統化地考慮「選或不選」每一項決策，確保能找到真正的最大價值。
*   **回溯路徑 (Backtracking)**：若需回溯具體的活動組合，可從 $OPT(n)$ 開始倒推：若 $v_j + OPT(p(j)) > OPT(j-1)$，代表選擇了活動 $j$，接著往 $OPT(p(j))$ 倒退；否則代表沒選活動 $j$，往 $OPT(j-1)$ 倒退。

### 2. 費氏數列 (Fibonacci Sequence)
雖然不是最佳化問題，但它是展示 DP 優化邏輯最經典的教學案例。
*   **問題描述**：計算第 $n$ 項的值。定義為：$F_0 = 0, F_1 = 1, F_n = F_{n-1} + F_{n-2}$。
*   **DP 關鍵**：如果直接使用樸素遞迴，其執行複雜度會呈指數級 $O(2^n)$（因為 $F_n$ 會分裂成兩個子遞迴，以此類推形成一棵滿二元樹）。利用 DP 記錄或迭代計算，可將時間複雜度降為線性時間 $O(n)$。

#### 💻 Fibonacci 兩種優化實作對比
```python
# 實作 A: Top-down 備忘錄法
def fib_memo(n, memo={}):
    if n in memo: return memo[n]
    if n <= 1: return n
    memo[n] = fib_memo(n-1, memo) + fib_memo(n-2, memo)
    return memo[n]

# 實作 B: Bottom-up 空間優化至 O(1) 迭代法 (Tabulation)
def fib_tab_optimized(n):
    if n <= 1: return n
    prev2, prev1 = 0, 1
    for _ in range(2, n + 1):
        curr = prev1 + prev2
        prev2, prev1 = prev1, curr
    return prev1
```

### 3. 子集合加總問題 (Subset Sum Problem)
這是一個極為經典且難度較高的 NP-Complete 問題。
*   **問題描述**：給定 $n$ 個正整數的集合 $S = \{w_1, w_2, \dots, w_n\}$，以及一個限制總和目標值 $W$。目標是在集合中挑選一個子集，使其元素加總在不超過 $W$ 的情況下，數值最大化（即盡量逼近並塞滿限制值 $W$）。
*   **DP 關鍵**：因為選擇物品會影響「剩餘容量」，我們必須將「物品索引 $i$」與「當前剩餘容量 $w$」作為兩個狀態變數，這需要建立一個二維的 DP 表格。
*   **狀態與轉移方程**：
    定義 $DP[i][w]$ 為：使用前 $i$ 個物體，能否湊出精確總重量 $w$（布林值）。
    $$DP[i][w] = DP[i-1][w] \quad \text{if } w < w_i$$
    $$DP[i][w] = DP[i-1][w] \vee DP[i-1][w-w_i] \quad \text{if } w \ge w_i$$

### 4. 背包問題 (Knapsack Problem)
通常指 0/1 背包問題，與子集合加總高度相似，但加入了「價值」維度。
*   **問題描述**：給定 $n$ 個物品，每個物品有重量 $w_i$ 與價值 $v_i$，以及背包的最大限重 $W$。目標是在背包不超重的情況下，使挑選物品的總價值最大化。
*   **狀態與轉移方程**：
    定義 $DP[i][w]$ 為前 $i$ 個物品在背包容量為 $w$ 時的最大價值：
    $$DP[i][w] = \max(DP[i-1][w], DP[i-1][w-w_i] + v_i) \quad \text{if } w \ge w_i$$
    $$DP[i][w] = DP[i-1][w] \quad \text{if } w < w_i$$
*   **💻 空間優化 (一維滾動陣列) Python 實作**：
    ```python
    def knapsack_01(weights, values, W):
        dp = [0] * (W + 1)
        for i in range(len(weights)):
            # 必須反向遍歷，避免同一個物品在同一輪中被重複選取
            for w in range(W, weights[i] - 1, -1):
                dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
        return dp[W]
    ```
*   **效能與擬多項式時間 (Pseudo-polynomial time)**：
    0/1 背包問題的時間複雜度為 $O(nW)$。雖然看起來是多項式時間，但 $W$ 是輸入的數值大小，其二進位表達的長度為 $\log W$。因此，相對於輸入規模而言，其複雜度其實是關於 $\log W$ 的指數級。**當背包容量 $W$ 極大時，DP 的表格會極大，效率將顯著下降。**

### 5. Bellman-Ford 最短路徑演算法 (Shortest Path)
這是在圖形演算法中應用動態規劃的經典案例。
*   **問題描述**：在含有負邊權重 (Negative weights) 的有向圖中，尋找從單一源點 $s$ 到所有點 $v \in V$ 的最短路徑。
*   **DP 關鍵**：Dijkstra 演算法（屬於貪婪法）在負權重下會因為「一經確定便不回溯」而失效。Bellman-Ford 透過對**「路徑經過的邊數上限」**進行歸納，在一層層子問題的推導下尋找最短路徑。
*   **狀態與轉移方程**：
    定義 $DP[i][v]$ 為：從起點 $s$ 出發，經過最多 $i$ 條邊到達點 $v$ 的最短路徑長度。
    $$DP[i][v] = \min \left( DP[i-1][v], \min_{(u, v) \in E} (DP[i-1][u] + w(u, v)) \right)$$
    因為一條無負環的最短路徑最多只包含 $V-1$ 條邊，因此我們只需要迭代計算到 $i = V-1$。如果到第 $V$ 次迭代仍能更新距離，即代表圖中存在**負環 (Negative cycle)**。

### 6. 迷宮繞線 (Maze Routing)
此問題常見於 VLSI（超大型積體電路）實體設計中的晶片自動繞線（Lee Algorithm）。
*   **問題描述**：在充滿障礙物的格狀平面上，尋找從起點 (Source) 到終點 (Target) 的最短無衝突路徑。
*   **DP 關鍵**：
    利用 **波前傳遞 (Wave propagation)** 的概念。從起點出發，將距離為 1 的鄰格填上 1，距離為 2 的鄰格填上 2（如水波向外擴散），遇到障礙物則繞道。直到波前觸及終點。最後再從終點依據填入的數字遞減**回溯 (Backtrack)** 到起點，即可找出最短完整路徑。時間與空間複雜度均為 $O(V) = O(MN)$。

### 7. 旅行推銷員問題 (Traveling Salesman Problem, TSP)
由 Richard Bellman 於 1962 年提出的經典狀態壓縮 DP（State Compression DP）嘗試。
*   **問題描述**：給定 $n$ 個城市與城市之間的距離，找出一條路徑走訪所有城市恰好一次，並最終回到原點的最小花費（最短路徑）。
*   **DP 關鍵**：
    這是一個著名的 NP-Hard 問題。暴力枚舉所有排列的時間複雜度是 $O(n!)$。我們可以使用二進位位元（Bitmask）來表示已經走訪過的城市集合 $S$，這稱為「狀態壓縮」。
*   **狀態與轉移方程**：
    定義二進位整數 $S$ 代表已走訪的城市集合。狀態 $DP[S][i]$ 表示：**走訪過集合 $S$ 中的城市，且當前位於城市 $i$ 的最短路徑長度**。
    $$DP[S][i] = \min_{j \in S, j \neq i} (DP[S \setminus \{i\}][j] + dist(j, i))$$
*   **複雜度優化**：
    使用 DP 可以將時間複雜度降為 **$O(n^2 2^n)$**。雖然對於較大的 $n$ 依然會面臨指數級的狀態爆炸，但比起 $O(n!)$，這已經是極其顯著的優化（例如當 $n=20$ 時，$20! \approx 2.4 \times 10^{18}$，而 $20^2 \times 2^{20} \approx 4.1 \times 10^8$，後者在電腦中可在數秒內算完）。

## 四、 最長共同子序列 (Longest Common Subsequence, LCS)

給定兩個序列 $X = \langle x_1, x_2, \dots, x_m \rangle$ 與 $Y = \langle y_1, y_2, \dots, y_n \rangle$，尋找兩者的最長共同子序列。

*   **狀態定義**：設 $c[i][j]$ 表示子序列 $X[1 \dots i]$ 與 $Y[1 \dots j]$ 的 LCS 長度。
*   **狀態轉移方程**：
    $$c[i][j] = c[i-1][j-1] + 1 \quad \text{if } x_i = y_j$$
    $$c[i][j] = \max(c[i-1][j], c[i][j-1]) \quad \text{if } x_i \neq y_j$$

```python
def lcs(X, Y):
    m, n = len(X), len(Y)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if X[i-1] == Y[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[m][n]
```
