# 7. 基礎資料結構與排序演算法 (Basic Data Structures & Sorting)

> [!ABSTRACT]
> 本章介紹計算機科學中最基礎的資料結構（堆疊、佇列、二元搜尋樹與堆積）與核心排序演算法（堆積排序與線性時間計數排序），為複雜演算法實作提供基石。

---

## 一、 基礎資料結構 (Basic Data Structures)

### 1. 堆疊 (Stack) 與 佇列 (Queue)
- **堆疊 (Stack)**：後進先出 (LIFO)。支援 `push` 與 `pop` 操作，時間複雜度均為 $O(1)$。常用於深度優先搜尋 (DFS) 與函數遞迴調用。
- **佇列 (Queue)**：先進先出 (FIFO)。支援 `enqueue` 與 `dequeue` 操作，時間複雜度均為 $O(1)$。常用於廣度優先搜尋 (BFS)。

### 2. 二元搜尋樹 (Binary Search Tree, BST)
- **性質**：對於任意節點 $x$，其左子樹中所有節點的鍵值均小於 $x.key$，其右子樹中所有節點的鍵值均大於 $x.key$。
- **複雜度**：
  - 搜尋、插入、刪除的最佳/平均時間複雜度為 $O(\log n)$（樹呈平衡狀態）。
  - 最壞情況下為 $O(n)$（樹退化為單鏈串列）。這促使了自平衡樹（如紅黑樹 RBT、AVL 樹）的發展。

### 3. 二元堆積 (Binary Heap)
- **定義**：一棵完全二元樹 (Complete Binary Tree)，滿足堆積性質：
  - **最大堆積 (Max-Heap)**：父節點的鍵值大於或等於子節點的鍵值。
  - **最小堆積 (Min-Heap)**：父節點的鍵值小於或等於子節點的鍵值。
- **關鍵操作**：
  - `Heapify`（維持堆積性質）：$O(\log n)$。
  - `Build-Heap`（自無序陣列建堆）：線性時間 $O(n)$。

---

## 二、 經典排序演算法

### 1. 堆積排序 (Heap Sort)
*   **原理**：
    1. 將輸入陣列構造成一個最大堆積。
    2. 每次將堆積頂端（最大值，即 $A[1]$）與堆積的最後一個元素交換。
    3. 減少堆積的大小，並對新的堆積頂端執行 `Heapify` 以重新維持最大堆積性質。
    4. 重複步驟 2-3 直到堆積大小為 1。
*   **時間複雜度**：$O(n \log n)$（無論最佳、最壞還是平均情況）。
*   **空間複雜度**：$O(1)$（原地排序 In-place）。

### 2. 線性時間排序：計數排序 (Counting Sort)
*   **原理**：非比較排序 (Non-comparison Sort)。假設輸入的 $n$ 個元素都是介於 $0$ 到 $k$ 之間的整數。我們透過計算每個元素出現的次數，並將其累加得到每個元素在輸出陣列中的正確索引位置。
*   **時間複雜度**：$\Theta(n + k)$。當 $k = O(n)$ 時，其時間複雜度為 $\Theta(n)$。
*   **空間複雜度**：$\Theta(n + k)$（需要額外輔助空間）。
*   **穩定性**：計數排序是穩定排序（相同價值的元素在輸出中的相對順序與輸入相同），這使它常被用作基數排序 (Radix Sort) 的子程序。
