# 5. 圖論演算法 (Graph Algorithms)

> [!ABSTRACT]
> 本章介紹圖的表示法與走訪（BFS/DFS），深化單源最短路徑 Dijkstra 演算法，並引入可處理負權邊的 Bellman-Ford 演算法，最後探討最小生成樹（MST）與拓撲排序的實作。

---

## 一、 圖的基本概念與表示法

圖 $G = (V, E)$ 由頂點集 $V$ (Vertices) 與邊集 $E$ (Edges) 組成。在將圖結構輸入電腦時，有兩種最常用的表示法：

### 1. 鄰接矩陣 (Adjacency Matrix)
使用一個大小為 $|V| \times |V|$ 的二維陣列 $A$ 來表示圖。若頂點 $u$ 到頂點 $v$ 有邊相連，則 $A[u][v] = 1$（或其邊權重 $w$），否則為 $0$（或 $\infty$）。
*   **優點**：可以在 $O(1)$ 時間內查詢任意兩點之間是否有邊相連。
*   **缺點**：空間複雜度為 $\Theta(V^2)$，當圖很稀疏時會浪費大量記憶體空間。

### 2. 鄰接串列 (Adjacency List)
為每一個頂點 $u \in V$ 維持一個串列，記錄所有從 $u$ 出發能直接到達的鄰居節點。
*   **優點**：空間複雜度為 $\Theta(V + E)$，對於稀疏圖極為節省空間。
*   **缺點**：查詢任意兩點之間是否有邊相連需要遍歷串列，最壞情況下需要 $O(V)$ 時間。

### 3. 表示法性能對比表

| 操作 / 特性 | 鄰接矩陣 (Adjacency Matrix) | 鄰接串列 (Adjacency List) |
| :--- | :--- | :--- |
| **適用場景** | 稠密圖 (Dense Graph, $E \approx V^2$) | 稀疏圖 (Sparse Graph, $E \ll V^2$) |
| **空間複雜度** | $\Theta(V^2)$ | $\Theta(V + E)$ |
| **查詢兩點是否有邊 $(u, v)$** | $O(1)$ | $O(\text{degree}(u)) \subseteq O(V)$ |
| **遍歷點 $u$ 的所有鄰居** | $\Theta(V)$ | $\Theta(\text{degree}(u))$ |

## 二、 基本圖走訪：BFS 與 DFS

圖的走訪是沿著圖的邊，有系統地訪問圖中所有節點的過程。

### 1. 廣度優先搜尋 (Breadth-First Search, BFS)
*   **核心策略**：以「由近及遠」的同心圓方式探索。先訪問起點的鄰居，再訪問鄰居的鄰居，以此類推。
*   **關鍵機制**：使用**佇列 (Queue)** 來記錄待訪問的節點，並使用一個陣列標記已被訪問的節點，避免重複走訪。
*   **應用場景**：
    *   求解**無權重圖**的單源最短路徑。
    *   尋找連通元件（Connected Components）。
*   **時間複雜度**：$O(V + E)$。

### 2. 深度優先搜尋 (Depth-First Search, DFS)
*   **核心策略**：以「一路探索到底，不撞南牆不回頭」的策略走訪。當前節點還有未訪問的鄰居時，就繼續往深處走；無路可走時，再回溯 (Backtrack) 到上一個節點。
*   **關鍵機制**：使用**遞迴 (Call Stack)** 或顯式的**堆疊 (Stack)**。
*   **時間戳記與性質**：DFS 會記錄每個節點的「發現時間 (Discovery Time, $d[u]$)」與「結束時間 (Finished Time, $f[u]$)」，這在進階圖演算法中非常關鍵。
*   **應用場景**：
    *   偵測圖中是否有環。
    *   拓撲排序 (Topological Sort)。
    *   尋找強連通元件 (Strongly Connected Components, SCC) 的 Tarjan / Kosaraju 演算法。
*   **時間複雜度**：$O(V + E)$。

## 三、 最短路徑演算法 (Shortest Paths)

單源最短路徑問題 (Single-Source Shortest Paths) 是指給定圖 $G=(V,E)$ 與源點 $s$，尋找從 $s$ 到所有其他頂點 $v \in V$ 的最短路徑。

### 1. Dijkstra 演算法
*   **前置條件**：圖中的所有邊權重必須為非負值（$w(u, v) \ge 0$）。若有負邊，該演算法的貪婪選擇會失效。
*   **核心技術**：**鬆弛操作 (Relaxation)**。
    若發現透過點 $u$ 可以讓點 $v$ 的預估距離更短，則更新它：
    $$\text{Relax}(u, v, w): \textbf{if } d[v] > d[u] + w(u, v) \textbf{ then } d[v] = d[u] + w(u, v)$$
*   **運作流程**：
    1. 初始化所有頂點的距離為 $\infty$，源點距離為 $0$。
    2. 使用最小優先佇列 (Min-Priority Queue) 存放未確定最短路徑的頂點。
    3. 每次取出距離最小的頂點 $u$，對其所有鄰居 $v$ 進行鬆弛操作。
    4. 重複步驟 3 直到佇列為空。
*   **時間複雜度**：
    *   使用 Binary Heap / `heapq` (Python) 實作：$O((V + E) \log V)$。
    *   使用 Fibonacci Heap 實作：$O(V \log V + E)$。

### 2. Bellman-Ford 演算法
*   **特點**：可以處理**包含負權重邊**的圖，並且能夠**偵測圖中是否存在負環 (Negative Cycles)**。
*   **核心機制**：動態規劃。一條不包含環的最短路徑最多只包含 $V-1$ 條邊。因此，對所有邊進行 $V-1$ 次完整的鬆弛操作，即可確保找出最短路徑。
*   **負環偵測**：在進行完 $V-1$ 次鬆弛後，進行第 $V$ 次遍歷所有邊。若此時仍有邊可以被成功鬆弛，說明圖中存在一個邊權重總和為負的環路（因為可以無限繞環來減少路徑長度）。
*   **時間複雜度**：$O(VE)$。

#### 💻 Bellman-Ford 極簡 Python 實作
```python
def bellman_ford(vertices, edges, source):
    # 初始化距離陣列
    dist = {v: float('inf') for v in vertices}
    dist[source] = 0
    
    # 進行 V-1 次鬆弛操作
    for _ in range(len(vertices) - 1):
        for u, v, w in edges:
            if dist[u] != float('inf') and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                
    # 第 V 次檢查是否含有負環
    for u, v, w in edges:
        if dist[u] != float('inf') and dist[u] + w < dist[v]:
            raise ValueError("圖中含有可達到的負權環！無法計算最短路徑。")
            
    return dist
```

## 四、 最小生成樹 (Minimum Spanning Tree, MST)

針對無向連通權重圖 $G = (V, E)$，尋找一個包含所有頂點、無環，且邊權重總和最小的樹狀子圖。

> [!NOTE]
> 最小生成樹的算法都基於**貪婪選擇性質**。常見的有 Kruskal 與 Prim 二種。

### 1. Kruskal's 演算法 (加邊法)
*   **策略**：將圖中所有的邊按權重從小到大排序，依序加入不與已選邊形成環 (Cycle) 的邊。
*   **關鍵資料結構**：**不相交集合 (Disjoint-Set / Union-Find)**。
    *   使用 **Path Compression (路徑壓縮)** 與 **Union by Rank (按秩合併)** 技術優化，使得每次的 `Find` 與 `Union` 操作幾乎可在常數時間 $O(\alpha(V))$ 內完成。
*   **時間複雜度**：$O(E \log E)$ 或 $O(E \log V)$，主要代價在於邊的排序。

### 2. Prim's 演算法 (加點法)
*   **策略**：從單一源點開始，每次選取連接「已加入頂點集」與「未加入頂點集」之間權重最小的切邊 (Cut Edge)，並將對應頂點加入。
*   **時間複雜度**：與 Dijkstra 相同，取決於優先佇列的實作方式。使用 Binary Heap 時為 $O(E \log V)$。

## 五、 進階圖論應用：拓撲排序 (Topological Sort)

### 1. 定義
拓撲排序是對**有向無環圖 (Directed Acyclic Graph, DAG)** 的頂點進行的一種線性排序。使得對於任何一條有向邊 $(u, v)$，頂點 $u$ 在排序中都出現在頂點 $v$ 之前。

### 2. 經典演算法：Kahn 演算法 (基於入度)
Kahn 演算法基於 BFS 思想：
1. 計算所有頂點的**入度 (In-degree)**。
2. 將所有入度為 $0$ 的頂點放入佇列中（這代表這些節點沒有任何先決條件）。
3. 當佇列不為空時，取出一個頂點 $u$，將其加入排序結果中。
4. 對於從 $u$ 出發的所有鄰邊 $(u, v)$，將點 $v$ 的入度減 $1$。若減完後 $v$ 的入度變為 $0$，則將 $v$ 放入佇列中。
5. 重複步驟 3 - 4 直到佇列為空。若最終排序結果中的頂點數小於 $|V|$，說明圖中**含有環**，無法進行拓撲排序。

*   **時間複雜度**：$O(V + E)$。
