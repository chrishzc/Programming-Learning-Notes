# 4. 高級設計策略二：貪婪演算法 (Greedy Algorithms)

> [!ABSTRACT]
> 本章介紹貪婪演算法的貪婪選擇性質與其和動態規劃的本質差異，並以霍夫曼編碼、分數背包問題與活動選擇問題展示如何進行局部最優選擇。

---

## 一、 適用性質 (Greedy-Choice Property)

1. **貪婪選擇性質 (Greedy-Choice Property)**：可以透過做出局部最佳（Local Optimal）選擇來達到全局最佳（Global Optimal）解。
2. **最佳子結構**：同動態規劃。

---

## 二、 貪婪法與動態規劃的本質差異

*   **動態規劃**：每一步決策取決於所有相關子問題的解，通常由下而上（Bottom-up）填表後才能確定最終路徑。
*   **貪婪演算法**：在當前步驟直接做出當下最優選擇，不需要考慮後續子問題的解，通常是由上而下（Top-down）進行。

### 經典對比：0/1 背包問題 vs. 分數背包問題
- **0/1 背包問題**：物品不可分割。若採用貪婪法（例如優先選取單位價值最高的物品），可能會因為無法填滿背包容量而導致最終價值非最優。因此**必須用動態規劃**求解。
- **分數背包問題 (Fractional Knapsack)**：物品可以任意分割（例如金粉、液體）。此時**可以使用貪婪演算法**：按單位重量價值（$v_i/w_i$）由高到低進行選擇，直到背包填滿。

```python
def fractional_knapsack(weights, values, capacity):
    # 計算單位價值並排序
    items = sorted(zip(weights, values), key=lambda x: x[1]/x[0], reverse=True)
    total_value = 0.0
    for w, v in items:
        if capacity >= w:
            capacity -= w
            total_value += v
        else:
            total_value += v * (capacity / w)  # 切分物品
            break
    return total_value
```

## 三、 經典範例一：活動選擇問題 (Activity Selection)

給定一組活動，每個活動都有開始時間 $s_i$ 與結束時間 $f_i$。求互不衝突的最大相容活動集合。
*   **貪婪策略**：每次選擇**結束時間最早 ($f_i$ 最小)**且與已選活動相容的活動。

```python
def activity_selection(start_times, finish_times):
    # 將活動依結束時間升序排序
    activities = sorted(zip(start_times, finish_times), key=lambda x: x[1])
    selected = [activities[0]]
    last_finish = activities[0][1]
    
    for s, f in activities[1:]:
        if s >= last_finish:
            selected.append((s, f))
            last_finish = f
    return selected
```

## 四、 經典範例二：霍夫曼編碼 (Huffman Codes)

*   **策略**：每次選擇出現頻率最低的兩個節點合併，建立一棵最佳前綴碼樹（Optimal Prefix Tree）。
*   **時間複雜度**：$O(n \log n)$，主要代價在於利用最小優先佇列（Binary Heap）進行節點存取。

## 五、 擬陣理論簡介 (Matroid Theory)

擬陣是一個組合結構，可以用來數學地證明某些問題是否可以用貪婪演算法求得最優解。一個擬陣 $M = (S, I)$ 包含一個有限集 $S$ 與其獨立子集的族 $I$，滿足：
1. **遺傳性 (Hereditary)**：若 $B \in I$ 且 $A \subseteq B$，則 $A \in I$。
2. **交換性 (Exchange Property)**：若 $A, B \in I$ 且 $|A| < |B|$，則存在一個元素 $x \in B - A$，使得 $A \cup \{x\} \in I$。
*   **結論**：若一個問題可以建模為尋找擬陣中權重最大的獨立子集，則**貪婪演算法保證能得到最優解**（如 Kruskal 演算法求解 MST）。
