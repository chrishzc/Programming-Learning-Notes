## 一、 均攤分析 (Amortized Analysis)

當一個操作序列中，某些特定操作代價極高，但多數操作代價極低時，用以計算其平均每次操作的代價。均攤分析保證的是最壞情況下的平均表現（Worst-case average），而非期望值。

### 💡 實例推導：動態陣列 (Dynamic Array) 的 push_back 擴容
考慮一個初始容量為 $1$ 的動態陣列，每次當陣列滿時，我們將容量翻倍。向陣列插入第 $i$ 個元素時：
- 若陣列未滿，實際代價為 $1$。
- 若陣列滿，我們需要分配雙倍空間並拷貝已有的 $i-1$ 個元素，實際代價為 $i$。

#### 1. 聚合分析 (Aggregate Analysis)
在 $n$ 次連續插入操作中，只有當 $i - 1$ 為 $2$ 的冪（即 $i-1 = 1, 2, 4, 8, \dots$）時會觸發擴容拷貝。
$n$ 次操作的總代價為：
$$T(n) = n + \sum_{j=0}^{\lfloor \log_2(n-1) \rfloor} 2^j < n + 2n = 3n$$
因此，每次操作的均攤代價為 $\frac{T(n)}{n} < 3 = O(1)$。

#### 2. 記帳法 (Accounting Method)
我們對每次普通的插入操作收取費率（均攤代價）為 $3$ 元。
- 其中 $1$ 元用來支付當前插入的實際成本。
- 剩餘的 $2$ 元作為「信用」(Credit) 存入該元素中。
當陣列在容量為 $k$ 滿載需要翻倍至 $2k$ 時，我們需要 $k$ 元來拷貝這 $k$ 個元素。此時，這 $k$ 個元素中後半段新插入的 $k/2$ 個元素，每個身上都存有 $2$ 元信用，總共剛好有 $k$ 元，恰好可以完全支付擴容拷貝的代價。
因此，均攤代價為 $O(1)$。

#### 3. 位能法 (Potential Method)
定義位能函數 $\Phi(D) = 2 \cdot size - limit$。
- 當容量翻倍後，立即有 $size = limit/2$，位能 $\Phi = 0$。
- 在觸發下一次翻倍前，$\Phi$ 隨著新插入元素穩定增長，直到 $size = limit$ 時位能達到最大值 $\Phi = limit$。
- 操作 $i$ 的均攤代價為：$\hat{c}_i = c_i + \Phi(D_i) - \Phi(D_{i-1})$。
  - **情況 A（不擴容）**：$c_i = 1$，$\Phi(D_i) - \Phi(D_{i-1}) = (2 \cdot size_i - limit) - (2 \cdot (size_i - 1) - limit) = 2$。
    $$\hat{c}_i = 1 + 2 = 3$$
  - **情況 B（觸發擴容）**：$c_i = size_i$，擴容前 $size_{i-1} = limit_{i-1}$，擴容後 $limit_i = 2 \cdot limit_{i-1}$。
    $$\Phi(D_{i-1}) = limit_{i-1}$$
    $$\Phi(D_i) = 2 \cdot size_i - 2 \cdot limit_{i-1} = 2(limit_{i-1} + 1) - 2 \cdot limit_{i-1} = 2$$
    $$\hat{c}_i = size_i + 2 - limit_{i-1} = (limit_{i-1} + 1) + 2 - limit_{i-1} = 3$$
解得不論是否擴容，均攤代價皆為 $3 = O(1)$。

## 二、 NP-Completeness 理論概要

- **P 類別 (Polynomial Time)**：可在多項式時間內**求解**的問題集合。
- **NP 類別 (Nondeterministic Polynomial Time)**：給定一個候選解，可在多項式時間內**驗證**其正確性的問題集合。
- **NP-Hard**：若所有 NP 問題皆可在多項式時間內規約（Reduce）至問題 $A$，則問題 $A$ 為 NP-Hard。
- **NP-Complete (NPC)**：若問題 $A$ 同時屬於 NP 且屬於 NP-Hard，則 $A$ 為 NP-Complete。

### 1. 經典 NPC 問題列表
1. **3-SAT**：給定一個 Boolean 公式，每個子句包含 3 個文字，是否存在一組真值賦予使公式為真。
2. **獨立集問題 (Independent Set)**：在圖中尋找一個大小為 $k$ 的頂點集合，集合內任意兩點皆無邊相連。
3. **頂點覆蓋問題 (Vertex Cover)**：尋找一個大小為 $k$ 的頂點集合，使得圖中每條邊都至少有一個端點屬於該集合。
4. **旅行推銷員問題 (TSP)**：尋找一條拜訪所有頂點恰好一次且總權重小於 $k$ 的迴路。

### 2. 規約實例：從 3-SAT 規約到 Independent Set
要證明 Independent Set 是 NPC，我們首先驗證其屬於 NP，接著將已知的 NPC 問題 3-SAT 規約至 Independent Set。

**規約構造步驟**：
1. 設 3-SAT 公式有 $k$ 個子句，例如：$C_1 = (x_1 \vee \neg x_2 \vee x_3)$。
2. 對於每個子句，我們在圖中建立一個包含 3 個頂點的「三角形」(Triangle)，三個頂點分別對應子句中的三個文字。三角形的邊保證了同一個子句中只能選擇至多一個頂點作為獨立集成員。
3. 對於所有代表互為相反變數的文字（例如 $x_1$ 與 $\neg x_1$），在它們對應的頂點之間連上一條邊。這保證了在獨立集中，我們不能同時將 $x_1$ 與 $\neg x_1$ 都設為真。
4. **結論**：該構造可在多項式時間內完成。3-SAT 公式有解，若且唯若該構造出的圖存在一個大小為 $k$（子句數）的獨立集。規約完成。
