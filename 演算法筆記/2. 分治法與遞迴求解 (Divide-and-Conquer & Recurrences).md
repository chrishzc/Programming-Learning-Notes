# 2. 分治法與遞迴求解 (Divide-and-Conquer & Recurrences)

> [!ABSTRACT]
> 本章介紹分治法（Divide-and-Conquer）的核心三步驟與經典合併排序法，並深入探討求解遞迴式複雜度的三種核心方法：代入法、遞迴樹法與主定理。

---

## 一、 分治法三步驟

1. **Divide**：將原問題分割為數個規模較小的獨立子問題。
2. **Conquer**：遞迴地求解各個子問題。若子問題規模足夠小，則直接求解。
3. **Combine**：將子問題的解合併為原問題的解。

---

## 二、 經典範例一：合併排序法 (Merge Sort)
*   **分治應用**：
    1.  **Divide (分割)**：尋找中間點，將陣列切分為左右兩半。
    2.  **Conquer (克服)**：遞迴地調用排序函數，分別將左右兩半排好序。
    3.  **Combine (合併)**：這是 Merge Sort 的核心。**使用雙指標 (Double Pointers) 分別指向左右兩個已排序子陣列的起點，互相比較大小**，每次將較小的元素挑出來放入暫存陣列中，直到兩邊都取完。
*   **時間複雜度遞迴式**：$T(n) = 2T(n/2) + \Theta(n)$，解為 $T(n) = \Theta(n \log n)$。
*   **空間複雜度**：$\Theta(n)$。因為在 **Combine** 階段進行雙指標比較與合併時，必須建立一個新的暫存陣列（即代碼中的 `result = []`）來存放排序後的數據。
*   **正確性**：藉由雙指標的有序比較，可確保合併後的陣列依然維持正確的升序，且不會破壞相同元素的相對順序（穩定排序）。

#### 💻 Merge Sort Python 實作
```python
def merge_sort(arr):
    # Base Case: 當陣列只有 1 個元素時，已排序，直接回傳
    if len(arr) <= 1:
        return arr
        
    # 1. Divide: 尋找中間點，將陣列切分為左右兩半
    mid = len(arr) // 2
    
    # 2. Conquer: 遞迴地排序左半部與右半部
    left_sorted = merge_sort(arr[:mid])
    right_sorted = merge_sort(arr[mid:])
    
    # 3. Combine: 合併兩個已排序的子陣列，並回傳結果
    return merge(left_sorted, right_sorted)

def merge(left, right):
    result = []
    i = j = 0
    
    # 雙指標比較，將較小的元素依序放入 result
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
            
    # 補齊剩餘未放完的元素
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

## 三、 經典範例二：快速排序法 (Quick Sort) 

快速排序法也是分治法的經典應用，但與合併排序不同，它的核心在於**「分割 (Partition)」**而非「合併」。以下是快速排序法從簡單到複雜的設計演進過程：

### 階段一：直覺理解 — 極簡 Python 5 行版 (非原地排序)
這是最直觀的快速排序寫法，使用 Python 列表生成式，不需要任何複雜指標，能讓人立刻理解其分治核心：

```python
def quick_sort_simple(arr):
    # Base Case: 當陣列為空或只有 1 個元素時，不需排序，直接回傳
    if len(arr) <= 1:
        return arr
        
    # 1. Choose Pivot: 任意選取一個基準點（這裡選中間的元素）
    pivot = arr[len(arr) // 2]
    
    # 2. Divide: 將陣列分割為小於、等於、大於基準點的三部分
    left   = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right  = [x for x in arr if x > pivot]
    
    # 3. Conquer & Combine: 遞迴排序左右兩部分，並直接相加合併
    return quick_sort_simple(left) + middle + quick_sort_simple(right)
```

> [!WARNING]
> **階段一的效能瓶頸：**
> 雖然這個寫法極易閱讀，但它在每次分割時都透過 `[x for x in arr ...]` **在記憶體中建立全新的列表 (List)**。
> 這會消耗高達 $O(n \log n)$ 級別的額外記憶體空間，在處理大數據時會導致記憶體崩潰。因此，工業級實作必須進行「工程優化」。

---

### 階段二：工程優化 — 原地交換版 (In-place Lomuto Partition)
為了將額外空間複雜度優化至 $O(\log n)$（僅調用棧空間），我們必須直接在**原始陣列內部進行元素交換**，這稱為「原地排序 (In-place)」。這需要引入經典的 **Lomuto Partition 演算法**。

> [!IMPORTANT]
> **雙指標參數物理意義：**
> - **`arr`**：待排序的原始陣列（就地修改，無須回傳值）。
> - **`low`**：當前要排序子陣列的**起點索引 (包含)**。
> - **`high`**：當前要排序子陣列的**終點索引 (包含)**。我們通常選擇最後一個元素 `arr[high]` 作為比較基準點 (Pivot)。
> - **`i` (左指針)**：記錄**「所有已發現小於等於 Pivot 的元素中，最後一個的索引位置」**。一開始因為還沒有任何元素被比較，所以初始化為 `low - 1`（例如 `-1`）。
> - **`j` (右指針)**：從 `low` 掃描到 `high - 1` 的工作指針。
>
> 💡 **防呆問答：`i` 一開始是 `low - 1`（比如 `-1`），讀寫陣列不會越界出錯嗎？**
> 完全不會！因為在迴圈內，只有當我們發現 `arr[j] <= pivot` 時，我們是**先執行 `i += 1`，然後才進行交換 `arr[i], arr[j]`**。
> 也就是說，當發現第一個符合的元素時，`i` 已經從 `-1` 變為 `0`。在整個執行過程中，`i` 從未在值為 `-1` 時去讀寫陣列。

#### 💻 In-place Quick Sort Python 實作
```python
def quick_sort_inplace(arr, low, high):
    if low < high:
        # Divide: 分割並取得 Pivot 擺放的正確索引位置
        pivot_idx = partition(arr, low, high)
        
        # Conquer: 原地遞迴排序左半部與右半部
        quick_sort_inplace(arr, low, pivot_idx - 1)
        quick_sort_inplace(arr, pivot_idx + 1, high)

def partition(arr, low, high):
    pivot = arr[high]  # 選擇最右側元素作為基準點
    i = low - 1        # 初始化左指針（代表無效邊界）
    
    for j in range(low, high):
        # 若當前元素比基準點小，將其與左邊界下一個元素交換
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
            
    # 最後，將基準點放置到正確的分割點位置 (i + 1)
    arr[i + 1], arr[high] = arr[high], arr[i + 1]
    return i + 1  # 回傳分割點的索引
```

#### 🔍 Lomuto Partition 雙指標模擬實例 (以 `[4, 7, 3, 5]` 為例)
為了直觀理解交換機制，我們以陣列 `arr = [4, 7, 3, 5]` 進行一次 Partition 模擬。
- 基準點 **`pivot = 5`** (最右側元素，`high = 3`)。
- 左指標 **`i = -1`** (小於等於區邊界)。
- 工作指標 **`j`** 從 `0` 掃描到 `2`。

1. **當 `j = 0` (指向 `4`)**：
   - 比較：`4 <= 5` (成立)。
   - 動作：`i` 右移一格變為 `0`。交換 `arr[0]` 與 `arr[0]`（自己與自己交換，陣列不變：`[4, 7, 3, 5]`）。
   - *此時分區*：【小於等於區】為 `[4]`（索引 `0 ... 0`）。

2. **當 `j = 1` (指向 `7`)**：
   - 比較：`7 <= 5` (不成立)。
   - 動作：什麼都不做，`i` 保持在 `0`。
   - *此時分區*：【小於等於區】為 `[4]`，**【大於區】為 `[7]`**（`i` 與 `j` 開始拉開距離，大數 `7` 被卡在中間）。

3. **當 `j = 2` (指向 `3`)**：
   - 比較：`3 <= 5` (成立)。
   - 動作：`i` 右移一格變為 `1`（此時 `arr[1]` 存放的正是剛才卡在中間的大數 `7`）。交換 `arr[1]` 與 `arr[2]`（3 與 7 交換，陣列變為：`[4, 3, 7, 5]`）。
   - *此時分區*：【小於等於區】擴張為 `[4, 3]`（索引 `0 ... 1`），【大於區】變為 `[7]`（大數 `7` 被成功推到後面）。

4. **迴圈結束，將 Pivot (`5`) 插回中間**：
   - 動作：交換 `arr[i + 1]` (即 `arr[2]` 的 `7`) 與 `arr[high]` (即 `arr[3]` 的 `5`)。
   - 最終結果：陣列變為 **`[4, 3, 5, 7]`**，回傳分割點索引 `i + 1 = 2`。
   - *結論*：`5` 左側的 `[4, 3]` 皆比 `5` 小；右側的 `[7]` 皆比 `5` 大，分割完美成功！

#### 🔍 遞迴呼叫流程追蹤 (Recursive Call Trace)
接續上述步驟，此時第一層 `partition` 分割完畢，陣列狀態為 **`[4, 3, 5, 7]`**，回傳的分割點 **`pivot_idx = 2`**（對應元素為 `5`）。
這意味著元素 `5` 已經被擺放在它的「終極正確位置」上，後續遞迴不會再移動它。

接下來，`quick_sort_inplace(arr, 0, 3)` 經理函數會依據 `pivot_idx = 2`，拆分出左右兩個子區間，並指派兩個分身去遞迴排序：
- **左半部 (小於區)**：範圍從 `low` 到 `pivot_idx - 1` (即 `0` 到 `1`，對應子陣列 `[4, 3]`)。
- **右半部 (大於區)**：範圍從 `pivot_idx + 1` 到 `high` (即 `3` 到 `3`，對應子陣列 `[7]`)。

---

##### 1. 左分身執行：`quick_sort_inplace(arr, 0, 1)`（處理 `[4, 3]`）
1. 因為 `low (0) < high (1)`，條件成立，左分身繼續呼叫 `partition(arr, 0, 1)` 進行分割：
   - 基準點 **`pivot = 3`** (最右側元素，即 `arr[1] = 3`)。
   - `i = low - 1 = -1`。工作指標 `j` 從 `0` 掃描到 `0`。
   - 當 `j = 0` (指向 `4`)：比較 `4 <= 3` (不成立)。
   - 迴圈結束。交換 `arr[i + 1]` (即 `arr[0]` 的 `4`) 與 `arr[high]` (即 `arr[1]` 的 `3`)。
   - 交換後，子區間變為 `[3, 4]`，此時整張大陣列狀態為：**`[3, 4, 5, 7]`**。
   - 回傳新分割點 `pivot_idx = 0`。
2. 左分身繼續指派它的兩個子分身：
   - **左子分身**：`quick_sort_inplace(arr, 0, -1)` $\to$ `low(0) >= high(-1)`，直接停工。
   - **右子分身**：`quick_sort_inplace(arr, 1, 1)` $\to$ `low(1) >= high(1)`，直接停工。
3. 左分身宣告任務完成。

---

##### 2. 右分身執行：`quick_sort_inplace(arr, 3, 3)`（處理 `[7]`）
1. 因為 `low (3) >= high (3)`，條件不成立。
2. 右分身判定區間只剩 1 個元素，不需要排序，**直接宣告停工**。
3. 至此，所有遞迴分身皆停工。大陣列已被就地修改為：**`[3, 4, 5, 7]`**，排序完全正確！

---

### 階段三：安全防禦 — 隨機化快速排序 (Randomized Quick Sort)
*   **最壞情況的危機**：
    在階段二的實作中，如果我們總是固定選最右側 `arr[high]` 作為 Pivot，當遇到一個**已經排序好**（或逆序）的陣列時，每次 Partition 都會切分極度不均勻（一邊是 0 個，另一邊是 $n-1$ 個）。此時時間複雜度會退化到最差的 **$O(n^2)$**。
*   **解決方案**：
    在進行 Partition 之前，隨機選擇區間 `[low, high]` 內的一個元素，與最右側的 `arr[high]` 交換，這稱為「隨機化樞紐」。這樣一來，無論輸入資料如何分佈，期望時間複雜度都能穩定保持在 **$O(n \log n)$**。

#### 💻 Randomized Quick Sort Python 實作
```python
import random

def randomized_quick_sort(arr, low, high):
    """
    隨機化快速排序主函數 (工業級實作)
    初次呼叫：randomized_quick_sort(arr, 0, len(arr) - 1)
    """
    if low < high:
        pivot_idx = randomized_partition(arr, low, high)
        randomized_quick_sort(arr, low, pivot_idx - 1)
        randomized_quick_sort(arr, pivot_idx + 1, high)

def randomized_partition(arr, low, high):
    # 隨機選取一個索引，並與 high 位置 of 元素交換，打破特定的輸入分佈
    rand_idx = random.randint(low, high)
    arr[rand_idx], arr[high] = arr[high], arr[rand_idx]
    return partition(arr, low, high)  # 呼叫標準的 Lomuto Partition 進行分割
```

| 演算法                    | 平均時間複雜度       | 最壞時間複雜度         | 額外空間複雜度       | 穩定性 (Stable) | 核心特徵            |
| :--------------------- | :------------ | :-------------- | :------------ | :----------- | :-------------- |
| **Merge Sort**         | $O(n \log n)$ | $O(n \log n)$   | $O(n)$        | 是            | 穩定，但需要額外記憶體空間   |
| **Quick Sort (極簡版)**   | $O(n \log n)$ | $O(n^2)$        | $O(n \log n)$ | 否            | 極易閱讀，但極度浪費空間    |
| **Quick Sort (原地隨機版)** | $O(n \log n)$ | $O(n^2)$ （機率極低） | $O(\log n)$   | 否            | 工業級實作，省記憶體，快取友善 |

## 四、 二分搜尋法 (Binary Search) 的分治視角

二分搜尋法是分治法最簡單的應用之一（每次只 Conquer 一個子問題，不需要 Combine 步驟）：

```python
def binary_search(arr, low, high, target):
    if low > high:
        return -1  # 未找到
        
    mid = (low + high) // 2
    if arr[mid] == target:
        return mid
    elif arr[mid] > target:
        return binary_search(arr, low, mid - 1, target)  # 搜尋左半邊
    else:
        return binary_search(arr, mid + 1, high, target)  # 搜尋右半邊
```
*   **時間複雜度遞迴式**：$T(n) = T(n/2) + O(1)$，解為 $T(n) = O(\log n)$。

## 五、 遞迴式求解方法 (Solving Recurrences)
以合併排序法 (Merge Sort) 為例： 它把大小為 n 的問題，切成 2 個大小為 n/2n/2 的子問題（各花費 T(n/2)T(n/2) 步驟），最後用花費 n 步驟的 merge 函數合併。 因此，我們寫出它的步驟方程式： T(n)=2T(n/2)+nT(n)=2T(n/2)+n 這個等號右邊依然包含 TT 的方程式，就叫做 **「遞迴式」**。
==所謂的「求解」，就是要把這個包含 TT 的遞迴式，**「轉化為不包含 TT 的、乾淨的漸進時間複雜度表示式**==

1. **代換法 (Substitution Method)**：猜測一個解，再利用數學歸納法（Mathematical Induction）證明。
2. **遞迴樹法 (Recursion-Tree Method)**：將遞迴程序想像成一棵樹，每個節點代表該層級運算所貢獻的複雜度，最後將整棵樹所有節點的貢獻加總。
3. **主定理 (Master Theorem)**：==把最常見的分治法變成公式查詢==。適用於 $T(n) = aT(n/b) + f(n)$ 型態，其中 $a \ge 1, b > 1$。
    - 若 $f(n) = O(n^{\log_b a - \epsilon})$，則 $T(n) = \Theta(n^{\log_b a})$。
    - 若 $f(n) = \Theta(n^{\log_b a} \log^k n)$，則 $T(n) = \Theta(n^{\log_b a} \log^{k+1} n)$。
    - 若 $f(n) = \Omega(n^{\log_b a + \epsilon})$ 且滿足 regularity condition，則 $T(n) = \Theta(f(n))$。

### 💡 主定理無法直接求解之例外推導
考慮遞迴式：$T(n) = 2T(n/2) + n \log n$。
此處 $a=2, b=2$，臨界項 $n^{\log_b a} = n^1 = n$。而 $f(n) = n \log n$。
因為 $\lim_{n \to \infty} \frac{n \log n}{n} = \log n$，這比 $n$ 增長得快，但不是多項式地快（無法表示為 $n^{\epsilon}$ 形式），所以主定理的第一與第三情況均不適用。此外，雖然與第二情況相似，但其指數不是常數。

#### 遞迴樹解法推導：
1. 樹的第 $i$ 層有 $2^i$ 個節點，每個節點 the 規模為 $n/2^i$。
2. 第 $i$ 層的總步驟數為：
   $$2^i \cdot \left( \frac{n}{2^i} \log \frac{n}{2^i} \right) = n \log \left( \frac{n}{2^i} \right) = n (\log n - i)$$
3. 樹的高度為 $\log n$。將所有層的步驟數加總：
   $$T(n) = \sum_{i=0}^{\log n - 1} n (\log n - i) + \Theta(n)$$
   $$T(n) = n \log n \sum_{i=0}^{\log n - 1} 1 - n \sum_{i=0}^{\log n - 1} i + \Theta(n)$$
   $$T(n) = n \log^2 n - n \frac{(\log n - 1)\log n}{2} + \Theta(n) = \frac{1}{2} n \log^2 n + \frac{1}{2} n \log n + \Theta(n)$$
4. 因此，該遞迴式的緊密複雜度為：**$T(n) = \Theta(n \log^2 n)$**。
