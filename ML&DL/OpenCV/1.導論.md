# OpenCV 影像處理導論

> [!ABSTRACT]
> 本篇筆記介紹 OpenCV 影像處理的基礎觀念，包含色彩模型（BGR、灰階、透明度通道）、基本影像讀取與寫入，以及如何透過 NumPy 對影像矩陣進行像素修改與切片操作。

---

## 1. 色彩模型基礎

### 1.1 紅綠藍色彩空間 (RGB & BGR)
**RGB** 是最常用的數位色彩模型。它是透過將紅（Red）、綠（Green）、藍（Blue）三種顏色的光，以不同的比例與亮度混合，從而創造出自然界中絕大數的色彩（又稱「加色法」，Additive Color）。

> [!WARNING]
> **OpenCV 預設色彩通道順序為 BGR**
> 與常見的 RGB 順序不同，OpenCV 在讀取彩色圖片時，預設的通道順序是 **B（藍）、G（綠）、R（紅）**。
>
> 影像資料在記憶體中以 NumPy 三維陣列結構儲存，形狀為 `(高度, 寬度, 通道數)`（即 `(Row, Col, Channel)`）。
> - **單通道 (Single Channel)**：顯示卡只會收到亮度值，影像會以**灰階 (Grayscale)** 顯示。
> - **雙通道 (Double Channels)**：缺少一個顏色通道，顏色會嚴重失真。
> - **三通道 (Triple Channels)**：正常的 BGR 彩色影像。

---

### 1.2 灰階影像 (Grayscale)
灰階影像中，每個像素僅包含一個「亮度」數值，範圍介於 `0`（全黑）到 `255`（全白）之間。當 RGB 三個通道的數值完全相等時，該像素即呈現灰色。

#### 灰階轉換方法：加權平均法
人眼對不同顏色的光敏感度不同（對綠色最敏感，對藍色最遲鈍）。為了使轉換後的灰階影像更符合人類視覺感受，OpenCV 與標準演算法常採用以下加權公式進行轉換：

$$Y = 0.299 \times R + 0.587 \times G + 0.114 \times B$$

## 2. 影像基本操作

在 Python 中使用 OpenCV，必須先匯入 `cv2` 模組：
```python
import cv2
```

### 2.1 讀取影像 (`cv2.imread`)
語法：
```python
img = cv2.imread(filepath, flags)
```

#### 常用的讀取模式標記 (Flags)
| 數值 | 語法標記 (Flags) | 含意 |
| :---: | :--- | :--- |
| **`-1`** | `cv2.IMREAD_UNCHANGED` | 保持原格式不變（若有 alpha 通道則保留）。 |
| **`0`** | `cv2.IMREAD_GRAYSCALE` | 以單通道灰階模式讀取影像。 |
| **`1`** | `cv2.IMREAD_COLOR` | 以 3 通道 BGR 彩色模式讀取影像（**預設值**）。 |

範例：
```python
# 讀取彩色圖片 (預設)
img_color = cv2.imread('./image/baby.jpg', cv2.IMREAD_COLOR)

# 讀取灰階圖片 (使用數值 0)
img_gray = cv2.imread('./image/baby.jpg', 0)
```

---

### 2.2 顯示影像 (`cv2.imshow`)
語法：
```python
cv2.imshow(winname, mat)
```

#### 數值對應與顯示邏輯
- **8 位元無符號整數 (`uint8`)**：像素值範圍 `[0, 255]`，直接按原樣顯示。
- **32 位元或 64 位元浮點數 (`float32` / `float64`)**：像素值會被乘以 255。即範圍 `[0.0, 1.0]` 會被映射至 `[0, 255]` 顯示。

> [!IMPORTANT]
> **必須搭配 `cv2.waitKey` 與 `cv2.destroyAllWindows`**
> 如果在顯示影像後不呼叫 `waitKey()`，視窗將會立即關閉或呈現當死（無回應）狀態。
> - `cv2.waitKey(delay)`：等待鍵盤事件。參數為等待毫秒數；若設為 `0` 則表示無限等待，直到使用者按下任意鍵。
> - `cv2.destroyAllWindows()`：關閉所有建立的 OpenCV 視窗。

範例：
```python
# 顯示影像視窗
cv2.imshow('My Window', img_color)

# 等待任意鍵按下 (若按下 ESC 鍵則退出，ESC 的 ASCII code 為 27)
key = cv2.waitKey(0)
if key == 27:
    print("使用者按下 ESC 退出")

# 釋放所有視窗資源
cv2.destroyAllWindows()
```

---

### 2.3 儲存影像 (`cv2.imwrite`)
語法：
```python
cv2.imwrite(filename, img)
```

範例：
```python
# 讀取並轉換成灰階影像
img_gray = cv2.imread('./image/baby.jpg', cv2.IMREAD_GRAYSCALE)

# 將灰階影像儲存為新檔案
cv2.imwrite('./image/baby_gray.jpg', img_gray)
```

## 3. 影像矩陣與像素操作 (NumPy)

OpenCV 在 Python 中直接將影像讀入為 **NumPy N維陣列 (`numpy.ndarray`)**。因此，所有對影像的操作本質上都是對 NumPy 陣列的切片與數值存取。

### 3.1 取得與修改像素值
> [!NOTE]
> 像素的存取順序為：`img[row, col, channel]`，對應到影像座標即是 `img[y, x, channel]`。
> 由於預設是 BGR 格式，通道索引分別為：`0 -> Blue`, `1 -> Green`, `2 -> Red`。

```python
# 取得位於 (y=100, x=50) 的 BGR 三通道數值
pixel_bgr = img[100, 50]
print(pixel_bgr)  # 輸出例如: [255, 120, 50] (依序為 B, G, R)

# 取得特定通道數值
b_val = img[100, 50, 0]  # 取得 Blue 數值 (索引為 0)
g_val = img[100, 50, 1]  # 取得 Green 數值 (索引為 1)
r_val = img[100, 50, 2]  # 取得 Red 數值 (索引為 2)

# 修改特定像素的顏色 (例如將該點設為純紅色)
img[100, 50] = [0, 0, 255]
```

---

### 3.2 影像裁剪 (ROI - Region of Interest)
在影像處理中，常需擷取特定區域（感興趣區域）。我們可以使用 NumPy 的二維切片來完成：
```python
# 語法：img[y_start:y_end, x_start:x_end]
# 擷取 y軸 (row) 100~400，x軸 (col) 150~415 的影像區域
roi_cut = img[100:400, 150:415]
```

---

### 3.3 色彩通道分割與合併

#### 使用 OpenCV 內建函式
```python
# 分割通道
b, g, r = cv2.split(img)

# 合併通道
img_merged = cv2.merge([b, g, r])
```

#### 使用 NumPy 切片進行分割（**推薦，效率更高**）
直接透過陣列索引切片來獲取單一通道，執行速度比 `cv2.split` 更快：
```python
b = img[:, :, 0]  # 取得所有像素的 Blue 通道
g = img[:, :, 1]  # 取得所有像素的 Green 通道
r = img[:, :, 2]  # 取得所有像素的 Red 通道
```

## 4. 透明度通道 (Alpha Channel / BGRA)

除了 BGR 三個色彩通道外，有些影像格式（如 PNG）還包含第四個通道：**Alpha 通道（透明度）**。
- 數值範圍：`0`（完全透明）至 `255`（完全不透明）。
- 載入時若要保留透明度通道，必須將 `cv2.imread` 的 flags 設為 `-1` (`cv2.IMREAD_UNCHANGED`)。
```python
# 讀取帶有透明通道的 PNG 圖片
img_bgra = cv2.imread('./image/logo.png', cv2.IMREAD_UNCHANGED)
print(img_bgra.shape)  # 輸出為 (Height, Width, 4)
```
