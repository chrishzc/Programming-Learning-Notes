# OpenCV 局部特徵擷取與匹配

> [!ABSTRACT]
> 本篇筆記介紹電腦視覺中的局部特徵（Local Features）辨識技術。內容涵蓋關鍵點檢測（Keypoint Detection）、特徵描述子（Descriptor）生成演算法（FAST, SIFT, ORB），以及特徵比對匹配器（BFMatcher, FLANN）與幾何過濾技術（Ratio Test, RANSAC）。

---

## 1. 局部特徵擷取原理

要讓電腦辨識影像中的特定物件，關鍵在於掌握其局部幾何特徵，而非比對整張影像。其標準流程為：
1. **關鍵點檢測 (Keypoint Detection)**：在影像中尋找具有代表性、容易定位的特殊點（如角點 Corners、邊緣 Edges 或斑點 Blobs）。
2. **特徵描述子生成 (Feature Extraction)**：針對每個關鍵點周邊的局部區域進行數學統計，生成一個能代表該區域花紋、且對環境變化不敏感的特徵向量（Descriptor）。
3. **特徵匹配 (Feature Matching)**：藉由比對兩張影像中描述子之間的距離（歐氏距離或漢明距離），找出對應的特徵點連線。

---

## 2. 特徵檢測與描述子演算法

OpenCV 提供了多種特徵點檢測與描述子提取演算法，其效能與適用場景各不相同。

### 2.1 FAST 演算法 (角點快篩)
- **物理邏輯**：基於加速分割測試（Features from Accelerated Segment Test）。以像素 $p$ 為中心畫圓（半徑為 3，圓周共 16 個像素），若圓周上有連續 $n$ 個像素的亮度顯著大於（或小於）中心點 $p$（加上閾值門檻），則 $p$ 視為角點。
- **快篩加速**：優先測試圓周上每隔 $90^{\circ}$ 的四個點（點 1, 9, 5, 13）。若其中至少有三個點滿足閾值條件，才對剩餘點進行完整測試；否則直接剔除。
- **非極大值抑制 (NMS)**：避免角點在影像邊緣「扎堆」（過度集中），在局部區域內僅保留響應值最大的角點。
- **特點**：**速度極快**，適用於即時性（Real-time）視訊分析或效能較低的嵌入式硬體。

#### 程式範例：
```python
import cv2

img = cv2.imread('input.jpg', cv2.IMREAD_GRAYSCALE)

# 建立 FAST 檢測器
fast = cv2.FastFeatureDetector_create(threshold=20, nonmaxSuppression=True)
# 僅偵測關鍵點 (FAST 沒有描述子計算功能)
keypoints = fast.detect(img, None)

# 繪製關鍵點
img_fast = cv2.drawKeypoints(img, keypoints, None, color=(0, 255, 0))
```

---

### 2.2 SIFT 演算法 (尺度與旋轉不變特徵變換)
- **物理邏輯**：
  1. **高斯差分金字塔 (DoG)**：將原圖以不同尺度進行高斯模糊並縮小，相鄰圖層相減得到高斯差分圖，保留影像的多尺度邊緣細節。
  2. **關鍵點定位**：候選點必須與當前圖層周圍 8 個像素，以及上下鄰近圖層共 18 個像素（共計 26 個比對點）相比。只有絕對的極值點才會晉級，並剔除不穩定的邊緣點。
  3. **分配方向 (旋轉不變性)**：統計關鍵點周圍的梯度走向，為其分配一個主方向（插上指南針）。當比對時將方向轉回正北即可實現角度對齊。
  4. **特徵描述子 (128維向量)**：以關鍵點為中心框出 $16 \times 16$ 區域，分割為 $4 \times 4$ 的 16 個小方塊。統計每個小方塊中 8 個梯度方向的激烈度，最終形成 $16 \times 8 = 128$ 維的浮點數向量。
- **特點**：精確度極高，對影像**旋轉、尺度縮放、亮度變化**等具有極佳的不變性，但計算開銷較大。

#### 程式範例：
```python
# 建立 SIFT 偵測器
sift = cv2.SIFT_create()

# 同時偵測關鍵點與計算 128 維描述子
keypoints, descriptors = sift.detectAndCompute(img, None)

# 繪製帶有大小與方向的富特徵點
img_sift = cv2.drawKeypoints(img, keypoints, None, flags=cv2.DRAW_MATCHES_FLAGS_DRAW_RICH_KEYPOINTS)
```

---

### 2.3 ORB 演算法 (Oriented FAST and Rotated BRIEF)
- **物理邏輯**：
  1. **FAST 檢測關鍵點**：進行地毯式快篩，並引入影像金字塔解決尺度缩放問題。
  2. **質心方向魔改 (Oriented)**：計算關鍵點鄰域內「幾何中心」與「亮度重心」的偏移向量，作為關鍵點的主方向，賦予其旋轉不變性。
  3. **Rotated BRIEF 描述子 (二進位密碼)**：在關鍵點周圍隨機挑選 256 對像素點。比對每對像素的明暗關係（亮為 1，暗為 0），形成 256 位元（32 字節）的超輕量二進位陣列。在比對前會配合主方向進行角度旋轉對齊。
- **特點**：**執行效率極佳**，且無版權限制。是 SIFT 的優秀替代方案，特別適合點雲重構與即時 SLAM 定位。

#### 程式範例：
```python
# 建立 ORB 偵測器，限制最大偵測特徵點數為 500
orb = cv2.ORB_create(nfeatures=500)
# 偵測關鍵點並計算 32 字節的二進位描述子
keypoints, descriptors = orb.detectAndCompute(img, None)
```

## 3. 特徵匹配技術 (Feature Matching)

獲得兩張影像的描述子（`descriptors`）後，需通過比對找出最相似的特徵點對。

### 3.1 匹配器類型對照表
| 匹配技術 | 運作邏輯 | 距離度量方式 | 適用戰場與核心搭配 |
| :--- | :--- | :--- | :--- |
| **BFMatcher**<br>(暴力匹配) | **地毯式搜尋**。將圖 A 的每一個描述子與圖 B 的所有描述子計算距離，找出距離最近者。 | **漢明距離 (`NORM_HAMMING`)**：用於比對 ORB 二進位描述子。<br>**歐氏距離 (`NORM_L2`)**：用於 SIFT。 | 搭配 **ORB 描述子**。<br>特徵點數量較少、要求底層硬體加速時。 |
| **FLANN Matcher**<br>(快速最近鄰) | **樹狀結構快速搜尋**。使用優化後的隨機 KD 樹（Randomized KD-Tree）演算法進行高維空間快速近似最近鄰搜尋。 | 預設為**歐氏距離**。 | 搭配 **SIFT 描述子**。<br>高精度要求、巨量特徵點（大數據集）大戰。 |

---

### 3.2 幾何過濾技術 (篩選真愛)
- **比例測試 (Ratio Test)**：
  - 在匹配時尋找距離最近的前兩名。若 $\frac{\text{第一名距離}}{\text{第二名距離}} < 0.75$（即第一名顯著優於第二名），則採信該匹配；否則捨棄。這能有效過濾掉因為重複紋理（如棋盤格或大片素色）產生的誤判連線。
- **隨機抽樣一致 (RANSAC)**：
  - 從匹配對中隨機抽樣，計算兩張影像間的**單應性矩陣（Homography Matrix）**。尋求符合絕大多數特徵點空間移動規律的矩陣，並徹底斬斷跨圖亂切、方向不合群的雜訊叛徒線。

---

## 4. 實戰範例：ORB 特徵檢測與暴力匹配

以下程式碼實作 ORB 局部特徵提取，並使用 BFMatcher 漢明距離搜尋兩張影像的前 50 個最佳對應點，最後繪製連線：

```python
import cv2

# 1. 載入兩張待比對影像 (例如同一物體不同角度拍攝的影像)
img1 = cv2.imread('query.jpg', cv2.IMREAD_GRAYSCALE)
img2 = cv2.imread('train.jpg', cv2.IMREAD_GRAYSCALE)

# 2. 建立 ORB 特徵偵測器
orb = cv2.ORB_create(nfeatures=1000)

# 3. 偵測關鍵點與計算描述子
kp1, des1 = orb.detectAndCompute(img1, None)
kp2, des2 = orb.detectAndCompute(img2, None)

# 4. 建立 BFMatcher (暴力匹配器)
# 對於 ORB 二進位描述子，必須指定 NORM_HAMMING 漢明距離
# crossCheck=True 啟用雙向交叉檢查，只有雙向皆為最近鄰的匹配對才會保留
bf = cv2.BFMatcher(cv2.NORM_HAMMING, crossCheck=True)

# 5. 執行特徵點比對
matches = bf.match(des1, des2)

# 6. 將匹配結果依距離排序 (歐氏/漢明距離越小，代表特徵越相似)
matches = sorted(matches, key=lambda x: x.distance)

# 7. 繪製前 50 個最優匹配點的連線
matched_img = cv2.drawMatches(
    img1, kp1, 
    img2, kp2, 
    matches[:50], 
    None, 
    flags=cv2.DrawMatchesFlags_NOT_DRAW_SINGLE_POINTS
)

# 8. 顯示匹配成果
cv2.imshow('ORB BF Feature Matching', matched_img)
cv2.waitKey(0)
cv2.destroyAllWindows()
```
