# 16. 進階影像分割與去背

> [!ABSTRACT]
> 本章介紹 OpenCV 中的進階影像分割與去背技術。影像分割旨在將目標主體從背景中分離。本章將探討自動化閾值選擇的 Otsu 演算法、解決物體粘連問題的分水嶺演算法（Watershed），以及用於高畫質前景去背的 GrabCut 演算法，並提供實用的 GrabCut 去背實戰程式碼。

---

## 一、 Otsu 自動閾值選擇 (Otsu's Binarization)

在影像二值化（Binarization）中，傳統的 `cv2.threshold` 需要手動指定一個閾值（Threshold）。然而，當光源或對比度改變時，固定閾值容易失效。

- **Otsu 演算法原理**：
  Otsu 是一種自動計算影像最佳閾值的方法。它適用於直方圖呈現**雙峰分佈**（即前景與背景灰階值有明顯差異）的影像。其運作方式是遍歷所有可能閾值，尋找一個能使**類內方差最小**（或類間方差最大）的灰階值作為閾值。
- **程式碼實現**：
  在呼叫時將閾值設為 `0`，並在方法參數中疊加 `cv2.THRESH_OTSU`：
  ```python
  # gray 為輸入灰階影像
  ret, thresh_img = cv2.threshold(gray, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)
  ```

## 二、 分水嶺演算法 (Watershed)

當影像中的多個目標物體彼此**粘連、接觸**時（例如桌上散落且靠在一起的硬幣），簡單的二值化與輪廓尋找會將它們誤判為同一個物件。分水嶺演算法能有效解決此問題。

- **運作原理**：
  將影像視為地形學的拓撲地圖，灰階值代表高度。演算法從使用者指定的「種子點」（高地或盆地）開始注入水流。當不同盆地流出的水相遇時，便在此處修築「堤壩」（即分割線），從而將粘連的物體分開。
- **核心步驟**：
  1. 對影像進行二值化，並利用形態學開運算（Opening）消除噪聲。
  2. 利用 **距離變換 (Distance Transform)** `cv2.distanceTransform()` 找出確定是前景的區域。
  3. 利用膨脹運算（Dilation）找出確定是背景的區域。
  4. 背景減去前景，得到「未知區域」（即物體的交界邊緣）。
  5. 建立標籤矩陣（Markers），並調用 `cv2.watershed()` 進行精確分割。

## 三、 GrabCut 前景去背演算法

GrabCut 是一種基於圖割（Graph Cut）的互動式前景提取演算法，極常被用來進行人像或商品去背。

- **運作模式**：
  使用者只需在影像中拉出一個**矩形框**，將要保留的主體框住。演算法會將框外的區域直接標記為「確定背景」，並在框內自動學習前景與背景的色彩分佈模型（使用高斯混合模型 GMM），透過多次迭代計算，最終將前景邊緣精確裁剪出來。
- **核心 API**：
  ```python
  mask, bgdModel, fgdModel = cv2.grabCut(img, mask, rect, bgdModel, fgdModel, iterCount, mode)
  ```
  - **`img`**：輸入的彩色影像。
  - **`mask`**：遮罩矩陣，大小與影像相同。用於標記四種狀態：確定背景 (`0`)、確定前景 (`1`)、可能背景 (`2`)、可能前景 (`3`)。
  - **`rect`**：包含前景目標的矩形框，格式為 `(x, y, w, h)`。
  - **`bgdModel` / `fgdModel`**：演算法內部使用的暫存陣列（大小必須是 $1 \times 65$ 的 `np.float64`）。
  - **`iterCount`**：迭代計算的次數。通常設定為 `5` 即可獲得良好效果。
  - **`mode`**：模式。初次計算使用 `cv2.GC_INIT_WITH_RECT`。

## 四、 實戰演練：GrabCut 去背實作

以下程式碼示範如何使用 GrabCut 自動提取框選區域內的前景，並將背景去除（填充為黑色）：

```python
import cv2
import numpy as np

# 1. 讀取彩色影像
img = cv2.imread("product.jpg")
h, w = img.shape[:2]

# 2. 建立一個大小與影像相同的遮罩 (初始化為確定背景)
mask = np.zeros((h, w), np.uint8)

# 3. 定義背景與前景的暫存模型
bgdModel = np.zeros((1, 65), np.float64)
fgdModel = np.zeros((1, 65), np.float64)

# 4. 定義包含主體目標的矩形框 (x, y, 寬度, 高度)
# 請根據您的圖片內容動態調整此座標
rect = (50, 50, w - 100, h - 100)

# 5. 執行 GrabCut 演算法
cv2.grabCut(img, mask, rect, bgdModel, fgdModel, iterCount=5, mode=cv2.GC_INIT_WITH_RECT)

# 6. 將遮罩中「確定前景」與「可能前景」的像素過濾出來
# 0 & 2 變為 0 (背景)，1 & 3 變為 1 (前景)
output_mask = np.where((mask == 2) | (mask == 0), 0, 1).astype("uint8")

# 7. 將原始影像與遮罩進行「位元與」運算 (去背)
result = cv2.bitwise_and(img, img, mask=output_mask)

# 8. 顯示結果
cv2.imshow("Original", img)
cv2.imshow("Cutout Result", result)
cv2.waitKey(0)
cv2.destroyAllWindows()
```
