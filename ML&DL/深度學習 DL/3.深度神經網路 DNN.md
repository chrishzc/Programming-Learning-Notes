## 一、 什麼是深度神經網路 (DNN)？

**深度神經網路 (Deep Neural Network, DNN)** 指的是一種具有多層隱藏層的類神經網路結構。深度學習能透過層層網路節點的非線性轉換，將大量無序的輸入訊號漸進式地轉換為有用的高階抽象特徵，以解決複雜的分類或迴歸問題。

- **結構認定**：在實務上，一個神經網路只要擁有至少 **2 層隱藏層**，加上輸入層與輸出層（總共 4 層或以上），即可被稱為深度神經網路。

---

## 二、 手動以張量建立 DNN

想要以低階張量（Tensor）方式手動設計一個多層的全連接神經網路（如圖下），必須為每一層獨立定義權重矩陣 $W$ 與偏差值向量 $b$。

> [!WARNING]
> **維度匹配與獨立性**
> 1. 有多少個全連接層，就需要定義相對應數量的 $W$ 與 $b$，且每層的參數只能在當前層使用，不可混淆。
> 2. **前向傳播公式**：第 $l$ 層的計算為：$O_l = X_l \times W_l + b_l$，接著通過激活函數：$S_l = \sigma(O_l)$。前一層的輸出 $S_l$ 會直接作為下一層的輸入 $X_{l+1}$。

### 3 維特徵矩陣的前向傳播實作
以下範例示範如何輸入一個形狀為 `[3, 784]` 的特徵矩陣（表示 3 筆 784 維度的樣本），通過包含 3 個隱藏層的 DNN，最後輸出為 `[3, 10]` 的矩陣：
```python
import tensorflow as tf

# 模擬輸入資料：3筆樣本，每筆有 784 個特徵
x = tf.random.normal([3, 784])    

# 隱藏層 1：輸入 784 -> 輸出 256
w1 = tf.Variable(tf.random.truncated_normal([784, 256], stddev=0.1))
b1 = tf.Variable(tf.zeros([256]))

# 隱藏層 2：輸入 256 -> 輸出 128
w2 = tf.Variable(tf.random.truncated_normal([256, 128], stddev=0.1))
b2 = tf.Variable(tf.zeros([128]))

# 隱藏層 3：輸入 128 -> 輸出 64
w3 = tf.Variable(tf.random.truncated_normal([128, 64], stddev=0.1))
b3 = tf.Variable(tf.zeros([64]))

# 輸出層：輸入 64 -> 輸出 10 (例如 10 分類任務)
w4 = tf.Variable(tf.random.truncated_normal([64, 10], stddev=0.1))
b4 = tf.Variable(tf.zeros([10]))

# 手動進行前向傳播運算
o1 = x @ w1 + b1
s1 = tf.nn.sigmoid(o1)

o2 = s1 @ w2 + b2
s2 = tf.nn.sigmoid(o2)

o3 = s2 @ w3 + b3
s3 = tf.nn.sigmoid(o3)

o4 = s3 @ w4 + b4
print("輸出矩陣形狀:", o4.shape)  # 輸出: (3, 10)
```

## 三、 Keras 快速建立網路模型的三種 API

TensorFlow 2.0 整合了高階 Keras API，提供了三種快速搭建全連接網路（Dense）的方法。開發者可依據模型複雜度選擇最合適的方法：

| API 類型 | 靈活度 | 難易度 | 特點說明 | 適用場景 |
| :--- | :---: | :---: | :--- | :--- |
| **Sequential API** | 低 | 極易 | 像積木一樣單向堆疊圖層，不支援多輸入、多輸出或跳躍連接。 | 快速建立標準、簡單的線性堆疊模型。 |
| **Functional API** | 中 | 易 | 以函數呼叫方式連接各圖層，支援非線性拓撲（如殘差網路、分支結構）。 | 適用於多數複雜、多輸入多輸出的應用模型。 |
| **Model Subclassing** | 高 | 難 | 繼承 `tf.keras.Model` 類別，在 `call` 中定義動態控制流。 | 適用於需要高度自訂研究、動態計算流程的科學研究。 |

### 1. Functional API 全連接神經網路
Functional API 以層接層（函數鏈式呼叫）的方式設計。利用 `layers.Dense()` 定義網路層，並指定其激活函數類型，再將輸入資料依序傳遞。
```python
import tensorflow as tf
from tensorflow.keras import layers  # 導入 layer 類別

# 1. 建立各網路層實例
fc1 = layers.Dense(256, activation=tf.sigmoid)  # 隱藏層 1
fc2 = layers.Dense(128, activation=tf.sigmoid)  # 隱藏層 2
fc3 = layers.Dense(64, activation=tf.sigmoid)   # 隱藏層 3
fc4 = layers.Dense(10, activation=None)          # 輸出層

# 2. 通過鏈式呼叫將資料傳遞過去
X = tf.random.normal([3, 784])
h1 = fc1(X)
h2 = fc2(h1)
h3 = fc3(h2)
out = fc4(h3)
print("Functional API 輸出形狀:", out.shape)  # 輸出: (3, 10)
```

### 2. Sequential API 網路容器
透過 `Sequential` 容器，可以將多個獨立網路層打包成一個整體的網路模組。資料會自動從首層傳播至最末層，代碼極為簡潔。

![[{8B525125-C935-4323-9EAD-0116383BEDB2}.png|360]]

#### 寫法一：直接在初始化時宣告圖層
```python
import tensorflow as tf
from tensorflow.keras import layers, Sequential

x = tf.random.normal([3, 784])

# 將多個層封裝進 Sequential 容器
model = Sequential([
    layers.Dense(256, activation=tf.nn.relu),
    layers.Dense(128, activation=tf.nn.relu),
    layers.Dense(64, activation=tf.nn.relu),
    layers.Dense(10, activation=None),
])    

out = model(x)
print("Sequential 輸出形狀:", out.shape)
```

#### 寫法二：使用 `add()` 方法動態加入圖層
`Sequential` 容器支援動態構建，可以在創立空容器後，依序使用 `add()` 添加新網路層，並使用 `summary()` 檢視模型架構。
```python
import tensorflow as tf
from tensorflow.keras import layers, Sequential

x = tf.random.normal([3, 784])

model = Sequential()  # 創建一個空的網路容器
model.add(layers.Dense(256, activation=tf.nn.relu))  # 加入隱藏層 1
model.add(layers.Dense(128, activation=tf.nn.relu))  # 加入隱藏層 2
model.add(layers.Dense(64, activation=tf.nn.relu))   # 加入隱藏層 3
model.add(layers.Dense(10, activation=None))         # 加入輸出層

out = model(x)
print(model.summary())  # 輸出模型各層的參數與架構狀況
```

### 3. Model Subclassing (模型子類化)
最靈活的高級方法。透過撰寫一個自訂類別並**繼承 `tf.keras.Model`**。在建構子 `__init__()` 中宣告網路子層，並在 `call()` 方法中定義前向傳播的拓撲順序。
```python
import tensorflow as tf
from tensorflow.keras import layers

class netModel(tf.keras.Model):
    def __init__(self):
        super().__init__()
        # 在建構子中宣告所有需要的網路層實例
        self.fc1 = layers.Dense(256, activation=tf.nn.relu)
        self.fc2 = layers.Dense(128, activation=tf.nn.relu)
        self.fc3 = layers.Dense(64, activation=tf.nn.relu)
        self.fc4 = layers.Dense(10)
        
    def call(self, inputs, training=None, mask=None):
        # 撰寫網路運算的資料傳遞順序，支援複雜的 Python 邏輯控制
        x = self.fc1(inputs)
        x = self.fc2(x)
        x = self.fc3(x)
        out = self.fc4(x)
        return out
```

利用 Model 繼承方式建立主網路模型架構後，接著要**實體化網路**。
