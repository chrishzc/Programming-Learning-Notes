# 8. CNN 卷積神經網路介紹

> [!ABSTRACT]
> 本章介紹卷積神經網路（CNN）的基礎概念。隨著網路層數加深，傳統全連接網路面臨參數量爆炸的挑戰。為解決此問題，CNN 引入了局部感受野、權值共享與池化三大核心思想，並搭配空洞卷積、展平層與全連接層等結構，以高效率提取影像的局部特徵與全局資訊。

---

## 一、 全連接網路的局限與挑戰

隨著神經網路的層數加深，網路的參數量會呈現爆炸性提升。
- **權重計算問題**：輸入影像越大，權重數就越高。
- **計算負載過大**：
  > [!WARNING]
  > 如果遇到更大或高解析度的影像（例如 $1024 \times 1024$ 或更高），且網路設計較深時，傳統全連接網路的參數可能到達數百億個。這樣的計算量過於龐大，在實務上非常不切實際。

---

## 二、 卷積神經網路 (CNN) 基礎概念

卷積神經網路（Convolutional Neural Network, CNN）是模仿人類大腦認知方式的一種學習方法。
- **感知模式**：先從局部區域特徵（區域關聯性）進行感知，然後在更高層的地方將局部訊息綜合起來，得到全局訊息，進而完成影像辨識。

在 TensorFlow 中，可以通過 `tf.nn.conv2d()` 函數方便地實現 2D 卷積運算：

- **輸入格式 (Input Format)**：
  `[batch, in_height, in_width, in_channels]` （分別代表：張數、長、寬、輸入通道數/顏色結果）
- **卷積核維度格式 (Filter Format)**：
  `[filter_height, filter_width, in_channels, out_channels]` （分別代表：長、寬、輸入通道數、輸出通道數）

#### 邊緣填充 (Padding) 參數設定
`padding` 參數主要有兩種設定方法：
1. **`padding='VALID'`**：不進行邊緣填充。產生的特徵圖（Feature Map）會受到 Filter 大小與步長（Stride）影響而變小。
2. **`padding='SAME'`**：進行邊緣填充。輸出的特徵圖尺寸會與輸入圖尺寸保持一致（當步長為 1 時）。

## 三、 CNN 降維與減少參數量之三大支柱

為了解決全連接網路參數量爆炸的問題，CNN 提出了三個基本想法：局部感受野、權值共享與池化。

### 1. 局部感受野 (Local Receptive Fields)
- 感受野（Receptive Field）在實務上即為 OpenCV 中學過的 **Kernel（卷積核）**，模型每次只關注影像中的一個局部區域。

### 2. 權值共享 (Shared Weights)
- 整張圖在同一個卷積特徵提取過程中，共享相同的權值。
- **整張圖的權值大小 = Kernel 大小**，大幅減少了需要訓練的參數個數。
- **填充 (Padding)**：當影像經過卷積運算後，得到的特徵圖尺寸會變小，因此需要進行邊緣填充以保留邊界特徵。

### 3. 池化層 (Pooling Layer)
池化層利用滑動視窗內的數值進行數值運算，主要用於對特徵圖進行降維。

> [!IMPORTANT]
> **池化層的核心功用**
> - 幫助縮小 Feature Map 的大小（即降維或稱降採樣），從而提高運算速度。
> - 減小雜訊（Noise）的影響，降低過擬合（Overfitting）風險。
> - 讓提取出來的特徵更具有健壯性（Robustness）。
> - **池化層本身沒有權重**。

#### 池化的兩種主要運算方式
- **Max pooling (最大池化)**：在框選的局部數值中挑出最大值。
- **Average pooling (平均池化)**：將框選的局部數值加總做平均計算。

```python
# TensorFlow 低階 API
tf.nn.max_pool()
tf.nn.avg_pool()
```

```python
# TensorFlow Keras 高階 API (在 tensorflow.keras.layers 模組中)
keras.layers.MaxPooling2D(pool_size=(2, 2), strides=None, padding='valid', data_format=None) 

keras.layers.AveragePooling2D(pool_size=(2, 2), strides=None, padding='valid', data_format=None)
```

> [!NOTE]
> 隨著現代網路架構的演進（如使用步長大於 1 的卷積層代替池化），在一些新型網路中，傳統池化層的使用頻率已有逐漸減少的趨勢。

## 四、 其他重要網路層與卷積技術

### 1. 空洞卷積 (Dilated Convolution / Atrous Convolution)
- **概念**：在卷積核的元素之間插入「空洞」（Dilation Rate），從而在不增加參數量的狀況下增大感受野。
- **優勢**：在影像檢測與分割任務中，對大尺寸物體能表現出較好的效果。

![[{359666DA-9484-49C3-BFE0-D016458F4419}.png]]

### 2. 展平層 (Flatten)
- **概念**：用於銜接卷積/池化層與全連接層（FC）。因為全連接層需要一維的向量作為輸入，展平層會將多維的特徵圖拉平為一維向量。
- **特點**：**展平層沒有權重**。

![[{02442B74-CB13-48C9-B7F5-9FE0821C133B}.png|400]]
*(範例中原本 $5 \times 5 \times 3$ 的特徵圖展平後變成長度為 75 的一維向量)*

### 3. 全連接層 (Fully Connected Layer, FC)
- **特徵整合**：全連接本質上是矩陣乘法，相當於一個特徵空間變換，可以把前面所有有用的局部資訊提取並整合。
- **非線性映射**：配合激活函數的非線性映射，多層全連接層理論上可以模擬任何非線性變換。
- **局限**：無法保持空間結構，且會透過學習全部的權重來整合「好」的特徵並減少其他特徵，參數量較大。

---

## 五、 卷積層與池化層的異同對比

- **通道（Channel）處理**：
  - **池化層**：池化前有幾個通道，池化後還是幾個通道（不改變通道數）。
  - **卷積層**：下一層的通道數由該層的卷積核數量（Filter 數量）決定，即上一層的特徵圖個數會與卷積核的通道數對齊，而輸出通道數與卷積核個數一致。
- **參數與核（Kernel/Filter）特性**：
  - 卷積對應有卷積核，池化對應有池化核。
  - **卷積核**內部含有待學習的參數（權重與偏差）。
  - **池化核**只是一個運算框架，裡面**沒有任何待學習的參數**。
- **超參數設定**：
  - 兩者在使用時都需要定義核心大小（Size）、步長（Stride）以及填充類型（Padding）。
  - 一般在卷積層中較常使用 `padding='SAME'`；而在池化層中則較常使用 `padding='VALID'`。
