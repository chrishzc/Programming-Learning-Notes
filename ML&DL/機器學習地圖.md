# 機器學習地圖

> [!ABSTRACT]
> 本篇筆記介紹人工智慧（AI）、機器學習（ML）與深度學習（DL）的產業現況、核心資料處理流程（Pipeline），並說明幾種主流機器學習演算法（SVM、決策樹、隨機森林）與硬體算力限制。

---

## 一、 AI 產業現況

目前 AI 領域研發與硬體算力（GPU/TPU）建置成本極高，多數中小企業通常透過以下方式切入：
- **租借雲端算力**：向主流雲端廠商租用算力來研發與部署 AI 應用。
- **接入大語言模型 (LLM)**：直接串接成熟的 LLM API (如 OpenAI, Gemini)，快速為既有產品增強競爭力。

---

## 二、 資料處理與機器學習流程

數據在套入模型前，需要經歷完整的 Pipeline 處理：

```
[資料預處理 / 數據探勘] ──> [資料轉換] ──> [特徵工程]
                                           │
         ┌─────────────────────────────────┘
         ├─> [機器學習] ──> 選擇演算法 (SVM、決策樹、隨機森林...)
         └─> [深度學習] ──> 選擇模型 (CNN, RNN, Transformer...)
```

> [!NOTE]
> **Epoch (訓練回合)**
> 一個 Epoch 代表完整的訓練資料集被送入神經網路中訓練完「一個回合」。

## 三、 機器學習 (Machine Learning)

機器學習又稱「淺層學習」，其特點是開發者先建立基礎模型，讓演算法透過訓練資料自主歸納規則，進而微調與優化程式。
- **學科基礎**：建立在資料科學、統計學、機率論與程式設計之上。
- **開發限制**：對訓練資料量的需求相對較少，但需要依賴領域專家進行手動的**特徵工程 (Feature Engineering)** 挑選關鍵特徵。

### 3.1 主流演算法
1. **支持向量機 (SVM, Support Vector Machine)**
   - **原理**：尋找一個最佳分割超平面（Hyperplane），使該平面距離兩側最近的資料點（支持向量）之距離（Margin）最大化。
   - **優點**：非常適合處理小樣本、高維度的資料（如基因序列、影像特徵）。
   - **缺點**：當資料量極大（數百萬筆以上）時，計算速度會變得很慢。
2. **決策樹 (Decision Tree)**
   - **原理**：使用二分樹結構，依據資料特徵逐步分流，進行分類或回歸預測。
   - **缺點**：結構較簡單，容易過擬合（Overfitting），不適合直接套用在超大型或高複雜度數據中。
3. **隨機森林 (Random Forest)**
   - **原理**：整合學習（Ensemble Learning）代表。隨機抽取樣本與特徵，建立成百上千棵各不相同的決策樹，最後以「投票法（少數服從多數）」決定預測結果。
   - **優點**：模型穩定性與抗噪能力極高，不易出錯。
   - **缺點**：比單一決策樹複雜，運算量與記憶體消耗較大。

### 3.2 機器學習類型
- **監督式學習 (Supervised Learning)**：使用包含正確標籤（Label）的資料進行訓練與預測。
- **非監督式學習 (Unsupervised Learning)**：對無標籤的資料進行分析，自動尋找其分群（Clustering）規律或降維。
- **強化學習 (Reinforcement Learning / 獎勵式學習)**：讓 Agent 藉由與環境互動，採取行動以獲取最大化累積獎勵分數（Reward），藉此學習最佳策略。

## 四、 深度學習 (Deep Learning)

深度學習建立在多層（通常為三層以上）的**類神經網路 (Artificial Neural Networks)** 基礎之上。
- **優點**：對極大數據集的表徵能力極佳，**不需要人工挑選特徵**，模型能自動學習高階特徵。
- **限制與瓶頸**：
  1. **硬體與算力限制**：高度依賴 GPU 進行並行運算（如 NVIDIA GPU 及 CUDA 運算庫），或是專為張量設計的 TPU (Tensor Processing Unit)。訓練千億級大模型的成本可高達數百萬至上億台幣。
  2. **高維度災難**：需要極大數量的訓練資料，並需適度進行降維或規範化，避免參數過多導致模型失真或算力需求爆炸。

---

## 五、 Windows 系統運算限制

- **記憶體鎖定限制**：在 Windows 上執行 CPU 密集運算時，預設會限制單個進程的 CPU 記憶體容量（通常是 2~5 GB）。在安裝 Python 時，可以在最後一步勾選「**Disable path length limit**」及調整系統限制以突破封鎖。
- **顯示記憶體 (VRAM) 限制**：若 GPU VRAM 空間不足，必須在程式中採用**分批訓練 (Batch Training / Mini-batch)** 方式，減少單次載入顯示卡的資料量。
