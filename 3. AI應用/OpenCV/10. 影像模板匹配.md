# OpenCV 影像模板匹配 (Template Matching)

> [!ABSTRACT]
> 本篇筆記介紹模板匹配的基本概念、六大匹配方法之數學邏輯對比，以及如何使用 OpenCV 進行影像目標搜尋、定位與標記的完整實作。

---

## 1. 模板匹配基本原理

模板匹配（Template Matching）是一種基礎的模式識別方法。其運作邏輯如下：
1. **設定模板**：準備一張已知的小影像（模板 Template，$w \times h$）。
2. **滑動比對**：將模板像滑動窗口一樣，在大影像（Source，$W \times H$）上從左到右、從上到下逐像素滑動。
3. **計算得分**：在每個滑動位置，計算模板與重疊影像區域的相似度（或差異度）。
4. **輸出地圖**：滑動完成後會產生一張大小為 $(W - w + 1) \times (H - h + 1)$ 的單通道浮點數得分地圖（Result Map）。
5. **尋找極值**：搜尋得分地圖中的極大值（或極小值）位置，該座標即代表最佳匹配目標的左上角點。

---

## 2. 匹配方法對照表

在執行匹配時，可選擇以下六種比對演算法：

| 模式常數 (method) | 尋找目標極值 | 分數區間 | 物理畫面的底層邏輯與特點 | 實戰點評與推薦度 |
| :--- | :---: | :---: | :--- | :--- |
| **`cv2.TM_SQDIFF`** | **`minVal`** (越小越像) | $0 \sim \infty$ | **平方差匹配**：計算兩者像素相減的絕對平方距離。完全相同時值為 `0`。 | ❌ **不推薦。** 極易受到整體暗色區域（黑洞）的虛報干擾。 |
| **`cv2.TM_SQDIFF_NORMED`** | **`minVal`** (越小越像) | $0 \sim 1$ | **標準平方差匹配**：平方差的歸一化版本，將相似度限制在 `0 ~ 1`。 | ⚠️ **普通。** 解決了暗區干擾，適合光源穩定、背景單純的場景。 |
| **`cv2.TM_CCORR`** | **`maxVal`** (越大越像) | $0 \sim \infty$ | **相關匹配**：兩者對應像素直接相乘。模板與影像亮點重疊時數值極大。 | ❌ **不推薦。** 偏好高亮度，若背景有強光反光，會產生嚴重誤判。 |
| **`cv2.TM_CCORR_NORMED`** | **`maxVal`** (越大越像) | $0 \sim 1$ | **標準相關匹配**：相關匹配的歸一化版本，限制得分在 `0 ~ 1`。 | ⚠️ **普通。** 雖壓制了部分強光，但對紋理起伏變化依然不夠敏感。 |
| **`cv2.TM_CCOEFF`** | **`maxVal`** (越大越像) | $-\infty \sim \infty$ | **相關係數匹配**：比對前先扣除兩張圖的「平均亮度」，以排除光影起伏。 | 👍 **好用。** 因為扣除了平均值，能有效免疫大片純黑或純白的環境光影變化。 |
| **`cv2.TM_CCOEFF_NORMED`** | **`maxVal`** (越大越像) | $-1 \sim 1$ | **標準相關係數匹配**：終極完美形態。將扣除平均值後的比對進行歸一化。 | ⭐️ **首選王牌！** 業界精準度最高、防干擾能力最強。分數明確便於調校門檻。 |

## 3. 模板匹配核心 API

### 3.1 模板比對 (`cv2.matchTemplate`)
```python
res = cv2.matchTemplate(image, templ, method)
```
- `image`：大圖來源（8位元或浮點數，彩色或灰階皆可）。
- `templ`：搜尋模板（尺寸不可大於大圖，且通道數與大圖必須一致）。
- `method`：匹配方法常數（如 `cv2.TM_CCOEFF_NORMED`）。
- `res`：回傳的得分影像（32-bit 單精度浮點數，大小為大圖與模板的寬高差值）。

### 3.2 尋找最值位置 (`cv2.minMaxLoc`)
在大圖中定位目標的左上角點，必須對 `res` 地圖使用 `minMaxLoc`。
```python
min_val, max_val, min_loc, max_loc = cv2.minMaxLoc(res)
```
> [!IMPORTANT]
> **座標選取規則：**
> - 若使用 **`cv2.TM_SQDIFF`** 或 **`cv2.TM_SQDIFF_NORMED`**，最佳匹配位置為 **`min_loc`**。
> - 若使用**其他四種**相關性匹配方法，最佳匹配位置為 **`max_loc`**。

## 4. 模板匹配完整實作範例

以下程式碼使用 NumPy 自建測試畫布與圖案，演示如何偵測目標並繪製綠色外接框：

```python
import cv2
import numpy as np

# 1. 建立測試彩色畫布 (黑色背景)
scene = np.zeros((400, 400), dtype=np.uint8)
# 在特定位置 (y=200, x=200) 畫一個實心白圓作為我們的目標物 (半徑 30)
cv2.circle(scene, (200, 200), 30, 255, -1)
# 畫上一些不同尺寸的干擾圓圈
cv2.circle(scene, (80, 80), 15, 255, -1)
cv2.circle(scene, (320, 300), 20, 255, -1)

# 2. 建立 60x60 的模板，裡面畫一個與目標相同大小的圓形 (作為搜尋模板)
template = np.zeros((60, 60), dtype=np.uint8)
cv2.circle(template, (30, 30), 30, 255, -1)

# 3. 執行模板匹配 (使用首選的歸一化相關係數匹配)
result_map = cv2.matchTemplate(scene, template, cv2.TM_CCOEFF_NORMED)

# 4. 尋找匹配得分最高的位置
min_val, max_val, min_loc, max_loc = cv2.minMaxLoc(result_map)

# 5. 取得定位座標與寬高
# 對於 TM_CCOEFF_NORMED，最大值位置 max_loc 即為最佳匹配點的左上角座標
top_left = max_loc
h, w = template.shape[:2]
bottom_right = (top_left[0] + w, top_left[1] + h)

print(f"最大匹配得分: {max_val:.4f}")
print(f"目標物定位座標：左上角 {top_left}，右下角 {bottom_right}")

# 6. 在原圖畫框標記 (轉換為 BGR 彩色以便畫綠色框)
scene_bgr = cv2.cvtColor(scene, cv2.COLOR_GRAY2BGR)
cv2.rectangle(scene_bgr, top_left, bottom_right, (0, 255, 0), 2)

# 顯示匹配結果
cv2.imshow('Scene Matching Result', scene_bgr)
cv2.waitKey(0)
cv2.destroyAllWindows()
```
