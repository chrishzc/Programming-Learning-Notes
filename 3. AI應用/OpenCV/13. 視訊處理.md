# OpenCV 視訊讀取、寫入與控制

> [!ABSTRACT]
> 本篇筆記介紹如何使用 OpenCV 進行即時相機串流讀取、讀取視訊檔案、查詢與設定視訊屬性，以及將處理後的影格輸出儲存為視訊檔案（`cv2.VideoWriter`）的實作方法。

---

## 1. 讀取視訊與相機串流 (`cv2.VideoCapture`)

在 OpenCV 中，讀取影像序列（不論是即時相機或現有的影片檔）均是透過建立一個 `cv2.VideoCapture` 類別實例來完成。

### 1.1 核心操作流程
1. **建立物件**：`cap = cv2.VideoCapture(device_or_file)`。
   - 傳入**整數**（如 `0`, `1`）：連接到對應的網路攝影機裝置（`0` 代表第一隻預設相機）。
   - 傳入**字串**（路徑）：開啟指定的影片檔案。
2. **檢查連線**：使用 `cap.isOpened()` 確認相機是否正確初始化。
3. **讀取影格**：在無窮迴圈中呼叫 `ret, frame = cap.read()`。
   - `ret`：布林值。`True` 代表成功接收影像，`False` 代表失敗（如影片播畢或相機被拔除）。
   - `frame`：該影格的 NumPy 影像陣列。
4. **釋放資源**：播畢或退出後，呼叫 `cap.release()` 釋放相機與系統資源。

---

## 2. 視訊屬性控制 (get & set)

可以透過 `get(propId)` 獲取視訊屬性，或使用 `set(propId, value)` 對支援的相機進行屬性設定（例如調整曝光、修改解析度等）。

### 2.1 常用屬性代碼對照表
| 屬性名稱 (Python API) | 常數代碼 | 說明與作用 |
| :--- | :---: | :--- |
| `cv2.CAP_PROP_POS_MSEC` | 0 | 視訊檔案當前播放的時間位置（毫秒）。 |
| `cv2.CAP_PROP_POS_FRAMES` | 1 | 當前影格的索引（從 0 開始計算的影格數）。 |
| `cv2.CAP_PROP_POS_AVI_RATIO` | 2 | 視訊檔案播放的相對進度（0 為開始，1 為結束）。 |
| **`cv2.CAP_PROP_FRAME_WIDTH`** | 3 | 視訊串流的影格**寬度**（畫素）。 |
| **`cv2.CAP_PROP_FRAME_HEIGHT`** | 4 | 視訊串流的影格**高度**（畫素）。 |
| **`cv2.CAP_PROP_FPS`** | 5 | 影片的影格率（FPS，每秒張數）。 |
| **`cv2.CAP_PROP_FOURCC`** | 6 | 視訊編解碼器的四字元代碼（FourCC）。 |
| **`cv2.CAP_PROP_FRAME_COUNT`** | 7 | 影片檔案的總影格數。 |
| `cv2.CAP_PROP_BRIGHTNESS` | 10 | 影像亮度（僅適用於支援的相機設備）。 |
| `cv2.CAP_PROP_CONTRAST` | 11 | 影像對比度（僅適用於相機）。 |
| `cv2.CAP_PROP_SATURATION` | 12 | 影像飽和度（僅適用於相機）。 |
| `cv2.CAP_PROP_EXPOSURE` | 15 | 相機曝光度。 |

## 3. 儲存與寫入視訊 (`cv2.VideoWriter`)

若要將處理後的影像串流存檔，必須建立一個 `cv2.VideoWriter` 物件。

### 3.1 語法
```python
out = cv2.VideoWriter(filename, fourcc, fps, frameSize[, isColor])
```
- `filename`：輸出影片的路徑名稱（如 `output.mp4`）。
- **`fourcc`**：四字元代碼（FourCC），用來指定影片的編解碼器。
  - 在 Python 中，可使用 `cv2.VideoWriter_fourcc(*'codec')` 來指定。常見的編碼有：
    - `*'mp4v'`：輸出 `.mp4` 檔案（相容性最好）。
    - `*'XVID'`：輸出 `.avi` 檔案。
- `fps`：寫入的影格率。
- `frameSize`：寫入影格的寬高尺寸元組 `(Width, Height)`。
  > [!WARNING]
  > **尺寸一致性**：寫入 `VideoWriter` 的影像尺寸，必須與建立物件時指定的 `frameSize` **完全一致**，否則寫入的影片會損毀或無法開啟。

## 4. 視訊處理實戰範例

### 4.1 範例 A：即時相機預覽與解析度修改
此程式開啟第一隻相機，調整解析度為 1280x720，即時將畫面轉為灰階後輸出，並偵測鍵盤事件：

```python
import cv2

# 1. 建立相機連線
cap = cv2.VideoCapture(0)

if not cap.isOpened():
    print("錯誤：無法開啟相機")
    exit()

# 2. 設定相機解析度為 1280x720
cap.set(cv2.CAP_PROP_FRAME_WIDTH, 1280)
cap.set(cv2.CAP_PROP_FRAME_HEIGHT, 720)

print("正在讀取相機，按下 'ESC' 鍵退出...")

while True:
    # 讀取影格
    ret, frame = cap.read()
    if not ret:
        print("無法取得影格")
        break
        
    # 將彩色影像轉為灰階影像
    gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
    
    # 顯示畫面
    cv2.imshow('Gray Camera Preview', gray)
    
    # waitKey(1) 暫停 1ms，若按下 ESC (ASCII 27) 則跳出
    if cv2.waitKey(1) == 27:
        break

# 釋放資源並關閉視窗
cap.release()
cv2.destroyAllWindows()
```

### 4.2 範例 B：讀取影片檔案並加工存檔
此程式讀取影片檔案，將其畫面尺寸縮小為二分之一，並重新儲存為新的 MP4 影片：

```python
import cv2

# 1. 開啟影片檔案
cap = cv2.VideoCapture('input.mp4')

if not cap.isOpened():
    print("無法開啟影片檔案")
    exit()

# 2. 獲取原影片之屬性資訊
fps = cap.get(cv2.CAP_PROP_FPS)
width = int(cap.get(cv2.CAP_PROP_FRAME_WIDTH))
height = int(cap.get(cv2.CAP_PROP_FRAME_HEIGHT))

# 計算目標尺寸 (縮小為二分之一)
target_size = (width // 2, height // 2)

# 3. 建立 VideoWriter 物件，指定 mp4v 編碼與目標尺寸
fourcc = cv2.VideoWriter_fourcc(*'mp4v')
out = cv2.VideoWriter('output_half.mp4', fourcc, fps, target_size)

while True:
    ret, frame = cap.read()
    if not ret:
        print("影片播放完畢")
        break
        
    # 縮小畫面尺寸
    resized_frame = cv2.resize(frame, target_size, interpolation=cv2.INTER_AREA)
    
    # 將處理後的影格寫入影片檔案
    out.write(resized_frame)
    
    # 預覽輸出進度
    cv2.imshow('Processing Window', resized_frame)
    if cv2.waitKey(10) == 27:
        break

# 釋放所有資源
cap.release()
out.release()
cv2.destroyAllWindows()
```
