# OpenCV 影像二值化與閾值處理

> [!ABSTRACT]
> 本篇筆記介紹影像二值化（Thresholding，即「門檻值」或「閾值」處理）的原理。內容包含標準固定閾值二值化（包含 5 種主要模式）、大津演算法（Otsu 自動閾值），以及適用於光照不均環境的自適應二值化（Adaptive Thresholding）實作。

> [!NOTE]
> 筆記標題中的「設定值處理」為 Threshold 的直譯，在電腦視覺中通常稱為**閾值處理**或**二值化**。

---

## 1. 標準固定二值化 (`cv2.threshold`)

當畫素值高於門檻值（或低於門檻值，依模式而定）時，將其設定為特定值（通常是白色 `255`），否則設定為另一值（通常是黑色 `0`）。

### 1.1 語法
```python
ret, dst = cv2.threshold(src, thresh, maxval, type)
```
- **`src`**：輸入影像。**必須是單通道影像**，通常為灰階圖（Grayscale）。
- **`thresh`**：手動設定的門檻值（閾值）。
- **`maxval`**：當畫素值滿足條件時被賦予的新值（通常為 `255`）。
- **`type`**：二值化模式標記（Flags）。
- **回傳值**：
  - **`ret`**：最終使用的門檻值（手動設定時即為 `thresh`；若使用大津演算法則為自動計算出的最佳值）。
  - **`dst`**：二值化後的輸出影像。

---

### 1.2 五大二值化模式對照表
假設設定門檻值 `thresh = 127`，最大值 `maxval = 255`：

| 模式標記 (Flags) | 核心數學邏輯 | 像素 100 的命運<br>(暗點 $\le 127$) | 像素 200 的命運<br>(亮點 $> 127$) | 視覺效果與主要用途 |
| :--- | :--- | :---: | :---: | :--- |
| **`cv2.THRESH_BINARY`** | 大於閾值變 `maxval`；其餘變 `0`。 | `0` (純黑) | `255` (純白) | **最常用**。產生極端黑白分明影像，常用於文字偵測或輪廓提取。 |
| **`cv2.THRESH_BINARY_INV`** | 大於閾值變 `0`；其餘變 `maxval`。 | `255` (純白) | `0` (純黑) | 效果與上者相反。常用於背景較亮、物體較暗，想將物體轉為白色的場景。 |
| **`cv2.THRESH_TRUNC`** | 大於閾值的**強行降為閾值**；其餘保持不變。 | `100` (原樣) | `127` (截斷) | 亮部被削平，暗部細節保留。常用於調整過曝影像。 |
| **`cv2.THRESH_TOZERO`** | 大於閾值的保持不變；其餘**直接歸零**。 | `0` (純黑) | `200` (原樣) | 抹去所有低於閾值的暗部，只保留高亮部分的原本色彩與漸層。 |
| **`cv2.THRESH_TOZERO_INV`** | 大於閾值的**直接歸零**；其餘保持不變。 | `100` (原樣) | `0` (純黑) | 抹去所有高於閾值的亮部，只保留暗部細節。 |

## 2. 大津自動閾值演算法 (Otsu)

大津演算法（Otsu's Method）假設影像直方圖呈現雙峰（由前景色和背景色組成）。它透過統計學方法自動尋找一個能使「**最大類間變異數**（Between-Class Variance）」最大的閾值，將前景和背景做最完美的分割，免去人工測試閾值的麻煩。

> [!WARNING]
> **大津演算法限制**
> Otsu 仍屬於「全域二值化」，適用於**光照均勻**但背景與主體界線分明的影像。若光照不均，Otsu 的效果會大幅下降。

### 2.1 程式範例
在 `cv2.threshold` 的 `type` 參數中，將基本二值化常數加上 `cv2.THRESH_OTSU` 即可啟用。此時輸入的 `thresh` 參數可隨意設為 `0`，演算法會自動計算最佳值並由 `ret` 回傳。

```python
import cv2

# 載入灰階圖
gray_img = cv2.imread('document.jpg', cv2.IMREAD_GRAYSCALE)

# 使用 Otsu 二值化 (THRESH_BINARY + THRESH_OTSU)
ret_val, otsu_img = cv2.threshold(gray_img, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)

print(f"Otsu 自動計算出的最佳門檻值為: {ret_val}")
```

## 3. 自適應二值化 (Adaptive Thresholding)

當影像中存在**光照不均勻、有陰影或局部反光**時，全域門檻值（如手動設定或 Otsu）會導致陰影區全黑或反光區全白。此時必須使用自適應二值化，它會根據像素周圍局部區域的亮度，**為每個像素獨立計算一個門檻值**。

> [!IMPORTANT]
> **自適應二值化是一個「獨立的函數」**
> 原筆記將其誤列為 `cv2.threshold` 的 `type` 標記。在實務上，它使用專屬函數 **`cv2.adaptiveThreshold`**，不可混淆。

### 3.1 語法
```python
dst = cv2.adaptiveThreshold(src, maxValue, adaptiveMethod, thresholdType, blockSize, C)
```
- **`src`**：輸入灰階影像（8-bit 單通道）。
- **`maxValue`**：滿足條件時賦予的新值（如 `255`）。
- **`adaptiveMethod`**：區域門檻值計算方法：
  - **`cv2.ADAPTIVE_THRESH_MEAN_C`**：計算鄰域區塊的**平均值**，再減去 `C`。
  - **`cv2.ADAPTIVE_THRESH_GAUSSIAN_C`**：計算鄰域區塊的**高斯加權和**，再減去 `C`。
- **`thresholdType`**：二值化類型，必須為 `cv2.THRESH_BINARY` 或 `cv2.THRESH_BINARY_INV`。
- **`blockSize`**：局部計算區塊大小。**必須是奇數且大於 1**（例如 `3, 5, 7, 11` 等）。值越大，計算的範圍越廣。
- **`C`**：常數（偏移值）。從計算出的平均值或加權平均值中扣除的數值，用於微調最終二值化效果。

### 3.2 程式範例
```python
import cv2

gray_img = cv2.imread('shadow_paper.jpg', cv2.IMREAD_GRAYSCALE)

# 使用高斯自適應二值化
# 局部區域大小為 11x11，微調常數 C 設為 2
adaptive_gaussian = cv2.adaptiveThreshold(
    gray_img, 
    255, 
    cv2.ADAPTIVE_THRESH_GAUSSIAN_C, 
    cv2.THRESH_BINARY, 
    11, 
    2
)

cv2.imshow('Adaptive Thresholding', adaptive_gaussian)
cv2.waitKey(0)
cv2.destroyAllWindows()
```
