# 13. 迭代器與生成器 (yield)

> [!ABSTRACT]
> 本章介紹 Python 中用於處理序列與大數據流的進階工具——迭代器（Iterator）與生成器（Generator）。我們將解析可迭代對象與迭代器的內在區別、說明 `yield` 關鍵字的執行掛起機制、比較列表與生成器的記憶體消耗，並提供在實務中流式讀取超大型數據檔案的實戰程式碼。

---

## 一、 迭代器 (Iterator) 與可迭代對象 (Iterable)

在 Python 中，循環遍歷資料結構的底層是建立在迭代器協議之上的：

- **可迭代對象 (Iterable)**：
  - **定義**：內部實現了 `__iter__()` 方法的物件。例如：`list`、`tuple`、`dict`、`set`、`str`。
  - **特點**：可以被 `for` 迴圈遍歷，但它們本身不是迭代器，無法直接呼叫 `next()` 獲取下一個值。
- **迭代器 (Iterator)**：
  - **定義**：內部同時實現了 `__iter__()` 與 `__next__()` 方法的物件。
  - **特點**：可以使用 `next()` 函數逐一獲取元素。當沒有元素可返回時，會拋出 `StopIteration` 異常。

### 手動迭代範例
```python
my_list = [10, 20]

# 將可迭代對象轉為迭代器
my_iter = iter(my_list) # 相當於 my_list.__iter__()

# 手動獲取下一個元素
print(next(my_iter)) # 輸出: 10
print(next(my_iter)) # 輸出: 20
# print(next(my_iter)) # 拋出 StopIteration 異常
```

## 二、 生成器 (Generator) 與 yield 關鍵字

生成器是一種特殊的迭代器，它是透過更簡單的語法來動態生成序列。

- **生成器函數**：只要函式內部包含了 **`yield`** 關鍵字，該函式就被稱為生成器函數。
- **執行掛起機制 (Lazy Evaluation)**：
  - 當呼叫生成器函數時，它不會立即執行函式體，而是回傳一個生成器物件。
  - 當第一次呼叫 `next()` 時，函數開始執行，直到遇到 `yield`。
  - 遇到 `yield` 時，函數會**暫停（掛起）**執行，並將 `yield` 後面的值回傳。此時，函數內部的所有區域變數狀態都會被保留。
  - 下一次呼叫 `next()` 時，函數會從上次暫停的 `yield` 後方第一行程式碼**恢復執行**，直到遇到下一個 `yield` 或執行結束。

```python
def simple_generator():
    print("開始執行...")
    yield 1
    print("恢復執行...")
    yield 2
    print("結束執行...")

gen = simple_generator() # 回傳生成器物件，此時不列印任何內容
print(next(gen)) # 列印 "開始執行..."，回傳 1
print(next(gen)) # 列印 "恢復執行..."，回傳 2
```

## 三、 生成器與列表的記憶體消耗對比

當我們處理數以百萬計的數據時，如果使用列表解析式（List Comprehension），Python 會一次性將所有資料計算好並載入內存中，這會佔用巨大的內存空間。而使用生成器表達式（Generator Expression），資料只會在需要時才動態計算，內存佔用極小。

```python
import sys

# 1. 列表解析式 (一次性載入記憶體)
list_comp = [x for x in range(1000000)]
print(f"列表佔用空間: {sys.getsizeof(list_comp)} 位元組") # 約 8MB

# 2. 生成器表達式 (括號包裹，動態產生)
gen_expr = (x for x in range(1000000))
print(f"生成器佔用空間: {sys.getsizeof(gen_expr)} 位元組") # 約 100~200 位元組！
```

## 四、 實戰演練：流式讀取超大型數據檔

以下程式碼示範如何利用生成器，逐行讀取一個容量數 GB 的日誌或 CSV 檔案，並在不爆記憶體的狀況下過濾並處理特定關鍵字數據：

```python
def read_large_file(file_path):
    """生成器：逐行讀取大型檔案，避免一次性載入內存"""
    with open(file_path, "r", encoding="utf-8") as file:
        for line in file:
            yield line.strip()

# 實務使用
log_file_path = "huge_server_log.txt"

# 建立生成器對象 (此時並未讀取檔案)
log_generator = read_large_file(log_file_path)

# 開始流式處理數據 (一次記憶體中只保留一行資料)
error_count = 0
for log_line in log_generator:
    if "[ERROR]" in log_line:
        error_count += 1
        print(f"發現錯誤日誌: {log_line}")
        
        # 達到特定數量限制時中斷
        if error_count >= 10:
            print("已達分析上限，停止讀取。")
            break
```
