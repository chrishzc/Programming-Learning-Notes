# 9. 執行與錯誤處理 (Exception)

> [!ABSTRACT]
> 本章介紹 Python 中的錯誤類型，掌握以 `try-except-else-finally` 為核心的例外處理機制，並學習如何自訂例外類別與主動觸發例外。

---

## 一、 錯誤類型介紹

在 Python 程式執行過程中，錯誤主要可以分為以下兩大類：

### 1. 語法錯誤 (Syntax Error / Parsing Error)
- **特點**：程式碼不符合 Python 語法規則，直譯器在執行前進行剖析時即會報錯。
- **結果**：程式**完全無法進入執行階段**。
- **範例**：括號未成對、縮排錯誤、`if` 後方遺漏冒號。

### 2. 執行錯誤 (Runtime Error / Exception)
- **特點**：語法正確，但在程式執行過程中，由於邏輯漏洞或外部因素（如檔案不存在、網路中斷）導致程式崩潰。
- **結果**：會產生一個**例外物件 (Exception Object)**。若未進行捕捉處理，程式會立即中斷執行。
- **範例**：除以零 (`ZeroDivisionError`)、索引超出範圍 (`IndexError`)、找不到檔案 (`FileNotFoundError`)。

---

## 二、 例外處理機制 (Exception Handling)

Python 使用 `try-except-else-finally` 結構來捕捉並處理例外，防止程式崩潰：

```python
try:
    # 可能會產生例外的程式碼
    num = int(input("請輸入一個整數: "))
    result = 10 / num
except ValueError as e:
    # 當 try 內發生 ValueError 時執行的區塊
    print(f"❌ 輸入錯誤！請確保您輸入的是數字。錯誤訊息: {e}")
except ZeroDivisionError:
    # 當 try 內發生 ZeroDivisionError 時執行的區塊
    print("❌ 錯誤！除數不能為零。")
except Exception as e:
    # 捕捉其他所有未預期的例外 (Exception 是所有標準例外的父類別)
    print(f"❌ 發生未知的錯誤: {e}")
else:
    # 當 try 內「沒有發生任何例外」時，才會執行的程式碼
    print(f"✅ 計算成功！結果為: {result}")
finally:
    # 無論 try 內是否發生例外，都「一定會執行」的清理程式碼
    # 常用於釋放系統資源，例如關閉檔案或資料庫連線
    print("🧹 清理工作完成。")
```

## 三、 自訂例外類別與主動觸發 (raise)

有些邏輯上的錯誤（例如：使用者輸入的年齡為負數），對 Python 直譯器而言是合法的數值，但在業務邏輯上是錯誤的。此時我們可以**自訂例外類別**並使用 **`raise`** 主動觸發錯誤。

### 自訂錯誤類別步驟
1. 建立一個新類別，並**繼承自官方的 `Exception` 類別**。
2. 在程式碼中，符合自訂錯誤的條件時，使用 `raise 類別名稱("錯誤說明")` 來拋出例外。

### 實作範例
```python
# 1. 自訂例外類別，繼承 Exception
class InvalidAgeError(Exception):
    def __init__(self, age, message="年齡必須在 0 到 150 歲之間"):
        self.age = age
        self.message = message
        super().__init__(self.message)

# 2. 定義一個會主動觸發例外的函數
def register_user(username, age):
    if age < 0 or age > 150:
        # 主動拋出自訂例外
        raise InvalidAgeError(age, f"無效的年齡: {age} 歲！註冊失敗。")
    print(f"🎉 使用者 {username} (年齡: {age}) 註冊成功！")

# 3. 測試例外處理
try:
    register_user("Alice", 25)  # 成功
    register_user("Bob", -5)    # 觸發例外
except InvalidAgeError as e:
    print(f"⚠️ 攔截到自訂錯誤：{e.message} (您傳入的值為: {e.age})")
```
