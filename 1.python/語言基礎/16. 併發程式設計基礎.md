# 16. 併發程式設計基礎

> [!ABSTRACT]
> 本章介紹 Python 中用於加速執行效率與處理多任務的進階程式技術——併發程式設計（Concurrency）。內容涵蓋併發與並行的差別、全域解釋器鎖（GIL）的運作限制、多執行緒（Threading）與多處理序（Multiprocessing）的選用對照，以及非同步協程（Asyncio）的基礎觀念，並提供基於執行緒池的網絡請求加速實戰程式碼。

---

## 一、 併發、並行與 Python GIL 鎖

在處理多任務時，我們常會遇到以下兩個核心觀念：

- **併發 (Concurrency)**：
  - 指系統有處理多個任務的能力，但不一定同時執行。作業系統透過在微秒級別快速輪替（切換）執行多個任務，給使用者一種「多個程式同時在跑」的直覺感受（常用於 I/O 密集型任務）。
- **並行 (Parallelism)**：
  - 指系統有「同時」執行多個任務的實體能力。這需要多核心 CPU 支援，每個核心實體執行一個不同的任務（常用於 CPU 密集型任務）。
- **全域解釋器鎖 (Global Interpreter Lock, GIL)**：
  - **定義**：這是 CPython（Python 預設解釋器）底層的 mutex 互斥鎖。
  - **限制**：**在同一個時間點，只允許一個執行緒執行 Python 的字節碼（Bytecode）。**
  - **影響**：這使得 Python 的多執行緒（Multi-threading）**無法利用多核心 CPU 進行真正的並行運算**。如果您的任務是 CPU 計算密集型，多執行緒甚至會因為頻繁切換鎖而變得比單執行緒更慢。但如果您的任務是等待網路回應、讀寫硬碟等 I/O 密集型，多執行緒依然能大幅節省等待時間。

---

## 二、 多執行緒 vs 多處理序 (Threading vs Multiprocessing)

為了解決 GIL 的限制並適應不同類型的加速任務，Python 提供了兩種併發模式：

| 對比項目 | 多執行緒 (Multi-threading) | 多處理序 (Multi-processing) |
| :--- | :--- | :--- |
| **底層物件** | `threading.Thread` | `multiprocessing.Process` |
| **內存共享** | 共享同一個行程的記憶體空間，變數可直接共享，但需注意執行緒安全（Race Condition）。 | 擁有完全隔離的獨立內存空間與 Python 解釋器，變數不共享，需透過 IPC（如 Queue, Pipe）通訊。 |
| **系統開銷** | 輕量，建立與切換速度極快。 | 較重，建立與銷毀進程的開銷較大。 |
| **GIL 限制** | 受制於 GIL。 | 繞過 GIL，每個進程擁有自己的解釋器與獨立的 GIL。 |
| **適用場景** | **I/O 密集型任務**（如網路爬蟲、下載圖片、資料庫讀寫、Web 服務 API 請求）。 | **CPU 密集型任務**（如高畫質影像處理、矩陣計算、機器學習模型推論、影片壓縮）。 |

---

## 三、 協程與非同步 I/O (asyncio)

除了多執行緒外，現代 Python 也極力推薦使用**協程（Coroutine）**來實現高併發。

- **協程原理**：
  - 協程是一種**用戶態的輕量級執行緒**。它不需要作業系統來進行上下文切換，而是透過程式碼在單一執行緒中主動「合作式」讓出 CPU 控制權。
  - 當遇到 I/O 等待時，協程會主動掛起，讓其他協程繼續工作，等 I/O 結束後再恢復執行。
- **關鍵語法**：
  - 使用 `async def` 宣告一個非同步協程函式。
  - 使用 `await` 標註耗時的非同步等待操作。
  - **優勢**：在處理百萬級的併發連接（如聊天伺服器、大型非同步爬蟲）時，其記憶體消耗與系統開銷遠比多執行緒小得多。

## 四、 實戰演練：多執行緒爬蟲請求加速

以下程式碼示範如何使用 Python 內建的 `concurrent.futures` 執行緒池，來同時併發請求多個網頁，大幅縮短依次排隊下載的等待時間（I/O 密集型任務）：

```python
import urllib.request
from concurrent.futures import ThreadPoolExecutor
import time

# 待爬取的目標網址列表
urls = [
    "https://www.google.com",
    "https://www.python.org",
    "https://www.github.com",
    "https://www.wikipedia.org"
]

def fetch_url(url):
    """網路 I/O 任務：下載網頁內容"""
    print(f"[開始下載] {url}")
    try:
        # 模擬網路請求
        with urllib.request.urlopen(url, timeout=5) as conn:
            data = conn.read()
            print(f"[下載成功] {url}: 共 {len(data)} 位元組")
            return len(data)
    except Exception as e:
        print(f"[下載失敗] {url}: {e}")
        return None

# ---- 1. 單執行緒依次下載 ----
start_time = time.time()
for url in urls:
    fetch_url(url)
print(f"⌛ 單執行緒總共耗時: {time.time() - start_time:.2f} 秒\n")

# ---- 2. 使用執行緒池併發下載 ----
start_time = time.time()
# 開啟最多 4 個執行緒的執行緒池
with ThreadPoolExecutor(max_workers=4) as executor:
    # map 方法會自動將任務分派給執行緒池中的執行緒併發執行
    results = executor.map(fetch_url, urls)

print(f"⚡ 多執行緒併發總共耗時: {time.time() - start_time:.2f} 秒")
```
