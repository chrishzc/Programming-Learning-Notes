# 2. 設定虛擬環境 (venv)

> [!ABSTRACT]
> 學習如何為每個 Python 專案建立獨立的虛擬環境，以解決套件版本不相容的問題，並掌握 UV 工具的基本指令。

---

## 一、 為什麼需要虛擬環境？
每一個專案可能會有不同的 **版本相容性** 需求。在專案目錄中建立虛擬環境，可以避免不同專案間的套件版本互相衝突，確保程式執行的穩定性。

---

## 二、 常用系統指令 (Terminal)
在操作虛擬環境前，需熟悉終端機的基本指令：
- `cd [路徑]`：讀取並進入路徑。
- `cd ..`：進入當前目錄的上一層。
- `dir`：查詢該目錄下的所有檔案與資料夾。

---

## 三、 使用 UV 管理環境與套件
UV 是目前效率極高的套件管理工具，以下為常用指令：

```bash
# 專案管理
uv init "專案名稱"    # 新增一個 Python 專案
uv venv             # 建立虛擬環境 (會產生 .venv 目錄)
uv sync             # 依照設定檔 (pyproject.toml) 重現環境變數與套件

# 套件管理
uv add "套件名稱"     # 安裝指定套件
uv remove "套件名稱"  # 移除指定套件
uv pip list          # 查看當前環境已安裝的所有套件
```

---

## 四、 環境啟用與檢查
> [!IMPORTANT]
> **如何啟用 venv？**
> 如果使用了 `uv sync` 還原環境後，需要執行啟用腳本：
> - **指令路徑**：`.venv/Scripts/activate`
> - **檢查方式**：檢查 Terminal 或 VS Code 右下角是否已顯示進入 `.venv` 環境。

---

## 五、 實戰案例：執行 GitHub 專案
當下載他人專案時，應遵循以下步驟：
1. 建立虛擬環境（`uv venv`）。
2. 依照套件安裝清單（如 `requirements.txt` 或設定檔）還原環境。

### 驗證虛擬環境套件是否成功載入：
在虛擬環境啟用後，建立一個測試檔案 `test_env.py`，寫入以下程式碼並執行：

```python
# 測試外部套件是否能在當前虛擬環境中被成功調用
try:
    # 假設我們安裝了 requests 套件
    import requests
    response = requests.get("https://api.github.com")
    print("✅ 套件導入成功！")
    print(f"GitHub API 回傳狀態碼: {response.status_code}")
except ImportError:
    print("❌ 導入失敗！請確認是否已經啟用虛擬環境 (.venv)，並使用 uv add requests 或 pip install requests 安裝套件。")
```
