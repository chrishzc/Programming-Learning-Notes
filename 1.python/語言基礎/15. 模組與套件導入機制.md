# 15. 模組與套件導入機制

> [!ABSTRACT]
> 本章介紹 Python 中的模組（Module）與套件（Package）系統，以及其底層導入（Import）機制。我們將解析 `__init__.py` 的歷史與作用、比較絕對導入與相對導入的語法特徵與報錯陷阱，並說明 `sys.path` 的運作原理，以協助開發者快速排除實務中常見的 `ModuleNotFoundError` 導包錯誤。

---

## 一、 模組 (Module) 與套件 (Package) 的定義

隨著程式規模擴大，我們不能把所有代碼寫在同一個檔案中。Python 提供了模組與套件來進行代碼結構化：

- **模組 (Module)**：
  - 一個副檔名為 `.py` 的 Python 檔案就是一個模組。檔案名稱即為模組名稱。
- **套件 (Package)**：
  - 套件是一個包含多個模組的**目錄（資料夾）**。
  - **`__init__.py` 的作用**：
    - 在 Python 3.3 之前，目錄下必須包含一個 `__init__.py` 檔案，Python 才會將該目錄識別為一個套件（之後的版本則支援無 `__init__.py` 的命名空間套件，但建立它仍是最佳實務）。
    - 當套件被導入時（例如 `import mypackage`），`__init__.py` 中的代碼會自動且最先被執行。它通常被用來進行套件內部的初始化配置，或在 `__all__` 列表中定義允許外部使用 `from mypackage import *` 導入的模組名單。

---

## 二、 絕對導入與相對導入 (Absolute vs Relative Imports)

當模組之間需要互相引用時，有兩種導入路徑的寫法：

### 1. 絕對導入 (Absolute Imports)
- **語法**：從專案的根目錄（頂層目錄）開始指定完整路徑。
  ```python
  from myproject.utils import db_helper
  from myproject.models.user import User
  ```
- **優點**：路徑清晰、可讀性高，即使目前模組的實體位置在專案內移動，只要根目錄不變，導入就不會出錯。**官方推薦在多數應用中優先使用絕對導入。**

### 2. 相對導入 (Relative Imports)
- **語法**：使用 `.`（代表當前目錄）與 `..`（代表上一層目錄）來導入相鄰的模組。
  ```python
  from . import config       # 導入當前目錄的 config.py
  from ..utils import helper # 導入上一層資料夾中的 utils.py 中的 helper
  ```
- **致命陷阱**：
  > [!WARNING]
  > **Attempted relative import in non-package 錯誤**
  > 相對導入是建立在「模組作為套件的一部分被加載」的前提下。如果您**直接執行**（例如在終端機輸入 `python my_sub_module.py`）一個含有相對導入的檔案，Python 會因為無法確定其上層套件架構而拋出 `ValueError: Attempted relative import in non-package` 錯誤。
  >
  > 因此，含有相對導入的模組，必須透過專案根目錄的進入點（如 `main.py`）進行間接呼叫，或使用 `python -m mypackage.my_sub_module` 方式來當作模組運行。

## 三、 sys.path 與導包錯誤排查

當我們在程式中執行 `import abc` 時，Python 解釋器會依序去以下三個地方尋找名為 `abc` 的模組：
1. **內建模組**（如 `sys`、`os`、`time`）。
2. **`sys.path` 列表中包含的目錄路徑**：
   - 包含：目前執行指令的當前路徑。
   - 包含：環境變數 `PYTHONPATH` 中設定的路徑。
   - 包含：第三方套件安裝路徑（如 `site-packages`）。

### 1. 動態解決 ModuleNotFoundError
當您嘗試導入一個不在同資料夾下的自訂模組而報錯時，可以透過動態修改 `sys.path` 來解決：
```python
import sys
import os

# 1. 取得專案根目錄的絕對路徑
project_root = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))

# 2. 將根目錄加入 sys.path 搜尋路徑中
if project_root not in sys.path:
    sys.path.append(project_root)

# 3. 此時即可成功導入其他目錄下的模組
from models.user import User
```
