# 14. 裝飾器 (Decorator)

> [!ABSTRACT]
> 本章介紹 Python 中強大的元程式設計（Metaprogramming）工具——裝飾器（Decorator）。裝飾器能讓我們在不修改現有函式代碼的狀況下，動態擴充或修改其功能。本章將從閉包（Closure）與一等公民函式出發，解析 `@` 語法糖的運作本質、探討帶有參數與多重裝飾器的進階寫法，並提供用於計算函式執行時間的計時裝飾器實戰程式碼。

---

## 一、 閉包與函式作為一等公民

在學習裝飾器之前，必須先理解 Python 中函式的特殊地位：

- **一等公民 (First-Class Citizen)**：
  - 函式在 Python 中可以被當作一般變數使用：它可以被賦值給其他變數、作為參數傳遞給另一個函式，或作為另一個函式的返回值。
- **閉包 (Closure)**：
  - 閉包是指在一個外部函式中定義了一個內部函式，內部函式引用了外部函式的局部變數，且外部函式最終返回了該內部函式。
  - 當外部函式執行結束後，被引用的變數狀態仍會被保留在內存中，不會被資源回收。

```python
def outer_func(msg):
    # 外部函式的局部變數
    def inner_func():
        # 內部函式引用外部變數
        print(f"訊息: {msg}")
    return inner_func # 返回內部函式對象

my_closure = outer_func("Hello")
my_closure() # 輸出: 訊息: Hello
```

## 二、 裝飾器運作原理與 @ 語法糖

裝飾器本質上就是一個**接收函式作為參數並返回一個新函式**的閉包。它可以在不修改原函式的前提下，在原函式執行前後插入自訂的邏輯（如日誌紀錄、權限驗證、快取等）。

### 1. 手動裝飾函式 (底層原理)
```python
def my_decorator(func):
    def wrapper():
        print("[前置處理] 準備執行函式...")
        func()
        print("[後置處理] 函式執行完畢。")
    return wrapper

def hello():
    print("哈囉，世界！")

# 手動包裝
decorated_hello = my_decorator(hello)
decorated_hello()
```

### 2. 使用 `@` 語法糖
Python 提供了簡潔的 `@` 符號語法糖，可以直接在宣告函式時套用裝飾器，這與手動包裝在效果上是完全等價的：
```python
@my_decorator
def hello():
    print("哈囉，世界！")

hello() # 呼叫時會自動套用裝飾器邏輯
```

## 三、 進階裝飾器：傳參、多重裝飾與 functools.wraps

### 1. 處理原函式的參數 (`*args` 與 `**kwargs`)
為了讓裝飾器適用於任何參數結構的函式，內部包裝函數（`wrapper`）必須使用萬用引數：
```python
def logger(func):
    def wrapper(*args, **kwargs):
        print(f"準備執行: {func.__name__}...")
        result = func(*args, **kwargs) # 執行並獲取返回值
        print("執行結束。")
        return result
    return wrapper
```

### 2. 裝飾器本身帶參數 (三層巢狀函式)
如果我們希望裝飾器本身能接收設定參數（例如自訂日誌的前綴），需要再包裝一層外部函式：
```python
def repeat(num_times):
    def decorator_repeat(func):
        def wrapper(*args, **kwargs):
            for _ in range(num_times):
                func(*args, **kwargs)
        return wrapper
    return decorator_repeat

@repeat(num_times=3)
def greet(name):
    print(f"哈囉 {name}")

greet("Alice") # 會連續印出三次 "哈囉 Alice"
```

### 3. 使用 functools.wraps
當我們使用裝飾器包裝函式後，原函式的元數據（如 `__name__` 與 `__doc__`）會被改寫成 `wrapper`。為了保留原函式的資訊，必須導入 `functools.wraps`：
```python
from functools import wraps

def my_decorator(func):
    @wraps(func) # 保留原函式的元數據
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper
```

## 四、 實戰演練：函式執行時間計算裝飾器

以下為一條極為實用的實務程式碼，撰寫一個能夠自動量測並列印任何函式執行耗時的計時裝飾器：

```python
import time
from functools import wraps

def timer_decorator(func):
    """計時裝飾器：測量函式執行的精確時間"""
    @wraps(func)
    def wrapper(*args, **kwargs):
        start_time = time.perf_counter() # 精確起始時間
        
        # 執行原函式
        result = func(*args, **kwargs)
        
        end_time = time.perf_counter() # 精確結束時間
        execution_time = end_time - start_time
        print(f"[⏱️ 計時器] 函式 '{func.__name__}' 執行耗時: {execution_time:.6f} 秒")
        return result
    return wrapper

# 實務測試：計算大範圍累加所耗時間
@timer_decorator
def calculate_sum(limit):
    total = 0
    for i in range(limit):
        total += i
    return total

# 執行
sum_result = calculate_sum(10000000)
```
