## 一、 文字探勘的核心應用

文字探勘技術旨在將非結構化的文本轉化為可分析的結構化數據，常見應用場景如下：

### 1. 搜尋與匹配 (Searching and Matching)
搜尋引擎（如 Elasticsearch, Apache Solr）利用索引技術快速尋找與關鍵字匹配的文件。
- **建構索引**：對大量文件進行預處理與索引，形成文件集（Documents）。
- **關鍵字查詢**：使用者輸入查詢關鍵字，系統在後台進行分詞與特徵匹配。
- **排名與顯示**：比對查詢詞與文件的相關性並進行排名（如使用 BM25 演算法），將最相關的結果呈現給使用者。

### 2. 訊息抽取 (Information Extraction)
從海量文本中自動找出專有名詞、實體關係與特定事件，並將其歸類至預定義的語意類別中（如人名、地名、日期等）。

### 3. 訊息分組 (Grouping Information)
對文本進行分類或分群並加上標籤，以利後續的自動化流程。例如電子郵件的垃圾信過濾、自動分門別類與設定處理優先等級。

## 二、 常見文字特徵提取模型

在將文字送入機器學習模型前，必須先進行「數值化」特徵提取。

### 1. 詞袋模型 (Bag of Words, BoW)
詞袋模型不考慮單字在句子中的語法結構與順序，僅統計每個詞項（Term）在各文件中出現的次數（詞頻）。

假設我們有 5 篇文件，且整個文件集總共包含 6 個不重複詞彙，其詞袋矩陣（BoW Matrix）表示如下：

| 文件 \ 詞彙 | 詞 1 (電腦) | 詞 2 (學習) | 詞 3 (視覺) | 詞 4 (人工) | 詞 5 (智慧) | 詞 6 (NLP) |
| :--- | :---: | :---: | :---: | :---: | :---: | :---: |
| **文件 1** | 0 | 3 | 0 | 1 | 1 | 0 |
| **文件 2** | 1 | 0 | 1 | 0 | 0 | 0 |
| **文件 3** | 8 | 2 | 7 | 7 | 8 | 1 |
| **文件 4** | 1 | 1 | 4 | 0 | 0 | 2 |
| **文件 5** | 10 | 0 | 1 | 2 | 0 | 0 |

### 2. TF-IDF 權重模型
TF-IDF (Term Frequency-Inverse Document Frequency) 是一種統計方法，用以評估一個詞語對於整份文件集中的其中一份文件的重要程度。

#### 數學公式定義
- **詞頻 (Term Frequency, TF)**：評估詞 $t$ 在文件 $d$ 中出現的頻率。
  $$\text{TF}(t, d) = \frac{\text{詞 } t \text{ 在文件 } d \text{ 中出現的次數}}{\text{文件 } d \text{ 的總詞數}}$$
- **逆文件頻率 (Inverse Document Frequency, IDF)**：評估詞 $t$ 在整個文件集合中的普遍程度。若詞在多篇文件中重複出現，則其 IDF 權重會降低。
  $$\text{IDF}(t, D) = \log\left( \frac{|D|}{1 + |\{d \in D : t \in d\}|} \right)$$
  *(分母加 1 是為了防止該詞未在任何文件出現時導致分母為零)*
- **TF-IDF 權重**：
  $$\text{TF-IDF}(t, d, D) = \text{TF}(t, d) \times \text{IDF}(t, D)$$

#### 模型特點
1. **重要詞彙篩選**：TF-IDF 傾向於過濾掉在所有文章中都高頻出現的常見詞（如 Stopwords），保留在特定文章中頻率高但整篇語料庫中罕見的關鍵詞。
2. **相似度比對**：在向量空間模型中，常配合**餘弦相似性（Cosine Similarity）**來計算兩份文件之間的語意相似度。
3. **限制**：TF-IDF 認為低頻詞越重要、高頻詞越不重要，這在專業領域文本中可能導致某些核心專業術語（雖高頻但具關鍵意義）被忽視。

#### 💻 Scikit-learn 計算 TF-IDF 實戰

```python
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity

# 1. 準備已分詞的文本資料
corpus = [
    "電腦 視覺 人工智慧 影像 處理",
    "自然語言 處理 機器 學習 影像",
    "人工智慧 自然語言 處理 電腦 學習"
]

# 2. 初始化 TF-IDF 向量化工具
vectorizer = TfidfVectorizer()

# 3. 計算 TF-IDF 矩陣
tfidf_matrix = vectorizer.fit_transform(corpus)

# 4. 取得特徵詞彙列表與矩陣維度
print("特徵詞彙:", vectorizer.get_feature_names_out())
print("矩陣形狀 (文件數, 詞數):", tfidf_matrix.shape)

# 5. 計算文件之間的餘弦相似度 (Cosine Similarity)
similarity_matrix = cosine_similarity(tfidf_matrix)
print("\n文件間相似度矩陣:")
print(similarity_matrix)
```

### 3. 詞向量模型 (Word2Vec)
Word2Vec 透過類神經網路將每個詞映射到一個高維的連續稠密向量空間中（通常為 100~300 維），使語意相近的詞在空間中的距離更加接近。

#### 兩種核心神經網路架構
* **CBOW (Continuous Bag of Words)**：輸入周圍的上下文詞彙，預測當前目標詞的生成機率。
* **Skip-gram**：輸入當前目標詞，預測其周圍上下文詞彙出現的機率。

#### 兩套優化加速機制
* **Hierarchical Softmax**：使用霍夫曼樹（Huffman Tree）降低計算 Softmax 分類時的運算負載。
* **Negative Sampling (負採樣)**：不再對詞彙表中的所有詞計算機率，而是只隨機挑選少數負樣本詞，將多分類問題轉換為高效的二分類邏輯。對於高頻字，選中作為負樣本的機率較大。

#### 💻 Gensim 訓練 Word2Vec 實戰

```python
from gensim.models import Word2Vec

# 1. 準備已分詞的句子集
sentences = [
    ["電腦", "視覺", "影像", "處理", "技術"],
    ["自然語言", "處理", "機器", "學習", "技術"],
    ["人工智慧", "技術", "核心", "電腦", "學習"]
]

# 2. 訓練 Word2Vec 模型
# vector_size: 向量維度, window: 上下文視窗大小, min_count: 忽略低頻詞門檻, sg: 0 代表 CBOW, 1 代表 Skip-gram
model = Word2Vec(sentences, vector_size=50, window=3, min_count=1, sg=0)

# 3. 取得特定詞彙的詞向量
vector = model.wv["電腦"]
print("電腦 的詞向量範例 (前 5 維):", vector[:5])

# 4. 計算兩個詞的語意相似度
sim = model.wv.similarity("電腦", "人工智慧")
print(f"電腦 與 人工智慧 相似度: {sim:.4f}")

# 5. 查找最相似的詞彙
similar_words = model.wv.most_similar("技術", topn=2)
print("與 技術 最相似的詞彙:", similar_words)
```
